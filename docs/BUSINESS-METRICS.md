## 2. 핵심 성과 지표 (KPI) 정의

### 2.1 사용자 지표

| 지표 | 정의 | 계산 방법 |
|------|------|-----------|
| **DAU** | 일간 활성 사용자 | 해당 일에 앱/웹 방문한 고유 사용자 |
| **WAU** | 주간 활성 사용자 | 해당 주에 1회 이상 방문한 고유 사용자 |
| **MAU** | 월간 활성 사용자 | 해당 월에 1회 이상 방문한 고유 사용자 |
| **MAU-A** | 월간 분석 사용자 | 해당 월에 1회 이상 분석 완료한 사용자 |

```typescript
// 활성 사용자 정의
interface ActiveUserCriteria {
  // DAU: 해당 일에 다음 중 하나 수행
  dauActions: [
    'app_open',
    'page_view',
    'analysis_complete',
    'product_click',
  ];

  // MAU-A: 해당 월에 분석 완료
  mauAActions: [
    'personal_color_analysis_complete',
    'skin_analysis_complete',
    'body_analysis_complete',
    'hair_analysis_complete',
    'makeup_analysis_complete',
  ];
}
```

## 3. 획득 지표 (Acquisition)

### 3.1 CAC (Customer Acquisition Cost)

```typescript
interface CAC {
  // 전체 CAC
  overall: {
    formula: 'total_marketing_cost / new_users';
    target: 3000;  // ₩3,000
  };

  // 채널별 CAC
  byChannel: {
    organic: 0;
    seo: 500;
    referral: 1000;
    influencer: 2000;
    paidSocial: 4000;
    paidSearch: 5000;
  };

  // 유료 사용자 CAC
  payingCAC: {
    formula: 'total_marketing_cost / new_paying_users';
    target: 50000;  // ₩50,000
  };
}
```

**CAC 목표**:

| 연도 | 전체 CAC | 유료 CAC | 목표 |

## 6. 수익화 지표 (Revenue)

### 6.1 ARPU & ARPPU

```typescript
interface RevenueMetrics {
  // ARPU (Average Revenue Per User)
  arpu: {
    formula: 'total_revenue / mau';
    targets: {
      year1: 2000,   // ₩2,000
      year2: 3500,   // ₩3,500
      year3: 5000,   // ₩5,000
    };
  };

  // ARPPU (Average Revenue Per Paying User)
  arppu: {
    formula: 'total_revenue / paying_users';
    targets: {
      year1: 6000,   // ₩6,000
      year2: 7500,   // ₩7,500
      year3: 9000,   // ₩9,000
    };
  };
}
```

### 6.2 LTV (Lifetime Value)

```
┌─────────────────────────────────────────────────────────────────┐
│                   LTV 계산 모델                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   기본 공식:                                                     │
│   LTV = ARPU × Average Lifespan                                  │
│       = ARPU × (1 / Monthly Churn Rate)                          │
│                                                                  │
│   1년차 예상:                                                    │
│   LTV = ₩2,000 × (1 / 0.10) = ₩20,000                           │
│                                                                  │
│   유료 사용자 LTV:                                               │
│   LTV_paid = ARPPU × (1 / Paid Churn Rate)                      │
│            = ₩6,000 × (1 / 0.07) = ₩85,714                      │
│                                                                  │
│   연도별 LTV 목표:                                               │
│   ──────────────────────────────────────────                    │
│   Year 1: ₩20,000 (전체) / ₩85,714 (유료)                       │
│   Year 2: ₩43,750 (전체) / ₩150,000 (유료)                      │
│   Year 3: ₩83,333 (전체) / ₩225,000 (유료)                      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 6.3 LTV/CAC 비율

| 연도 | LTV | CAC | LTV/CAC | 판정 |
