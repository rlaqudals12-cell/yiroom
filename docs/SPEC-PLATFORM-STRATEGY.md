# 이룸 플랫폼 전략

> 웹과 앱의 역할 분담 및 개발 우선순위

## 핵심 원칙

```yaml
전략: App-First
  - 앱: 핵심 경험 (분석, 트래킹, 쇼핑)
  - 웹: 유입 채널 (SEO, 공유 링크)

기능: 동일, UX: 최적화
  - 핵심 기능은 양쪽 동일
  - 플랫폼 강점에 맞게 UI/UX 차별화

리소스 배분: 7:3
  - 앱 개발: 70%
  - 웹 유지보수: 30%
```

## 플랫폼별 역할

### 앱 (1순위)

| 영역              | 설명                                | 우선순위 |
| ----------------- | ----------------------------------- | -------- |
| **카메라 분석**   | PC-1, S-1, C-1, N-1 실시간 촬영     | 필수     |
| **일일 트래킹**   | 운동 기록, 식단 기록, 물 섭취       | 필수     |
| **푸시 알림**     | 운동 리마인더, 물 마시기, 할인 알림 | 필수     |
| **위젯**          | 오늘 운동 플랜, 칼로리 현황         | 권장     |
| **건강 앱 연동**  | Apple Health, 삼성 헬스             | 권장     |
| **오프라인 모드** | 저장된 플랜, 운동 기록              | 필수     |

### 웹 (2순위)

| 영역              | 설명                         | 우선순위 |
| ----------------- | ---------------------------- | -------- |
| **SEO 랜딩**      | 검색 유입 "퍼스널 컬러 진단" | 필수     |
| **공유 링크**     | 결과 공유 → 신규 유저 유입   | 필수     |
| **상세 비교**     | 제품 비교 테이블 (넓은 화면) | 권장     |
| **분석 (업로드)** | 사진 업로드 방식 분석        | 유지     |
| **PWA**           | 앱 설치 전 체험용            | 유지     |

## 사용자 유형별 시나리오

### 타입 A: 앱만 사용 (예상 60%)

```
[앱 설치] → [분석] → [트래킹] → [쇼핑] → [리텐션]
           카메라    푸시알림    원클릭     위젯
```

### 타입 B: 웹만 사용 (예상 15%)

```
[검색 유입] → [웹 분석] → [결과 확인] → [앱 설치 유도]
  SEO         업로드       제한적          배너
```

### 타입 C: 둘 다 사용 (예상 25%)

```
[앱] 분석, 일일 트래킹
[웹] 제품 상세 비교, 구매 결정
     ↓
[데이터 동기화] 어디서든 접근 가능
```

## 플랫폼별 UX 차별화

### 분석 기능

| 기능      | 앱                 | 웹             |
| --------- | ------------------ | -------------- |
| 입력 방식 | 카메라 직접 촬영   | 사진 업로드    |
| 분석 속도 | 실시간 (빠름)      | 업로드 후 처리 |
| 정확도    | 높음 (조명 가이드) | 보통           |
| 재분석    | 즉시 가능          | 재업로드 필요  |

### 쇼핑 기능

| 기능       | 앱                   | 웹          |
| ---------- | -------------------- | ----------- |
| 결제       | 원터치 (Apple Pay)   | 일반        |
| 비교       | 카드 스와이프        | 테이블 비교 |
| 알림       | "찜한 제품 할인 중!" | 이메일만    |
| 위시리스트 | 오프라인 접근        | 온라인만    |

### 트래킹 기능

| 기능      | 앱              | 웹        |
| --------- | --------------- | --------- |
| 운동 기록 | 실시간 + 타이머 | 수동 입력 |
| 식단 기록 | 카메라 촬영     | 수동 입력 |
| 알림      | 푸시            | 없음      |
| 위젯      | 홈 화면         | 없음      |

## 핵심 결정사항

### 1. 기능 분리 vs 통합

```
❌ 분리: "앱=분석, 웹=쇼핑"
   → 사용자 여정 단절, 전환율 하락

✅ 통합: "기능 동일, UX 최적화"
   → 분석→추천→구매 한 플로우
   → 플랫폼 강점만 차별화
```

### 2. 개발 우선순위

```
✅ App-First
   → 핵심 기능(카메라)이 앱에 최적화
   → 일일 사용 패턴이 앱에 적합
   → 리텐션이 앱이 월등

웹은:
   → 유입 채널로 유지
   → PWA로 최소 기능 제공
   → 앱 설치 유도에 집중
```
