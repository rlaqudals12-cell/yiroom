# ADR-078: 홈 3-State 사용자 시뮬레이션 기반 의사결정

**Status**: Accepted
**Date**: 2026-03-07
**Context**: 홈/대시보드 UX 시뮬레이션 결과 기반 개선 판단

## 배경

홈 페이지 3-State 모델(New/Growing/Active)에 대해 4개 페르소나 시뮬레이션을 진행.
10개 개선 제안 중 P0(요구사항 의심) + P4(단순화) 원칙으로 재검증하여 2개만 채택.

## 시뮬레이션 결과 요약

| State           | 점수 | 핵심 강점                   | 핵심 약점                            |
| --------------- | ---- | --------------------------- | ------------------------------------ |
| New (0개)       | 78   | 깔끔한 첫인상, Social Proof | CTA 제한적                           |
| Growing (1-3개) | 82   | 인과 추천이 차별화 포인트   | 프로그레스 바 부담감                 |
| Active (4+개)   | 55   | 분석 요약 한눈에            | Daily Insight 고정, 위젯 도달 어려움 |
| 장기 리텐션     | 35   | -                           | 재분석 유도 부재                     |

종합: **62.8/100 (B-)**

## 결정: 채택 2개

### 1. 분석 요약 slice(0,4) → slice(0,6)

**문제**: 5-6개 분석 완료 시 일부가 표시되지 않음
**해결**: `HomeAnalysisSummary.tsx`에서 `analyses.slice(0, 6)`으로 변경
**근거**: 최대 6개 분석이므로 전부 표시. 2열 그리드 3행 = 허용 범위

### 2. 캐시 무효화 (visibility change)

**문제**: 분석 완료 → 홈 복귀 시 5분 캐시로 이전 State 유지
**해결**: `useAnalysisStatus.ts`에 `visibilitychange` 리스너 추가. 30초 이상 된 캐시만 무효화
**근거**: SPA 내비게이션에서 글로벌 캐시가 갱신되지 않는 버그. 30초 임계값으로 불필요한 재조회 방지

## 결정: 삭제 8개 (근거)

| 제안                        | 삭제 이유                                                     |
| --------------------------- | ------------------------------------------------------------- |
| 오늘 기록 0/0/0 격려 메시지 | 0은 매일 아침 기본값. 격려 문구가 매일 반복되는 새 문제 발생  |
| 게이미피케이션 위치 이동    | 현재 순서(기록→성장)가 논리적. "도달 불가"는 데이터 없는 가정 |
| Daily Insight 패턴 다양화   | 8→20은 중간 단계. 근본 해결은 AI 개인화이며 시기상조          |
| Growing 빈 최근 본 제품 CTA | 빈 상태에서 섹션 미표시가 정상 동작                           |
| 재분석 유도                 | MAU 0 상태에서 리텐션 최적화는 시기상조                       |
| Gemini 실시간 인사이트      | 홈 로딩에 AI 호출 추가 = 성능 저하. 필요시 재검토             |
| State 전환 축하 애니메이션  | 1회용 코드에 과투자. 콘텐츠가 자연스럽게 안내                 |
| Social Proof 날짜 변동      | 현재 sin(hour) 변동으로 충분. 실제 데이터 연동은 Phase 2      |

## 핵심 원칙 적용

- **P0**: "이 개선이 정말 필요한가?" → 8개는 불필요
- **P4**: "더 단순한 방법은?" → 코드 1줄 수정 vs 새 컴포넌트 추가
- **P1**: 장기 리텐션(35점)은 출시 후 실데이터로 재판단
