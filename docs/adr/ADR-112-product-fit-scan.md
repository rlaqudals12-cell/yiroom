# ADR-112: 제품 스캔 "나와의 적합도" 판정

**Status**: Accepted
**Date**: 2026-07-09
**Deciders**: 사용자(창업자) + Claude Code
**관련**: ADR-111(One Canon IA), [원리: ingredient-safety-timeline](../principles/ingredient-safety-timeline.md), [리서치](../research/claude-ai-research/2026-07-product-scan-grading-medical.md)

## 배경

사용자 요구: OASIS/Yuka처럼 "제품을 찍으면 성분·등급·나에게 맞는지"를 알려주는 기능. 코드 감사 결과 스캔 파이프라인의 ~70%(바코드·OCR·개인화 호환성 엔진·궁합 25규칙·제품함)가 기존재. 격차 = 성분 안전 DB 규모(97+20행), 효과 타임라인 부재, EWG 배지 dead-wiring, 법적 표현 미정립.

## 결정

1. **판정 모델 = 절대 등급이 아닌 "나와의 적합도"** — 제품에 0-100 안전등급을 매기지 않는다. 기존 `analyzeCompatibility()`(내 피부·PC 프로필 × 성분)를 전면 브랜딩하고 4-레이어(규제 사실 / 개인 적합도 / 궁합 / 타임라인)로 보강한다.
2. **데이터 소스 1차 = 식약처 공공데이터** — 원료성분정보 API(15111774) + 규제정보 API(15111773) + 알레르기 유발 착향제 25종 고시. 벌크 임포트 스크립트(`scripts/`, `DATA_GO_KR_API_KEY` env, dry-run 지원)로 `cosmetic_ingredients` 확충. EWG는 참고 지표(출처 병기)로만, 크롤링 금지. 화해 데이터 복제 금지.
3. **효과 타임라인 = 문헌 인용형 텍스트** — "임상 연구에서는 보통 N주 후 개선이 보고돼요(출처)" 형태. 핵심 활성성분 ~10종 수동 큐레이션(`lib/scan/ingredient-timeline.ts` 상수 + 논문 URL). 예측 이미지·기간 단정 금지(화장품법). 변화 추적 재분석 유도와 결합.
4. **OCR = 기존 Gemini 경로 유지 + 모델 표준화** — `ingredient-ocr.ts`의 `gemini-2.0-flash` 하드코딩을 프로젝트 표준 클라이언트(`lib/gemini/client`, GEMINI_MODEL env)로 통일. 별도 OCR SDK 도입 안 함(P4).
5. **법적 표현 가이드 코드화** — 금지: "치료/재생/보장/O주면 사라져요/이 제품은 나쁨". 허용: "적합도 N점", "식약처 지정 알레르기 유발 착향제 포함", "개선에 도움을 줄 수 있어요", 문헌 인용. 면책문구 유지·강화. 금지 표현은 테스트로 고정.

## 대안과 트레이드오프

- **Yuka식 절대 안전등급** — 기각: 농도 미반영이라 비과학적(원리 §1.1), 제조사 분쟁 표적(Yuka 프랑스 소송), 이룸 차별점(개인화) 희석.
- **EWG 크롤링으로 등급 대량 확보** — 기각: 라이선스 회색지대, 미국 편향. 식약처 공공데이터가 무료·합법·한국 정합.
- **타임라인을 AI 생성으로** — 기각: 근거 없는 생성은 정직성 위반. 문헌 큐레이션 상수가 결정론적이고 출처 검증 가능.

**수용한 트레이드오프**: 식약처 API 키 발급(공공데이터포털 활용신청)은 사용자 액션 필요 — 키 없이도 나머지(타임라인·배지 활성화·표현 정비)는 진행 가능하도록 스크립트를 분리. / 타임라인 성분 커버리지는 초기 ~10종(문헌 확보분만) — 커버 안 되는 성분은 표시하지 않음(지어내지 않음).

## 결과

- 스캔 결과 표면이 "등급표"가 아니라 "내 프로필 기준 컨설팅"이 된다(제품 방향 정합).
- `cosmetic_ingredients` 확충 후 EWG/규제 배지 dead-wiring 해소(DB 조인 연결).
- 신규 표면은 One Canon(ADR-111) 준수: 판정 정본 = 스캔 결과, 제품 상세·뷰티 추천은 동일 판정 컴포넌트 재사용.

## 검증 기준

- 금지 표현 테스트 그린(치료/보장/기간 단정 문자열 부재).
- 동일 성분 입력 → 동일 판정(결정론).
- 타임라인 항목 전수에 출처 URL 존재.
