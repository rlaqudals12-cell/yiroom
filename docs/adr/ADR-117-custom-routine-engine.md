# ADR-117: 맞춤 루틴 엔진 — 내 화장대 기반 루틴 배치

**Status**: Accepted
**Date**: 2026-07-10

**관련**: ADR-069(캡슐), ADR-112(스캔 적합도), [리서치: 내부 자산](../research/claude-ai-research/2026-07-rx-internal-assets.md) · [고도화](../research/claude-ai-research/2026-07-rx-advancement.md)

## 배경

창업자 요구: 보유 제품 등록 → 적합 판정 → **그 제품을 포함한 루틴** 구성 → 없는 단계는 맞는 제품 구매 연결 → 목표 반영 우선순위 → 단계 계획. 내부 감사 결론: 핵심 엔진(`generateRoutineFromShelf` — 배치·궁합·빈 슬롯, 완전 orphan / `attachSolutionProducts` — 카탈로그만 / 소진 예측 / 효과 추적)이 전부 존재하나 연결이 없음. 경쟁 지형: 실측 피부 ↔ 실보유 자산 ↔ 목표를 폐루프로 묶는 서비스 전무.

## 결정

1. **v1 = 배선** (신규 알고리즘 0): 오늘의 루틴(`attachSolutionProducts`)과 루틴 페이지(`enrichRoutineWithProducts`)를 **shelf-우선**으로 — 내 제품이 있으면 그 제품을 배치(`source:'shelf'`), 없으면 카탈로그 추천(`source:'catalog'`) = **빈 슬롯 구매 연결**(어필리에이트가 강매가 아닌 결손 채움으로 발생). 홈 위젯에 solutionProduct 렌더, 제품 상세에 "내 제품함에 담기" 진입점 신설(스캔 외 등록 경로).
2. **지어내기 금지**: shelf 배치는 `detectProductCategory`가 명확할 때만(unknown→미배치), shelf 조회 실패는 카탈로그 폴백(출처 표시 유지).
3. **concerns 정본화**: `deriveConcernsFromMetrics`를 lib/skincare 공개 API로 승격 — 3표면 제각각 파생과 데일리 미전달을 단일화(v1의 "목표 반영" = 지표 파생 관심사).
4. **용어**: "처방(Rx)"·"치료" 금지 — 전 표면 **"맞춤 루틴"** (클레임이 의약품 분류를 결정 — 리서치 §7.3 안전 표현표).
5. **inventory 이원화 봉인**: `user_product_shelf`(성분·궁합)가 루틴의 자산 소스. `inventory`(소진 예측)는 v2에서 어댑터로 결합 — 이번에 합치지 않음(스코프 폭발 방지).

## v2/v3 로드맵 (리서치 근거 — 이번 범위 아님)

- **v2**: 스킨 사이클링 주기 스케줄(4일 상태기계 — 각질→레티노이드→회복×2), 명시적 목표 선택 UI, 단계 계획("장벽 4주 → 미백 전환") 상태머신, 캡슐 화장대(중복 성분 지적 — "덜 사게도 말해준다"), 소진 예측 어댑터.
- **v3**: 폐루프 — 효과 추적으로 교체 제안 자동화, **최소 순응기간 게이트**(레티노이드·산 6~8주 전 정체는 정상 — 조기 교체 오판 방지), 재분석 리마인더.

## 대안과 트레이드오프

- **shelf/inventory 즉시 통합** — 기각: 마이그레이션·UI 대공사, v1 가치와 무관.
- **AI로 루틴 생성** — 기각: 규칙 엔진이 결정론적·무료·검증 가능(기존 정본 유지).

**수용**: 카테고리 키워드 매칭의 한계(오분류 가능 — unknown 처리로 방어, v2에서 정밀화) / 등록 제품의 성분은 카탈로그 key_ingredients 수준(전성분 아님 — 궁합 검출력 제한, 정직하게 그 범위만 판정).

## 검증 기준

- 제품 담기 → 오늘의 루틴에 "내 ○○" 표시 / 결손 스텝 → "맞는 제품 보기"
- '처방/치료' 문자열 부재 테스트, prod user_product_shelf 20컬럼 실재 확인(2026-07-10)
