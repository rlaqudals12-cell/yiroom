# ADR-058: Hub 패턴 연기 결정

> **Status**: Accepted
> **Date**: 2026-01-24
> **Deciders**: 창업자
> **Tags**: ui, layout, navigation, ux

## 0. 궁극의 형태 (P1)

### 이상적 최종 상태

```
"Hub 패턴 도입"
- 중앙 집중형 분석 모듈 선택 UI
- 모듈별 카드 기반 네비게이션
- 프리미엄 앱 경험
```

### 물리적 한계

| 항목 | 한계 |
|------|------|
| 사용자 학습 비용 | 기존 5탭 패턴에서 전환 시 혼란 |
| 개발 비용 | 레이아웃 변경 = 4.8배 작업량 |
| 롤백 복잡도 | 레이아웃 롤백은 CSS보다 복잡 |

### 100점 기준

- Hub 패턴 도입 + A/B 테스트 검증
- 사용자 이탈률 증가 없음
- 전환율 개선 수치 확보

### 현재 목표: 0% (연기)

- Hub 패턴 미도입
- 기존 5탭 네비게이션 유지
- 스타일 변경만 진행

### 의도적 제외

| 제외 항목 | 이유 | 재검토 시점 |
|----------|------|------------|
| Hub 패턴 도입 | P0 원칙 위반 (불필요한 레이아웃 변경) | MAU 5,000+ |
| 네비게이션 구조 변경 | 사용자 학습 비용 우려 | A/B 테스트 후 |
| 모듈 선택 UI 리디자인 | 현재 구조로 충분 | 사용자 피드백 수집 후 |

## 1. 컨텍스트

### 1.1 배경

YIROOM IDENTITY v3 (핑크 기반 다크 테마) 적용 시, 두 가지 선택지 존재:

| 옵션 | 범위 | 작업량 |
|------|------|--------|
| **Option A** | 스타일만 변경 (CSS 토큰 + 로고) | 4 atoms, ~2.5시간 |
| **Option B** | 스타일 + 레이아웃 (Hub 패턴 도입) | 9 atoms, ~12시간 |

### 1.2 Hub 패턴이란?

```
┌─────────────────────────────────────────────────────────────────┐
│                        Hub 패턴 예시                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐             │
│  │   피부분석   │  │  퍼스널컬러  │  │   체형분석   │             │
│  │    카드      │  │    카드      │  │    카드      │             │
│  └─────────────┘  └─────────────┘  └─────────────┘             │
│                                                                 │
│  중앙 집중형 모듈 선택 페이지                                    │
│  탭 터치 → Hub 페이지 → 모듈 카드 선택 → 분석 페이지            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 현재 구조

```
┌─────────────────────────────────────────────────────────────────┐
│                     현재 5탭 네비게이션                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [홈] [분석] [옷장] [웰니스] [마이페이지]                        │
│                                                                 │
│  "분석" 탭 터치 → 분석 목록 → 모듈 선택 → 분석 페이지           │
│                                                                 │
│  한국 사용자에게 익숙한 표준 패턴                                │
│  (카카오톡, 인스타그램, 네이버, 당근 등)                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 결정

### 2.1 선택: Option A (스타일만 변경)

```
Option A (스타일만) → 채택 ✅
Option B (스타일+레이아웃) → 연기 ⏸️
```

### 2.2 근거 (P0-P8 원칙 적용)

#### P0: 요구사항 의심

| 질문 | 답변 |
|------|------|
| Hub 패턴이 필요한가? | 증명되지 않음 |
| 레이아웃 변경 없이 목표 달성 가능한가? | ✅ 가능 (85%) |
| 누가 이것을 요구했나? | 명확한 사용자 요청 없음 |

#### P1: 궁극의 형태

| 지표 | Option A | Option B |
|------|----------|----------|
| 목표 달성률 | 85% | 90% |
| 차이 | - | +5% |

**5% 차이에 4.8배 비용은 ROI 부족**

#### P3: 원자 분해

| 옵션 | 원자 수 | 예상 시간 |
|------|---------|----------|
| Option A | 4개 | 2.5시간 |
| Option B | 9개 | 12시간 |
| **비율** | - | **4.8배** |

#### P4: 단순화

```
Hub 패턴은 "nice to have", "must have" 아님
→ 단순화 원칙에 의해 제외
```

### 2.3 사용자 관점 분석

#### Jakob's Law (야콥의 법칙)

```
"사용자는 대부분의 시간을 다른 앱에서 보낸다.
따라서 당신의 앱도 다른 앱처럼 작동하길 기대한다."
```

| 앱 | 네비게이션 패턴 |
|----|----------------|
| 카카오톡 | 5탭 |
| 인스타그램 | 5탭 |
| 네이버 | 5탭 |
| 당근 | 5탭 |
| **이룸 (현재)** | **5탭** |

#### 사용자 인식 비교

| 요소 | Option A | Option B |
|------|----------|----------|
| 첫인상 | "앱이 세련돼졌다" | "완전 새 앱이네" |
| 학습 비용 | 없음 | 중간~높음 |
| 이탈 위험 | 없음 | 존재 |

### 2.4 재검토 조건

Hub 패턴 도입을 재검토하는 조건:

| 조건 | 기준 | 현재 상태 |
|------|------|----------|
| MAU 달성 | 5,000+ | 미달성 |
| 사용자 피드백 | "모듈 찾기 어렵다" 다수 | 없음 |
| A/B 테스트 | 전환율 개선 확인 | 미실시 |
| 경쟁사 분석 | Hub 패턴 성공 사례 | 미확인 |

## 3. 대안 검토

### 3.1 Option A: 스타일만 변경 (채택)

**장점**:
- 개발 비용 낮음 (2.5시간)
- 사용자 학습 비용 없음
- 롤백 용이 (CSS만 되돌리면 됨)
- 이탈 위험 없음

**단점**:
- 목표 달성률 85% (100% 아님)
- "프리미엄 앱" 인식 약간 부족

**결론**: 채택 - ROI 우수, 위험 낮음

### 3.2 Option B: 스타일 + Hub 패턴 (연기)

**장점**:
- 목표 달성률 90%
- 프리미엄 앱 경험
- 모듈별 강조 효과

**단점**:
- 개발 비용 높음 (12시간, 4.8배)
- 사용자 학습 비용 발생
- 롤백 복잡
- 검증 없이 도입 = 위험

**결론**: 연기 - MAU 5,000+ 달성 후 A/B 테스트로 검증
