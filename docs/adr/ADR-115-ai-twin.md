# ADR-115: AI 트윈 — 나를 닮은 모델에 스타일을 입히는 시각 종착점

**Status**: Accepted
**Date**: 2026-07-10
**Deciders**: 사용자(창업자) + Claude Code
**관련**: ADR-113(표현 레이어 — 본 ADR은 그 하위 확장), ADR-110(3D 아바타), [리서치](../research/claude-ai-research/2026-07-ai-twin-fitting.md)

---

## 배경

창업자 아이디어: "사용자와 최대한 유사한 AI 모델을 만들어 피팅+화장법+퍼스널컬러+얼굴형을 결합 표현하면 실제 스타일리스트 효과를 낼 수 있지 않을까." 리서치 확인: Doji(자체 디퓨전 트윈, $14M)·Google Doppl(나노바나나 트윈)·에이블리(AI 착용, 반품 감소)로 이미 상용화. 경쟁사는 전부 옷 중심 — **PC+피부+얼굴형까지 결합한 뷰티 트윈은 빈 시장**. 비주얼 격차("첫 화면에 남의 사진 vs 나")의 근본 해법.

## 결정

1. **방식 = 나노바나나 트윈 캐싱** (Doppl 방식): 사용자 셀카(+선택 전신)로 풀바디 트윈 1장 생성(~$0.03-0.06) → **저장 후 재사용**. 결합 표현(옷/헤어/메이크업)도 장당 ~$0.03-0.06. LoRA 사용자별 파인튜닝은 배제(비용·운영 과투자).
2. **"이게 나 맞나요?" 승인 게이트 필수**: 생성 트윈을 사용자가 승인해야 활성(status: pending→approved). 거부 시 재생성(일 상한 내) 또는 포기. Doji 최대 불만("안 닮음")이 이룸 핵심가치를 위협하므로 — **안 닮은 트윈을 시스템이 강요하지 않는다**.
3. **체형 정합 주입**: 트윈 생성 프롬프트에 이룸의 실측 체형 데이터(body_analyses 비율·타입)를 제약으로 주입 — 체형 왜곡(리스크 2위) 억제, ADR-110 자산 재사용.
4. **레이어 순차 결합**: 옷+헤어+메이크업 동시 적용 금지 — 단계마다 트윈 원본 참조 재주입(정체성 드리프트 억제). v1은 결합 1종(코디 착장)부터.
5. **ADR-113 원칙 전면 상속**: `lib/visual-expression/twin/` 소속 — 분석 파이프라인 역류 금지, "AI 생성 이미지" 라벨 상시, 일 5회 생성 상한 공유, 트윈 이미지는 분석 이미지와 **분리된 Storage 경로**에 저장.
6. **저장**: 신규 `user_twins` 테이블(clerk_user_id RLS) + Supabase Storage 비공개 경로. 사용자 삭제권(트윈 삭제 = 파일+행 삭제).
7. **정책 방어**: 본인 사진만(타인 사진 업로드 금지 고지), 스토어 딥페이크 정책 대비 신고·삭제 경로, Gemini 전송 고지 문구.

## 대안과 트레이드오프

- **FASHN으로 트윈까지** — 기각: FASHN은 옷 특화(뷰티 결합 불가), 이미 Gemini 스택 보유.
- **LoRA/자체 학습(Doji 방식)** — 기각: 트윈 1회 $0.5~수$+GPU 운영, 세션 수백원 정책 위반.
- **승인 게이트 없이 자동 사용** — 기각: 안 닮은 트윈 강요 = 신뢰 붕괴.

**수용**: 트윈 품질은 나노바나나 성능에 종속(승인 게이트가 방어) / 결합마다 비용 발생(상한 내) / 18세 미만·초상권 이슈는 본인 사진 고지+신고 경로로 1차 방어(스토어 제출 전 재점검 항목).

## 결과

- 온보딩·[나] 탭에서 "내 트윈 만들기" → 승인 → 옷장/코디의 "트윈에게 입혀보기".
- 첫 화면 비주얼이 "남의 모델"이 아닌 "나"가 되는 기반.

## 검증 기준

- 승인 전 트윈이 어떤 표면에도 노출되지 않음(테스트)
- 분석 파이프라인 역류 import 0 / AI 라벨 상시 / 일 상한 공유 동작
- 삭제 시 Storage 파일+DB 행 동시 제거
