# ADR-114: 전속 뷰티팀 IA — 가입=첫 미팅 · 브리핑 홈 · 5탭 단일 내비게이션

**Status**: Accepted
**Date**: 2026-07-10
**Deciders**: 사용자(창업자) + Claude Code
**관련**: ADR-109(프로필 중심 홈), ADR-111(One Canon — 본 ADR이 정본 표 일부 갱신), [IA 리서치](../research/claude-ai-research/2026-07-ia-navigation.md)

## 배경

정체성 확정(2026-07-10): **"나를 다 아는 전속 뷰티팀(스타일리스트+외모 컨설턴트+피부 관리사)이 매일 옆에 있는 것"**, 일상 인터랙션 정본 = **매일 브리핑**(채팅은 후속 보조). 그러나 실측 결과 구조가 정체성과 불일치:

- 이중 IA(데스크톱 드롭다운 ≠ 모바일 4탭), (main) 페이지 141개 미로
- 컨설팅의 심장인 코치(/coach)는 기능 완성 상태인데 **네비 진입점 0**
- 가입 = 홈 랜딩 + 무동작 설명 오버레이 4스텝 ("가입=팀 첫 미팅"이 아님)
- 홈 = 위젯 나열 + 드래그 정렬 (브리핑이 아님)

## 결정

1. **5탭 단일 IA** — 모바일 BottomNav와 데스크톱 Navbar를 같은 5항목으로 통일: **[오늘] [물어보기] [뷰티] [스타일] [나]**. Navbar 드롭다운·"대시보드"류 잔재 폐기, 딥 페이지는 각 탭 아래 점진 공개.
2. **[물어보기] = /coach 1급 승격** — 퀵 질문을 상황 4종(뭐 입지/이 제품 맞아?/머리/화장)으로.
3. **가입 = 첫 미팅** — sign-up의 `forceRedirectUrl`(코드 레벨)로 통합분석 온보딩 모드(`?onboarding=1`, 스킵 가능) 직행. OnboardingTutorial 오버레이 폐기(실제 플로우가 온보딩을 대체, P4).
4. **[오늘] = 팀의 아침 메시지** — 위젯 대시보드가 아니라 1인칭 브리핑. `composeBriefing()` 규칙 조립(AI 호출 없음)에 화법 4요소를 코드화: ①말을 건다(관찰→조언 대화체) ②기억한다(제품함 후속·추이 언급) ③일관 인격(톤 가이드 상수) ④묻기 전에 우선순위 결정. **정직성 가드**: 실데이터 없는 문장은 생략, 일반론 채움 금지, 관찰엔 근거 수치 동반.
5. **ADR-111 정본 표 갱신**: 축 요약 정본 = **[나] 탭 ProfileCardGrid 단독**(홈에서 제거), 홈 = 브리핑, 새 분석 진입 = 온보딩 + 프로필 빈칸 CTA.

## 대안과 트레이드오프

- **4탭(뷰티+스타일 통합)** — 보류: 리서치 추천안이었으나 /beauty·/style 대통합은 별도 대공사이고 한국 표준(화해·에이블리)이 5탭. 통합은 후속 재검토.
- **브리핑을 AI 생성으로** — 기각: 비용·비결정성·환각 리스크. 규칙 조립이 결정론적이고 정직성 가드 가능. (데이터 축적 후 AI 문장 다듬기는 재검토 여지)
- **홈에 ProfileCardGrid 유지** — 기각: 브리핑과 프로필이 한 화면에 겹치면 다시 "대시보드"化. 정본 1곳 원칙.

**수용한 트레이드오프**: 첫 주 사용자의 브리핑은 문장이 단순(데이터 적음 — 쓸수록 사람다워지는 구조가 정직) / 기존 사용자의 홈 변화 학습 비용 / OnboardingTutorial·SortableWidgetList 의도적 삭제.

## 검증 기준

- 모바일·데스크톱 네비 항목 동일(이중 IA 0)
- composeBriefing: 데이터 없는 문장 생략·일반론 금지·근거 수치 — 테스트 고정
- 신규 가입 → 온보딩 착지·스킵 동작, 기존 사용자 → 브리핑 렌더
