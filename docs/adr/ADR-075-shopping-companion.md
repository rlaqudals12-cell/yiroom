# ADR-075: 쇼핑 컴패니언 매칭 전략

## 상태

**제안** (2026-03-03)

## 컨텍스트

캡슐 에코시스템에서 "부족 아이템 식별 → 구매 추천"은 C5(미니멀리즘)와
수익화(어필리에이트)를 연결하는 핵심 기능이다.
기존 ADR-050(Fashion-Closet)의 부족 아이템 진단과
ADR-067(어필리에이트)의 파트너 API를 통합해야 한다.

### 기존 시스템

| 시스템                         | 위치 | 현재 역할                                |
| ------------------------------ | ---- | ---------------------------------------- |
| lib/smart-matching/ (11모듈)   | 웹   | 2축 스코어링 (도메인 0-50 + 대중성 0-50) |
| lib/products/matching.ts       | 웹   | 매칭 알고리즘                            |
| lib/products/affiliate.ts      | 웹   | 클릭/전환 추적                           |
| user_product_shelf             | DB   | 사용자 보유 제품                         |
| cosmetic_ingredients (100시드) | DB   | 성분 안전 검증                           |
| ADR-050 종합점수               | 설계 | 색상(40%)+체형(35%)+TPO(25%)             |

### 미니멀리즘-어필리에이트 충돌 해소

P-1 C5 원칙: "적을수록 좋다" vs 어필리에이트 수익: "더 사게 하라"

→ **해소 전략**: "필요한 것만 정확히 추천"

- 불필요한 제품 추천 금지 (C5 위반)
- 캡슐 진단으로 진정한 공백만 식별
- 기존 보유 아이템으로 해결 가능하면 구매 추천 없음

## 결정

### 1. 캡슐 갭 분석 (Gap Analysis)

```
입력: [현재 캡슐 아이템] + [BeautyProfile]

Step 1: 이상적 캡슐 구성 산출
  → CapsuleEngine<T>.curate(profile) → 이상 캡슐

Step 2: 보유 아이템 매칭
  → user_product_shelf + wardrobe_items 조회
  → 보유 아이템과 이상 캡슐 비교

Step 3: 갭 식별
  → 이상 캡슐 - 보유 아이템 = 부족 아이템 목록
  → 각 갭의 우선순위: 호환성 영향도 기준

Step 4: 갭 해소 방안 결정
  a. 기존 보유 아이템으로 대체 가능 → "이것을 활용해보세요"
  b. 대체 불가 → 구매 추천 (쇼핑 컴패니언)

출력: GapReport {
  gaps: GapItem[];           // 부족 아이템 목록
  reusable: ReusableItem[];  // 기존 아이템 활용 제안
  recommendations: ProductRecommendation[]; // 구매 추천
}
```

### 2. 쇼핑 컴패니언 매칭

```
매칭 파이프라인:

1. 제품 검색:
   → 갭 카테고리 + BeautyProfile 기반 쿼리
   → lib/smart-matching/ 11모듈 활용

2. 3축 스코어링:
   도메인 적합도 (0-40): 분석 결과 기반 매칭
   캡슐 호환성 (0-35): 기존 캡슐과 CCS 점수
   가격/가성비 (0-25): 가격 대비 품질

3. Safety Profile 필터링:
   → ADR-070 BLOCK 아이템 제외
   → WARN 아이템 경고 표시

4. 어필리에이트 연결:
   → ADR-067 파트너 API로 딥링크 생성
   → lib/products/affiliate.ts로 전환 추적
```

### 3. UI 패턴

```
캡슐 대시보드:
  ┌────────────────────────────┐
  │ 나의 스킨케어 캡슐 (5/7)   │
  │ ■■■■■□□                    │
  │                            │
  │ 부족 아이템:                │
  │ ├── 수분 크림 (호환성 92)   │  ← 구매 추천
  │ └── 선크림 (호환성 85)      │  ← 구매 추천
  │                            │
  │ 기존 아이템 활용:            │
  │ └── 토너A → 에센스 대체 가능 │  ← 추가 구매 불필요
  └────────────────────────────┘

원칙: 기존 아이템 활용을 먼저 제안 → 정말 필요한 것만 구매 추천
```

### 4. 수익 모델 연동

```
어필리에이트 플로우:

  갭 식별 → 제품 추천 → 딥링크 클릭 → 파트너 사이트 구매 → 커미션

추적: lib/products/affiliate.ts
  - click_tracking: 클릭 이벤트 기록
  - conversion_tracking: 구매 전환 기록
  - attribution: 캡슐 갭 분석 경유 여부

KPI:
  - 캡슐 갭 → 추천 전환율 목표: 15%+
  - 기존 매칭 대비 전환율 2x 예상 (개인화 정밀도 향상)
```

## 대안

| 대안                      | 장점         | 단점                      | 기각 이유        |
| ------------------------- | ------------ | ------------------------- | ---------------- |
| 항상 구매 추천            | 수익 극대화  | C5 위반, 사용자 신뢰 저하 | 핵심 가치 훼손   |
| 갭 분석 없이 인기 순 추천 | 구현 간단    | 개인화 부재               | 캡슐 의미 없음   |
| 구매 추천 완전 제외       | C5 완전 준수 | 수익 모델 부재            | 지속 가능성 저하 |

## 결과

### 장점

- 기존 smart-matching + affiliate 시스템 최대 활용
- C5(미니멀리즘) 준수하면서 수익 모델 유지
- 갭 기반 추천으로 전환율 향상 기대
- Safety Profile 필터링으로 안전성 보장

### 단점

- 갭 분석 정확도가 캡슐 엔진 완성도에 의존
- 파트너 API 부재 시 시드 데이터 기반 추천만 가능
