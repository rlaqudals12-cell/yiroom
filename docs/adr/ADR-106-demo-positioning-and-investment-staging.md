# ADR-106: 데모/투자 단계 포지셔닝 — 진단 우선, 커머스 컨설턴트 = use of funds

- **Status**: accepted
- **Date**: 2026-06-17
- **관련**: [ADR-098](./ADR-098-identity-redefinition-5axis-model.md) (5축 정체성), `memory/support-programs-2026.md`, `memory/demo-polish-plan-2026-06.md`

---

## Context

- 6/28 마감 지원(SparkClaw 1기 + 프라이머 29기)을 앞두고, 데모(라이브 웹)와 지원서에서 **경쟁사(룩핀·화해) 대비 차별점**을 어떻게 서술하고, **스타일링/쇼핑 컨설턴트 레이어를 언제 만들지** 결정 필요.
- 실측 현황(2026-06-16): **진단 레이어(5축 PC/S/C/H/M)는 작동**(E2E 검증, 실제 Gemini). **커머스/스타일링 레이어는 코드만 존재, 데이터 공백**(`cosmetic_products`·`affiliate_products`·`brand_size_charts` 0행).
- ⚠️ 기존 피치 문서(`EXECUTIVE-SUMMARY`·`PITCH-DECK-OUTLINE`·`BUSINESS-MODEL` 등)는 **ADR-098 이전(2026-01)** 상태 — "통합 웰니스 플랫폼 / 7대 분석(영양·운동 포함)"으로 적혀 현재 정체성(거울+옷장, 5축, W/N 숨김)과 불일치.

## Decision

### 1. 차별점 = "제품이 아니라 사람에서 출발"

> **화해는 '성분'을, 룩핀은 '상품'을 본다. 이룸은 '나'를 본다.**

| 포인트                                 | 내용                                                                                                                                                          |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **진단에서 출발 (Personalized-first)** | 셀카 한 장 → 5축 시각 정체성. 경쟁사는 제품 DB/큐레이션이 출발점, 이룸은 '나의 이해'가 출발점                                                                 |
| **도메인 통합 (Cross-domain)**         | 화해=화장품·룩핀=패션 단일 도메인. 이룸은 5축을 엮어 _일관된_ 스타일링/쇼핑으로 연결                                                                          |
| **AI-native·자본효율**                 | 파운더 1인 + AI 에이전트 풀스택 구축. 전문가 90% 수준을 무료·즉시·24/7. 진짜 경쟁자 = "아무것도 안 함 / 자기를 모름" (SparkClaw엔 AI 단독 빌드 스토리가 본체) |

### 2. 빌드 staging — 커머스 컨설턴트 = use of funds

- **진단 엔진** = 완료·작동 (데모로 증명).
- **커머스/스타일링 컨설턴트 레이어**(사이즈·재질 매칭·코디·구매 연결) = **투자 후 구축 = 이번 라운드의 use of funds**. 데모 전 구축하지 않는다.
- 근거: (a) 시간 — 6/28까지 지원서가 우선 (b) 성격 — 코드 아닌 **데이터·제휴 작업**(실상품·어필리에이트·브랜드 사이즈)이라 투자가 funding할 영역 (c) 신뢰 — 빈 상품 = 거짓 약속 위험, 트랙션/자금 있을 때 제휴가 더 쉬움.

### 3. 메시징/랜딩

- 랜딩은 **진단 중심 유지**(현재 능력에 정직). 쇼핑/스타일링 아웃컴 섹션은 **커머스 데이터 확보 후** 추가. 빈 결과로 가는 약속성 UI 금지.
- 지원서 서술 공식: **"진단 작동(데모) + 컨설턴트 레이어 = use of funds + 파운더 AI 단독 빌드 스토리"**.

## Consequences

- ✅ "있는 척" 없이 비전·차별점·자금용처를 일관 서술 가능. Day Zero 심사 친화(작동 핵심 + 명확한 다음 단계 + 실행력).
- 🔴 **기존 피치 문서 stale** → 지원서 작성 시 **ADR-098 + 이 ADR 기준으로 갱신** 필요: `EXECUTIVE-SUMMARY.md`(슬로건·7대분석·웰니스 표현), `PITCH-DECK-OUTLINE.md`, `BUSINESS-MODEL.md`, `DEMO-SCENARIO.md`, `GOOGLE-STARTUP-PLAN.md`, `LAUNCH-PLAN.md`.
- 출시 후 #1 제품 우선순위 = 커머스 데이터 레이어(이 ADR의 use of funds 실행).
