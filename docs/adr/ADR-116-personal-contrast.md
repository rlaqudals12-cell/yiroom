# ADR-116: 퍼스널 대비 실측 — 시즌과 직교하는 강도 축

**Status**: Accepted
**Date**: 2026-07-10
**Deciders**: 사용자(창업자) + Claude Code
**관련**: [원리: personal-contrast](../principles/personal-contrast.md), [리서치](../research/claude-ai-research/2026-07-personal-contrast.md), ADR-105(배색 엔진)

---

## 배경

창업자 질문 "헤어+옷이 어울려도 피부색에 따라 안 맞는 문제, 퍼스널컬러로 해결되나?" → 리서치 답: 부분만. 시즌은 색상을, **대비(피부-모발 명도 격차)는 강도를** 결정하며 두 축은 직교. 기존 스키마에 `contrastLevel` 필드가 있으나 값이 **추측**(Mock 하드코딩·VLM 출력 — VLM은 상대 명도 판단에 불안정)이었고, 피부 L\*은 이미 `zone-sampler`로 정밀 실측 중.

## 결정

1. **대비를 실측한다** — 모발 L\* 샘플러(zone-sampler 패턴 복제, 비피부·저채도 필터) + `computePersonalContrast(|L*hair − L*skin|)`, 임계 25/45 → low/medium/high (상수 격리 — 한국인 분포 축적 후 재보정).
2. **추측 경로 제거** — Mock 하드코딩·VLM 출력 기반 contrastLevel을 실측값으로 대체. **실측 신뢰도 미달(픽셀 수·분산 기준) 시 미판정·필드 생략** — 지어내지 않는다.
3. **소비 3곳**: PC 결과 1줄 안내(풀이 병기 "얼굴의 밝고 어두움 차이") · 브리핑 배색 카드 명도 격차 조절 · 스타일/메이크업 규칙 intensity.
4. 눈(홍채) L\*은 v2 보류(iris refine 필요) — 모발-피부 대비만으로 실용적.

## 대안과 트레이드오프

- **VLM에게 대비 판정 유지** — 기각: 상대 명도 판단 불안정(리서치), 픽셀 실측이 결정론적·무료.
- **눈까지 3점 측정** — 보류: Face Mesh 478 랜드마크 필요, v1 가치 대비 과투자.

**수용**: 염색·조명에 따른 모발 측정 실패 가능(미판정 처리로 방어) / 흑발 지배 분포에서 임계 정밀도 한계(상수 재보정 여지 명시).

## 검증 기준

- 동일 이미지 → 동일 판정(결정론), 합성 픽셀 테스트(흑발+밝은 피부=high 등)
- 신뢰도 미달 → 생략(추측 0)
- 저장된 contrastLevel의 출처가 실측임을 코드 경로로 보장(추측 경로 삭제)
