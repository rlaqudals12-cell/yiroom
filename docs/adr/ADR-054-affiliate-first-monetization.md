# ADR-054: 어필리에이트 우선 수익화 전략 및 구독 시스템 보류

## 상태

`accepted`

## 날짜

2026-01-24

## 맥락 (Context)

### 현재 이룸의 단계

이룸은 현재 **MVP 완료, PMF 탐색 중** 단계에 있습니다. 주요 현황:

| 항목 | 상태 |
|------|------|
| **제품 단계** | MVP 완성 (7개 분석 모듈) |
| **사용자 기반** | 베타 준비 중 (0명) |
| **수익화** | 미적용 |
| **투자** | Pre-seed 준비 |

### 수익 모델 옵션

BUSINESS-MODEL.md에서 정의한 4가지 수익원:

| 수익원 | 비율 | 설명 | 구현 난이도 |
|--------|------|------|------------|
| **어필리에이트** | 40% | 제품 추천 커미션 (5-15%) | 낮음 |
| **구독** | 30% | Premium 기능 ₩9,900/월 | 높음 |
| **B2B** | 20% | API 라이선스 | 높음 |
| **광고** | 10% | 브랜드 파트너십 | 중간 |

### 초기 단계 제약 사항

1. **사용자 기반 부재**: 결제 시스템 구축 전 사용자 확보가 우선
2. **PMF 미검증**: 유료 기능 정의 전 핵심 가치 검증 필요
3. **개발 리소스**: 1인 창업자, 결제 시스템은 복잡도 높음
4. **시장 진입**: 초기 진입 장벽을 낮추어 사용자 획득 극대화

## P1: 궁극의 형태 (Ultimate Form)

> 원리 참조: [00-first-principles.md](../../.claude/rules/00-first-principles.md) P1

### 이상적 최종 상태

**제약이 없다면**:
- 4가지 수익원 균형 있게 운영 (어필리에이트 40%, 구독 30%, B2B 20%, 광고 10%)
- 월 반복 수익 (MRR) 안정적 성장
- LTV/CAC 비율 3:1 이상
- 사용자 경험을 해치지 않는 자연스러운 수익화
- 글로벌 어필리에이트 네트워크 구축

### 물리적 한계

| 제약 | 현실 | 완화 |
|------|------|------|
| 사용자 기반 부재 | 현재 0명 | 무료 서비스로 획득 우선 |
| PMF 미검증 | 핵심 가치 불확실 | 어필리에이트로 검증 |
| 개발 리소스 | 1인 창업자 | 결제 시스템 보류 |
| 파트너 조건 | 커미션 변동 가능 | 다중 파트너 분산 |

### 100점 기준

| 항목 | 100점 기준 | 현재 목표 |
|------|-----------|----------|
| ARPU | ₩3,500+/월 | ₩500 |
| LTV/CAC | 3:1 이상 | 1.5:1 |
| 수익원 다양성 | 4개 균형 | 어필리에이트 100% |
| 전환율 | 15%+ | 5% |
| 파트너 수 | 10개+ | 3개 |

### 현재 목표

**Phase 1: 30%** (어필리에이트 단일 수익원)
- 무료 서비스 + 어필리에이트
- 주요 파트너 3곳 (쿠팡, iHerb, 무신사)
- ARPU ₩500~1,000 목표
- 사용자 획득 장벽 최소화

### 의도적 제외

| 제외 항목 | 사유 | 재검토 시점 |
|----------|------|------------|
| 구독 결제 시스템 | 사용자 기반 필요 | MAU 10,000+ |
| B2B API 라이선스 | 제품 성숙도 필요 | Series A 이후 |
| 광고 네트워크 | 트래픽 부족 | MAU 50,000+ |
| 글로벌 파트너 | 한국 시장 우선 | 글로벌 확장 시 |

## 결정 (Decision)

### 1. 어필리에이트 우선 수익화 전략 채택

**이룸은 초기에 사용자 확보를 위해 무료 서비스 + 어필리에이트 수익 모델을 채택합니다.**

```
┌─────────────────────────────────────────────────────────────────┐
│                    Phase 1: 어필리에이트 우선                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  사용자                                                          │
│    │                                                            │
│    ├── 무료 분석 (PC-1, S-1, C-1, W-1, N-1, H-1, M-1)          │
│    │                                                            │
│    └── 개인화 제품 추천 ──────────────────────────────────────┐ │
│                                                                │ │
│                  ┌───────────────────────────────────────────┐ │ │
│                  │           어필리에이트 파트너              │ │ │
│                  │  ┌─────────┬─────────┬─────────┐         │ │ │
│                  │  │ 쿠팡    │ iHerb   │ 무신사  │         │ │ │
│                  │  │ 5-10%   │ 10-15%  │ 5-8%    │         │ │ │
│                  │  └─────────┴─────────┴─────────┘         │ │ │
│                  └───────────────────────────────────────────┘ │ │
│                                   │                            │ │
│                                   ▼                            │ │
│                           커미션 수익 ◄────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 보류 항목

다음 항목은 **의도적으로 보류**합니다:

| 보류 항목 | 사유 | 재검토 시점 |
|----------|------|------------|
| **구독 결제 시스템 SDD** | 사용자 기반 확보 후 유료 기능 정의 | MAU 10,000+ |
| **베타 사용자 확보 캠페인** | PMF 검증 후 본격 마케팅 | MVP 안정화 후 |
| **B2B API 라이선스** | 제품 성숙도 필요 | Series A 이후 |

### 3. 초기 수익화 우선순위

```
Phase 1 (현재 ~ MAU 10,000):
├── 어필리에이트 100% (쿠팡, iHerb, 무신사)
└── 예상 ARPU: ₩500~1,000

Phase 2 (MAU 10,000 ~ 50,000):
├── 어필리에이트 60%
├── 구독 30% (Premium 기능 도입)
└── 광고 10%
└── 예상 ARPU: ₩2,000

Phase 3 (MAU 50,000+):
├── 어필리에이트 40%
├── 구독 30%
├── B2B 20%
└── 광고 10%
└── 예상 ARPU: ₩3,500+
```

## 근거 (Rationale)

### 1. 사용자 획득 장벽 최소화

| 모델 | 사용자 진입 장벽 | 전환율 (예상) |
|------|-----------------|--------------|
| 유료 구독 | 높음 | 1-3% |
| Freemium | 중간 | 3-5% |
| **무료 + 어필리에이트** | **낮음** | **N/A (무료)** |

### 2. 제품-시장 적합성 (PMF) 탐색

- 무료 서비스로 사용자 피드백 수집
- 유료화 시점의 핵심 기능 정의
- 실제 결제 의향 검증

### 3. 어필리에이트 장점

| 장점 | 설명 |
|------|------|
| **즉시 수익** | 구현 완료된 제품 추천 시스템 활용 |
| **자연스러운 UX** | 분석 결과 → 제품 추천 → 구매 플로우 |
| **낮은 리스크** | 결제 시스템 구축 비용 없음 |
| **데이터 수집** | 구매 패턴으로 유료 기능 정의 가능 |

### 4. 리스크 완화

| 리스크 | 완화 전략 |
|--------|----------|
| 어필리에이트 수익 불안정 | 다중 파트너 (쿠팡, iHerb, 무신사) |
| 사용자 이탈 | 지속적 기능 개선, 게이미피케이션 |
| 파트너 조건 변경 | 자체 제품 DB 구축 |

## 대안 (Alternatives Considered)

### 1. 구독 우선 모델

**기각 사유**:
- 사용자 기반 없이 결제 시스템 구축은 비효율
- PMF 미검증 상태에서 유료 기능 정의 어려움
- 개발 복잡도 증가 (결제 연동, 구독 관리)

### 2. 광고 우선 모델

**기각 사유**:
- 초기 트래픽으로는 광고 수익 미미
- 광고 네트워크 최소 요구사항 충족 어려움
- UX 저하 우려

### 3. Freemium 즉시 도입

**기각 사유**:
- 무료/유료 기능 경계 정의 필요 (PMF 전에는 불명확)
- 결제 시스템 구축 필요
- 복잡도 증가

## 영향 (Consequences)

### 긍정적

1. **빠른 시장 진입**: 결제 시스템 없이 출시 가능
2. **사용자 획득 용이**: 완전 무료로 진입 장벽 제거
3. **수익 검증**: 어필리에이트로 수익 모델 검증
4. **리소스 집중**: 핵심 기능 개발에 집중

### 부정적

1. **수익 제한**: 어필리에이트만으로는 ARPU 낮음
2. **투자자 설득**: 다양한 수익원 부재로 IR 시 설명 필요
3. **파트너 의존**: 어필리에이트 파트너 조건에 의존

### 중립적

1. **구독 SDD 작성 지연**: MAU 10,000+ 시점까지 보류
2. **마케팅 비용 절감**: 유료 전환 마케팅 불필요
