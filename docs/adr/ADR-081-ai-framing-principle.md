# ADR-081: AI Framing Principle

## 상태

`accepted`

## 날짜

2026-03-09

## 0. 궁극의 형태 (P1)

### 이상적 최종 상태

```
"모든 AI 분석 결과 텍스트가 성장 지향적이면서 바이럴 가능한 톤을 유지한다"

- 부정적 판정 없이 성장 방향 제시
- 점수는 유지 (성취감 + 바이럴)
- 타입 라벨로 정체성 강화
```

### 현재 목표: 90%

- D4 부정 표현 벌크 수정 완료 (커밋 23410c50, 28파일)
- 추가: S-1/OH-1 타입 라벨의 프레이밍 가이드

### 의도적 제외

| 제외 항목                     | 이유        | 재검토 시점               |
| ----------------------------- | ----------- | ------------------------- |
| AI 생성 텍스트 실시간 톤 검증 | 복잡도 높음 | Gemini 프롬프트 고도화 시 |

## 1. 맥락 (Context)

AI 분석 결과 텍스트의 프레이밍이 사용자 행동에 미치는 영향:

- **부정 프레이밍**: "수분도 45점 — 부족합니다" → 이탈/불만
- **성장 프레이밍**: "수분도를 높이면 더 건강한 피부를 만들 수 있어요" → 재방문/공유
- **정체성 프레이밍**: "나는 민감 건성 타입" + 78점 → 바이럴 + 성취감

## 2. 결정 (Decision)

3가지 프레이밍 원칙을 적용한다:

### 원칙 1: Identity-First (ADR-080)

타입 라벨을 점수보다 먼저 표시.

### 원칙 2: Growth-Oriented

결과 텍스트에서 부정적 판정 대신 성장 방향 제시:

| 금지 패턴         | 권장 패턴                |
| ----------------- | ------------------------ |
| "부족합니다"      | "더 높일 수 있어요"      |
| "나쁜 상태입니다" | "케어를 시작하면 좋아요" |
| "문제가 있습니다" | "신경 쓸 부분이 있어요"  |
| "위험합니다"      | "전문가 상담을 권해요"   |

### 원칙 3: Score as Achievement

점수를 판정이 아닌 성취 지표로 표현:

```
❌ "45점 — 낮은 점수입니다"
✅ "45점 — 적극적인 케어를 시작하면 좋아요"
✅ "78점 — 좋은 상태이지만 더 발전할 수 있어요"
✅ "92점 — 전반적으로 건강한 상태예요"
```

## 3. 대안 (Alternatives Considered)

| 대안      | 장점                    | 단점         | 제외 사유                     |
| --------- | ----------------------- | ------------ | ----------------------------- |
| 점수 제거 | 부정 프레이밍 원천 차단 | 바이럴 약화  | 사용자 피드백: 점수 공유 원함 |
| 현상 유지 | 변경 없음               | 부정 톤 잔존 | D4 수정 이미 시작됨           |

## 4. 결과 (Consequences)

### 긍정적 결과

- 사용자 만족도 향상 (성장 지향 → 재방문)
- 바이럴 유지 (점수 + 타입 라벨)
- D4 수정과 일관성

### 부정적 결과

- 없음 (기존 D4 수정 방향과 동일)

## 5. 구현 가이드

이미 적용된 패턴 (`classifyByRange`):

```typescript
classifyByRange(score, [
  { max: 60, result: '적극적인 케어를 시작하면 좋아요' },
  { max: 80, result: '좋은 상태이지만 더 신경쓸 부분이 있어요' },
  { min: 80, result: '전반적으로 건강한 상태예요' },
]);
```
