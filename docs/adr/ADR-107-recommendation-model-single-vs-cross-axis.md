# ADR-107: 추천 모델 — 단일축 즉시추천 + 크로스축 종합 컨설팅 (2-tier)

- **Status**: accepted
- **Date**: 2026-06-17
- **관련**: [ADR-098](./ADR-098-identity-redefinition-5axis-model.md)(5축 정체성), [ADR-099](./ADR-099-integrated-analysis-flow.md)(통합 분석), [ADR-106](./ADR-106-demo-positioning-and-investment-staging.md)(투자 단계 — 쇼핑=use of funds)

---

## Context

각 분석 축이 "추천"을 어떻게/어디서 제공할지 일관 기준이 없어 불균형 발생. 실측(2026-06-17):

- **PC**: 색상 팔레트 추천 (단일축) ✅
- **피부**: 케어 루틴·성분 (단일축) ✅
- **체형**: 실루엣/핏 가이드 (단일축) ✅ + `/closet/recommend`(PC+체형+날씨)
- **헤어**: 진단 + 케어팁만. **얼굴형(face_shape)을 계산·저장하면서도 컷 추천에 미사용**. 🔴 스타일 추천 부재
- **메이크업**: 진단 + PC기반 색상 제품 추천 ⚠️ (얼굴형별 기법은 미구현)
- **통합 분석**: PersonaNarrative + AxesSummary + ActionPlan + CrossInsights(체형×얼굴형 스타일 1줄) + Curation(lip/base/skincare/outfit/hair 카테고리 링크)
- 🔴 버그: CurationCard 헤어 링크가 **존재하지 않는 `/style/hair`**로 감(죽은 링크)

## Decision

### 원칙: 추천의 범위 = 그 추천을 잘 하는 데 필요한 입력 축의 수

| 추천 종류         | 필요 입력 | 위치                                        |
| ----------------- | --------- | ------------------------------------------- |
| 단일축으로 충분   | 1개 축    | **단독 분석 결과 (즉시추천)**               |
| 여러 축 결합 필요 | 2+ 축     | **통합 분석 (종합 컨설팅 = 비주얼 디렉터)** |

### 4대 표현 규칙

1. **추천 우선, 진단은 근거** — 결과 화면은 "무엇을 하세요"(추천)를 먼저, 진단 점수는 그 근거로. (리포트가 아니라 컨설턴트)
2. **단일축 추천 최대화** — 각 모듈은 _자기 데이터만으로 가능한_ 추천을 전부 제공. (헤어 = 케어 **+ 얼굴형 기반 컷** 둘 다 단일축)
3. **크로스축 의존성 가시화 (teaser → unlock)** — 단독에서 "여기까진 지금 가능 + [크로스축 추천]은 OO 분석 완료 시 종합 분석에서" 형태로 안내 → 완성 동기 + 통합 가치 체감.
4. **통합 = 비주얼 디렉터** — 크로스축 스타일링/코디/메이크업 종합 컨설팅의 목적지. (이미 CrossInsights+Curation 골격 존재)

### 축별 적용

| 축       | 단독(단일축 즉시)              | 통합(크로스축)                                     |
| -------- | ------------------------------ | -------------------------------------------------- |
| PC       | 색상 팔레트                    | (입력원)                                           |
| 피부     | 케어 루틴/성분                 | 베이스 메이크업 방향(×PC)                          |
| 체형     | 실루엣/핏 가이드               | 코디 색+실루엣(×PC)                                |
| **헤어** | 케어(단일축) + 종합분석 teaser | **컷(×얼굴형) + 염색(×PC) + 스타일 vibe** ⚠️아래   |
| 메이크업 | PC기반 색상 제품               | **얼굴형별 기법(신규, ×PC/피부)**                  |
| 코디     | —                              | PC+체형 기반 코디(통합 Curation→/closet/recommend) |

### 조언 vs 쇼핑 분리 (ADR-106 연계)

- **조언 추천**(컷 종류·메이크업 기법·스타일 가이드) = 입력 데이터로 _지금_ 생성 가능 → 구현.
- **쇼핑 추천**(실제 살 상품·링크) = 상품 DB 공백이라 **투자 후 use of funds**. 빈 결과로 가는 약속성 링크 금지.

## Consequences

### ⚠️ 데이터 모델 정정 (2026-06-17 실측)

- `face_shape`(얼굴형)는 **`makeup_analyses`에만 저장**되고 **`hair_analyses`엔 없음**. `lib/analysis/hair/face-shape-analyzer.ts`가 계산은 하나 hair 테이블엔 미persist.
- 따라서 **헤어 "컷" 추천도 face_shape 필요 → 크로스축**(헤어 단독으로는 불가). 헤어 스타일(컷+염색) 전체가 통합 플로우(메이크업의 face_shape + PC 결합) 소관.
- `recommendHairstyles(faceShape)` 함수는 `lib/analysis/hair/index.ts`에 이미 export됨 → 통합 플로우에서 makeup.face_shape를 입력으로 연결만 하면 됨.

### 즉시 구현 (조언)

- 🔴 헤어 단독: 케어 유지 + **"어울리는 헤어스타일·염색은 종합 분석에서(얼굴형·퍼스널컬러 결합)" teaser/CTA** 추가. (단독엔 스타일 추천 자체를 두지 않음 — 데이터 부재)
- 🔴 CurationCard 헤어 죽은 링크(`/style/hair`, 미존재) 수정 → 통합 내 헤어 섹션 또는 헤어 결과로. (상품 큐레이션은 DB 공백이라 use of funds.)
- 통합 플로우: `recommendHairstyles(makeup.faceShape)` 연결로 컷 추천 표면화(조언). 염색은 ×PC.
- 메이크업: 얼굴형별 기법 가이드(조언) 추가 검토.

### 연기 (use of funds)

- 모든 "실제 상품 구매" 큐레이션은 상품 DB 확보 후.

### 문서

- 지원서/로드맵에서 "왜 통합 분석이 차별점인가" = 이 ADR의 크로스축 종합 컨설팅. (룩핀·화해는 단일 도메인이라 크로스축 불가)
