# ADR-104: 이룸 출시 조건 체크리스트 — 비전 완성 기준

## 상태

`accepted` (Phase F/G 코드 완성, 수동 검증 대기)

## 날짜

2026-04-24 (원본) · 2026-04-25 (Phase F/G 반영 업데이트)

## 0. 궁극의 형태 (P1)

### 이상적 최종 상태

```
"이룸답지 않은 이룸은 출시하지 않는다"

- 출시 시점 = 슬로건 "온전한 나를 찾는 여정"이 실제로 구현된 시점
- 5개 분석 결과의 나열이 아닌, "1명의 나"로 통합된 결과 제공
- 사용자가 결과를 받고 "그래서?"를 떠올리지 않게 다음 행동 명시
- 실제 AI 응답 품질이 Mock 대비 정성적으로 나음
- 축 간 연결이 사용자에게 시각적으로 보임
- 어필리에이트가 통합 프로필 기반 1개 큐레이션
```

### 물리적 한계

| 항목                      | 한계                                                                  |
| ------------------------- | --------------------------------------------------------------------- |
| 정성 평가 주관성          | "Mock 대비 나음"은 측정 어려움 — 본인 + 지인 3-5명 피드백으로 대체    |
| Gemini API 비용           | 실제 호출 전환 시 월 비용 증가 (Mock은 무료) — MAU 1만 기준 ~$50 예상 |
| 개발 시간 vs 기회비용     | 출시 지연 = 경쟁사 선점 리스크 (Oh My Skin 이미 출시 상태)            |
| 1인 개발 범위             | 5개 Phase F 전체는 1-2주 작업 — 우선순위 엄수 필요                    |
| "비전 완성" 정의의 모호성 | 체크리스트 없으면 "조금만 더"가 반복됨 — 본 ADR로 명문화              |

### 100점 기준

- 출시 시 사용자가 이룸 한 문장 정의 가능: "내가 어떤 사람인지 AI가 이해해주고 뭘 하면 좋을지 알려주는 앱"
- 통합 분석 완료 사용자의 결과 페이지 체류 ≥ 2분 (읽는 분량 있음)
- Partial Success 아닌 Complete 성공률 ≥ 85%
- 지인 피드백 "이거 진짜 나에 대해 알고 하는 말 같다" 과반수
- 출시 후 첫 주 재방문율 ≥ 30%

### 현재 목표: 95% (2026-04-25 업데이트)

- ✅ ADR-104 (본 문서) 작성 + 5개 체크리스트 항목 확정
- ✅ Phase F.1~F.5 **코드 전체 완성** — 5개 항목 모두 구현됨
- ✅ Phase G 보강 (4/24~25) — 큐레이션 링크가 실제 목적지에서 파라미터 파싱/배너 노출/빈 옷장 우회까지 동작 (기준 #5 현실화)
- 🟡 **남은 5%**: 수동 검증만 남음
  - FORCE_MOCK_AI=false 전환 후 실제 Gemini 품질 체감 (기준 #3 본인/지인 평가)
  - 실기기에서 결과 페이지 체류/재방문 확인 (100점 기준 §)
- 출시 결정은 여전히 **체크리스트 3개 이상 ✅** 기준 — 현재 코드 기준 5개 모두 ✅ (지인 평가만 남음)

### 의도적 제외

| 제외 항목              | 이유                                          | 재검토 시점        |
| ---------------------- | --------------------------------------------- | ------------------ |
| A/B 테스트로 비전 검증 | MAU 소규모, 통계적 유의 불가                  | MAU 1만+           |
| 외부 마케팅/홍보 계획  | 앱 완성 먼저, 마케팅은 완성 후                | 체크리스트 3개+    |
| 투자자 피치/대회 지원  | 자체 비전 달성이 우선 (지원 결과 모두 불합격) | 자력 트랙션 확보   |
| 다국어 지원            | 한국 사용자부터 완성                          | 한국 검증 후       |
| 모바일 완전 포팅       | 웹 먼저 비전 완성 → 모바일은 결과 표시만      | 웹 체크리스트 완료 |

## 1. 맥락 (Context)

### 1.1 방향 전환 배경 (2026-04-24)

사용자 결정: **"출시는 나중에 해도 되, 이룸이 이루고자 하는 것을 이루는 것이 더 중요"**

이전까지의 압박 원인:

- MEMORY 기록: Google Play 5/1 목표
- 창구 8기, GFSA, 정주영 창업경진대회 모두 불합격 → 트랙션 확보 필요성
- Oh My Skin이 동일 스택으로 이미 출시 (경쟁 압박)

그러나 Phase A~E 완성 후 상태 진단:

- ✅ "5축 측정 + 통합 플로우" 기술적 완성
- ❌ "1명의 나"로 통합된 결과 (여전히 5개 따로)
- ❌ "그래서 뭘 해야 하는지" (링크 5개 나열)
- ❌ 실제 AI 응답 (Mock 상태)
- ❌ "세상과 연결" (축별 산발적)

지금 출시하면 **"잘 만든 데모"** 수준. 이룸의 슬로건 "온전한 나를 찾는 여정"을 증명 못 함.

### 1.2 이룸이 "이루고자 하는 것" 재확인

CLAUDE.md 핵심 가치:

- **슬로건**: 온전한 나를 찾는 여정 / Know yourself, wholly.
- **궁극의 목적지**: 나를 가장 잘 아는 존재, 그래서 세상과 나를 연결해주는 존재
- **3단계 진화**: 분석기(현재) → 조언자(다음) → 동반자(궁극)
- **수익 원칙**: 사용자는 영원히 무료. 가치 먼저, 수익은 그다음.

ADR-098: "거울 + 옷장을 이해해주는 앱 — 당신의 시각적 정체성 5축을 해석하고, 당장 실행할 수 있게 세상과 연결한다"

### 1.3 출시 시점 재정의

> **기존**: "코드가 돌아가고 DB 준비되면 출시"
> **신규**: "이룸답지 않으면 출시 안 함" — 체크리스트 기반

## 2. 결정 (Decision)

### 2.1 출시 조건 체크리스트 (5개)

| #   | 조건                  | 검증 방법                                                   | 완료 기준      |
| --- | --------------------- | ----------------------------------------------------------- | -------------- |
| 1   | **1명의 나로 통합**   | 결과 페이지에 "나 프로필" 내러티브 섹션 존재 + AI 호출 동작 | Phase F.1 완료 |
| 2   | **다음 행동이 명확**  | 결과 페이지에 "지금/이번주/다음달" 3단계 조언 표시          | Phase F.2 완료 |
| 3   | **실제 AI 응답 품질** | 본인 + 지인 3명 중 2명이 "Mock 대비 나음" 평가              | Phase F.3 완료 |
| 4   | **축 간 연결이 보임** | 결과 페이지에 "PC × S", "C × H" 등 조합 인사이트 3개 이상   | Phase F.4 완료 |
| 5   | **통합 큐레이션**     | 세션 ID 기반 제품 세트 1개 이상 노출 (어필리에이트 연결)    | Phase F.5 완료 |

### 2.2 출시 트리거 규칙

```
5개 중 3개 이상 ✅ → 출시 준비 착수
5개 모두 ✅        → 비전 완성 출시
5개 중 2개 이하    → 출시 미실시 (추가 Phase 지속)
```

**필수 포함**: 체크리스트 #1 (나 프로필 내러티브)은 반드시 포함. 나머지 중 2개 선택.

### 2.3 Phase F 우선순위

```
F.1 나 프로필 내러티브     ← 최우선 (체크리스트 #1)
F.3 실제 AI 연결           ← F.1 품질 평가를 위해 병행
F.2 조언자 모드 (3단계)    ← 체크리스트 #2
F.4 크로스 모듈 시너지     ← 체크리스트 #4
F.5 통합 큐레이션          ← 체크리스트 #5 (어필리에이트)
```

### 2.4 Phase F 범위 외 (Phase G+로 연기)

- 동반자 단계 (일상 맥락 결합, LLM Agent 상담)
- 소셜 기능 (친구와 비교, 스타일 공유)
- 시간축 비교 (세션 2개 이상 필요 — 출시 후)
- 다국어 (한국 검증 후)
- 22위젯 홈 재구조화 (MAU 1만+)

## 3. 대안 (Alternatives Considered)

| 대안                      | 장점                 | 단점                                    | 제외 사유                |
| ------------------------- | -------------------- | --------------------------------------- | ------------------------ |
| 즉시 출시 + 점진 개선     | 데이터 수집 빠름     | "잘 만든 데모" 수준, 첫인상 회복 어려움 | ADR-098 비전 훼손 리스크 |
| 완벽주의 (5개 모두 ✅)    | 완벽한 출시          | 출시 무한 지연                          | 기회비용, 1인 개발 현실  |
| MAU 기반 로드맵           | 데이터 기반 우선순위 | MAU 없으면 판단 불가                    | 닭-달걀 문제             |
| 체크리스트 없이 "감" 출시 | 빠름                 | "조금만 더"가 반복                      | 명문화 없이 객관성 불가  |
| 2개만 완료해도 출시       | 빠름                 | 비전 미달                               | ADR-104 의의 약화        |

## 4. 결과 (Consequences)

### 긍정적 결과

- **명확한 기준** — "출시했다/안 했다"가 체크리스트로 자동 판단
- **"잘 만든 데모" 함정 회피** — 비전 달성 없이 출시 안 됨
- **우선순위 자동화** — Phase F.1→F.3→F.2→F.4→F.5 순서 고정
- **사용자 결정 명문화** — 미래의 본인/협업자도 같은 기준으로 판단
- **실질적 차별화** — 경쟁사(Oh My Skin)가 못 따라오는 비전 완성

### 부정적 결과

- **출시 지연** — 최소 1-2주 추가 (Phase F 전체)
- **경쟁사 선점 확대** — Oh My Skin이 시장 더 점유
- **Gemini 비용 발생** — Mock 대비 실제 호출 시 월 $50+ (MAU 1만 가정)
- **피드백 루프 지연** — 사용자 데이터 없이 비전 추진

### 리스크

- **"완벽주의 함정"** — 체크리스트 자체가 늘어나 출시 영구 연기
  - 완화: **체크리스트는 5개 고정. 추가 항목은 Phase G로.**
- **정성 평가의 모호성** — "Mock 대비 나음"이 감정적 판단
  - 완화: 본인 + 지인 3명 과반수로 객관화, 기록 남김
- **기회비용** — 출시했으면 받았을 피드백을 놓침
  - 완화: 체크리스트 3개 ✅이면 **베타 출시**(소수 사용자 대상) 가능
- **Gemini 품질 이슈** — 실제 응답이 기대에 미달 시 F.3 통과 어려움
  - 완화: 프롬프트 엔지니어링으로 대응, Level 2 가이드(.claude/rules/ai-integration.md §원리 기반) 활용

### 5.4 Phase G 보강 (2026-04-25 추가)

Phase F 완료 후 기준 #5 "통합 큐레이션"의 **링크 실행 단계**에서 사용자 기대 위반이 발견되어 Phase G로 보강했습니다.

**발견 (2026-04-25)**:

- 큐레이션이 생성하는 URL `/beauty?category=skincare&source=integrated&session=...`은 목적지 페이지가 존재하나 `useSearchParams` 미적용 → 파라미터 무시
- outfit 카드 → `/closet/recommend` 이동 시 **옷장이 비어있으면 빈 상태**로 직행 (사용자 기대 위반 심각)
- v2 API route 4개가 UI 미호출 orphan 상태

**Phase G 조치**:

1. **웹**: `/beauty`·`/closet/recommend` 양쪽에 `useSearchParams` 추가 — 카테고리 자동 선택, 통합 배너 표시, 빈 옷장 시 "먼저 옷장 등록하기" 우회 CTA
2. **모바일**: `useHasClosetItems` 훅 신규 + `/(tabs)/beauty`·`/(closet)/recommend` 동일 로직 포팅 (웹↔모바일 동기화)
3. **큐레이션 로직**: `ComposeCurationInput.hasClosetItems` 옵션 추가 → outfit 카드 href를 상황에 맞춰 분기
4. **v2 API**: `@deprecated` JSDoc 마커 + 관련 문서 5개 "Superseded" 안내 (향후 물리 삭제는 별도 클린업)

**결과**: 기준 #5 "통합 큐레이션"이 "선언적"에서 "실제로 사용자를 올바른 맥락으로 데려가는" 상태로 업그레이드. 코드/문서: Phase G 블록 커밋 대기.
