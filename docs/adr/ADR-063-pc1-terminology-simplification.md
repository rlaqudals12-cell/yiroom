# ADR-063: PC-1 전문 용어 → 일상 용어 변경

## 상태

accepted

## 날짜

2026-02-06

## 컨텍스트

PC-1 결과 페이지 및 드레이핑 시뮬레이션에서 전문 용어가 사용되고 있어,
타겟 사용자(20-35세 일반 소비자)의 접근성이 떨어진다.

### 현재 문제 용어

| 위치      | 현재 용어     | 문제                                   |
| --------- | ------------- | -------------------------------------- |
| 탭 이름   | "드레이핑"    | 퍼스널컬러 전문 용어, 일반인 이해 불가 |
| 서브탭    | "시뮬레이터"  | 기술 용어                              |
| 서브탭    | "색상 팔레트" | 디자인 용어                            |
| 점수 라벨 | "균일도"      | 통계 용어, 높을수록 좋은지 불명확      |

## 결정

전문 용어를 일상 한국어로 변경한다.

| 변경 전             | 변경 후       | 근거                   |
| ------------------- | ------------- | ---------------------- |
| 드레이핑            | 색상 입혀보기 | 행동 중심, 직관적 이해 |
| 시뮬레이터          | 미리보기      | 친숙한 UI 용어         |
| 색상 팔레트         | 나의 색상     | 개인화 느낌            |
| 균일도              | 어울림 점수   | 결과 중심, 긍정적 표현 |
| 드레이핑 시뮬레이션 | 색상 입혀보기 | 카드 제목 일관성       |

## 대안

### 대안 1: 전문 용어 + 툴팁

- 장점: 교육적 효과
- 단점: 모바일에서 툴팁 UX 불량, 매번 확인 필요
- **탈락 이유**: 타겟 사용자는 학습이 아닌 체험 목적

### 대안 2: 용어집 페이지

- 장점: 궁금한 사용자만 참조
- 단점: 첫 화면에서 혼란 여전, 별도 페이지 이탈
- **탈락 이유**: 첫인상에서 접근성 해결 안 됨

### 대안 3: 이중 표기 (일상 + 전문)

- 예: "어울림 점수 (균일도)"
- 장점: 양쪽 모두 충족
- 단점: UI 복잡, 모바일 공간 부족
- **탈락 이유**: P4(단순화) 위배

## 영향

- 변경 파일 6개 (UI 텍스트만)
- API 변경 없음, DB 변경 없음
- 기존 테스트 수정 필요 (텍스트 매칭 테스트가 있다면)
