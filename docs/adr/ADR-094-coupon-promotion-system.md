# ADR-094: 쿠폰/프로모션 시스템

## 상태

`accepted`

## 날짜

2026-03-15

## 0. 궁극의 형태 (P1)

### 이상적 최종 상태

"사용자가 어필리에이트 제품 구매 시 적용 가능한 쿠폰/프로모션을 자동으로 제시받고, 최적의 할인 조합을 원클릭으로 적용하여 최저가로 구매할 수 있는 상태"

### 물리적 한계

| 항목        | 한계                                                       |
| ----------- | ---------------------------------------------------------- |
| 외부 쿠폰   | 쿠팡/올리브영 등 외부 쇼핑몰 쿠폰은 이룸에서 제어 불가     |
| 결제 시스템 | 직접 결제 미구현 → 어필리에이트 링크 기반 할인 정보 제공만 |
| 쿠폰 악용   | 다중 계정으로 쿠폰 남용 가능성                             |

### 100점 기준

| 지표        | 100점                         | 현재 | 목표               |
| ----------- | ----------------------------- | ---- | ------------------ |
| 쿠폰 타입   | 5+ (정액/정률/배송/번들/시즌) | 0    | 3 (정액/정률/배송) |
| 적용 자동화 | 자동 최적 조합                | N/A  | 수동 적용          |
| 파트너 연동 | 실시간 쿠폰 동기화            | N/A  | 이룸 자체 쿠폰만   |

### 현재 목표: 60%

### 의도적 제외

- 외부 쇼핑몰 쿠폰 크롤링
- 자동 최적 쿠폰 조합 추천
- 쿠폰 거래/양도 기능
- 포인트 적립 시스템

## 1. 컨텍스트

이룸은 어필리에이트 수익 모델을 사용(ADR-054). 사용자 리텐션과 구매 전환율 향상을 위해 이룸 자체 쿠폰/프로모션 시스템이 필요. 직접 결제가 없으므로 "할인 정보 제공 + 어필리에이트 링크"를 결합한 형태.

## 2. 결정

### 쿠폰 타입 3종

| 타입             | 설명      | 예시                    |
| ---------------- | --------- | ----------------------- |
| `percentage_off` | 정률 할인 | 10% 할인 (최대 5,000원) |
| `fixed_off`      | 정액 할인 | 3,000원 할인            |
| `free_shipping`  | 무료 배송 | 배송비 무료             |

### 적용 범위

- **파트너별**: 특정 파트너(쿠팡/올리브영) 제품 한정
- **카테고리별**: 스킨케어/메이크업/영양제 등 한정
- **전체**: 모든 제품 적용

### DB 스키마

```sql
promotions (id, title, promotion_type, discount_value, min_purchase, max_discount,
            partner_name, category, starts_at, expires_at, max_uses, is_active)
user_coupons (id, clerk_user_id, promotion_id, coupon_code UNIQUE, is_used, used_at, expires_at)
```

### 보안

- 쿠폰 코드: `crypto.randomUUID()` (예측 불가)
- 중복 사용 방지: DB UNIQUE 제약 + `is_used` 플래그
- RLS: `clerk_user_id` 기반 본인 쿠폰만 조회/사용

### 비즈니스 모델 연동

- 쿠폰은 어필리에이트 커미션에서 차감하는 형태 (이룸 부담)
- 프로모션 기간 한정으로 비용 관리
- `max_uses` 필드로 전체 발급 수량 제한

## 3. 대안 검토

| 대안                    | 장점             | 단점                 | 결정     |
| ----------------------- | ---------------- | -------------------- | -------- |
| **A. 포인트 시스템**    | 유연함           | 구현 복잡, 결제 필요 | 탈락     |
| **B. 외부 쿠폰 크롤링** | 사용자 가치 높음 | 법적 리스크, 불안정  | 탈락     |
| **C. 자체 쿠폰**        | 제어 가능, 단순  | 외부 할인 미반영     | **채택** |
