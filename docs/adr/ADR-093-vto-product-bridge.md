# ADR-093: Virtual Try-On 제품 브릿지

## 상태

`accepted`

## 날짜

2026-03-15

## 0. 궁극의 형태 (P1)

### 이상적 최종 상태

"사용자가 Virtual Try-On으로 메이크업을 시뮬레이션한 후, 동일한 색상의 실제 제품을 즉시 추천받고 원클릭으로 구매할 수 있는 상태"

### 물리적 한계

| 항목         | 한계                                                            |
| ------------ | --------------------------------------------------------------- |
| 색상 차이    | 화면 색상 ≠ 실제 제품 색상 (모니터/조명 영향)                   |
| 제품 DB 색상 | 제품 대표 색상 데이터 부족 (수동 태깅 필요)                     |
| 매칭 정확도  | Delta-E ≤ 10이면 "비슷한 색상"이지만 사용자 기대와 다를 수 있음 |

### 100점 기준

| 지표         | 100점       | 현재 | 목표         |
| ------------ | ----------- | ---- | ------------ |
| 매칭 정확도  | Delta-E ≤ 5 | N/A  | Delta-E ≤ 15 |
| 추천 제품 수 | 10+         | 0    | 5개          |
| 구매 전환율  | 15%+        | 0%   | 측정 시작    |

### 현재 목표: 85%

### 의도적 제외

- AR 실시간 구매 (VTO 중 구매 버튼)
- 색상 보정 AI (모니터 캘리브레이션)

## 1. 컨텍스트

VTO 5개 엔진(립/블러셔/헤어/아이섀도/파운데이션)과 시즌별 프리셋(season-presets.ts)이 완성되어 있으나, 시뮬레이션 결과에서 실제 제품으로의 연결이 없음.

## 2. 결정

**Lab 색공간 Delta-E 기반 제품 매칭 + 시즌 보정** 방식 채택.

### 매칭 알고리즘

1. VTO 적용 색상 (RGBA) → Lab 색공간 변환
2. 제품 DB의 대표 색상과 Delta-E(CIE76) 계산
3. Delta-E ≤ 15인 제품 필터링
4. 시즌 보정: 사용자 PC-1 시즌과 제품 시즌 일치 시 +15점
5. 매칭 점수 + 인기도 + 가격 접근성으로 최종 정렬
6. 상위 5개 반환 + 어필리에이트 딥링크

### MakeupType → 카테고리 매핑

```typescript
const VTO_CATEGORY_MAP = {
```

## 3. 대안 검토

| 대안                     | 장점                   | 단점                        | 결정     |
| ------------------------ | ---------------------- | --------------------------- | -------- |
| **A. 프리셋 매핑**       | 정확도 높음 (1:1 매핑) | 확장성 낮음, 수동 작업 필요 | 탈락     |
| **B. RGB 유클리드 거리** | 구현 간단              | 인간 시각과 불일치          | 탈락     |
| **C. Lab Delta-E**       | 인간 시각에 근접       | Lab 변환 필요               | **채택** |
