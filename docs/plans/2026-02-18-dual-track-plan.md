## 3. Track 2: 모바일 UI 재설계

### 3.1 목표

웹 디자인 시스템 기반으로 모바일 앱 UI를 전면 재설계하여 브랜드 일관성 확보

### 3.2 현재 상태 분석

| 영역            | 웹 (S등급)           | 모바일 (현재)     | 격차 |
| --------------- | -------------------- | ----------------- | ---- |
| 색상 체계       | 이룸 브랜드 컬러     | 기본 React Native | 큼   |
| 타이포그래피    | Pretendard, 해요체   | 시스템 폰트       | 큼   |
| 컴포넌트 스타일 | shadcn/ui 기반       | 기본 NativeWind   | 큼   |
| 레이아웃        | 반응형, 카드 기반    | 기본 스크롤뷰     | 중간 |
| 분석 결과 UI    | 차트+그래프+제품카드 | 텍스트 위주       | 큼   |
| 온보딩          | 단계별 가이드        | 기본 폼           | 중간 |

### 3.3 재설계 단계

#### Phase M-1: 디자인 토큰 + 공통 컴포넌트 (3세션)

```
- 색상/타이포/스페이싱 토큰 정의 (NativeWind tailwind.config)
- Button, Card, Input, Badge 등 기본 컴포넌트
- 탭 네비게이션 스타일링
- 아이콘 세트 통일
```

#### Phase M-2: 핵심 화면 재설계 (5세션)

```
- 홈 대시보드 (웹 홈과 정보 구조 일치)
- 분석 화면 (카메라 → 결과 → 제품 추천)
- 프로필/설정
- 온보딩 플로우
```

#### Phase M-3: 부가 화면 + 인터랙션 (3세션)

```
- 운동/영양 탭
- 기록/리포트
- 챌린지/소셜
- 애니메이션/트랜지션
```

#### Phase M-4: 빌드 + 배포 (2세션)

```
- Production AAB 재빌드 (env 포함)
- Google Play 내부 테스트 등록
- 스토어 스크린샷/설명 작성
- 심사 제출
```

### 3.4 디자인 원칙

1. **웹 우선 참조**: 웹 UI를 기준으로 모바일 최적화 (역방향 아님)
2. **NativeWind 활용**: Tailwind 클래스명 웹과 최대한 일치
3. **플랫폼 네이티브 UX**: 제스처, 햅틱, 안전 영역 등 모바일 특화 UX 적용
4. **점진적 개선**: Phase M-1 완료 후 빌드 가능한 상태 유지
