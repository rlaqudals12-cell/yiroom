## 1. 개요

### 1.1 목표

웹의 Refined Expertise v2 **색상 시스템만** 모바일에 동기화 (레이아웃 유지)

### 1.2 핵심 원칙

| 원칙            | 설명                                  |
| --------------- | ------------------------------------- |
| **색상만 변경** | 기존 레이아웃/컴포넌트 구조 100% 유지 |
| **웹-앱 통일**  | 동일한 색상 토큰으로 브랜드 일관성    |
| **최소 변경**   | NativeWind/StyleSheet 색상 값만 수정  |
| **기능 무변경** | 기존 기능 100% 유지                   |

```
변경 O: 색상 값 (primary, background, module colors)
변경 X: 레이아웃, 컴포넌트 구조, 네비게이션, 기능
```

### 4.2 Colors.ts 변경 예시

```typescript
// 변경 전 (블루 기반)
export const Colors = {
  light: {
    primary: '#2e5afa', // 이룸 블루
    background: '#f8f9fc', // 쿨톤 화이트
  },
};

// 변경 후 (민트 기반 - 웹과 동일)
export const Colors = {
  light: {
    primary: '#4A8B7C', // 로고 민트 (oklch 0.45 0.12 155)
    background: '#FDFCFB', // 웜 화이트 (oklch 0.99 0.005 90)
    card: '#FFFFFF', // 순백
    text: '#1C1C1E', // 소프트 블랙
    border: '#E8E7E5', // 웜톤 보더

    // 모듈 색상 (뮤트 톤)
    workout: '#D4A574', // 뮤트 오렌지
    nutrition: '#7AB095', // 뮤트 그린
    skin: '#C9A0A0', // 뮤트 핑크
    body: '#7A9BC0', // 뮤트 블루
    personalColor: '#9B7BA8', // 뮤트 퍼플
  },
  dark: {
    primary: '#5CA08F', // 민트 (다크모드)
    background: '#1C1C1E',
    // ...
  },
};
```

## 5. 컴포넌트 라이브러리

### 5.1 버튼

| 타입      | 스타일                   | 용도        |
| --------- | ------------------------ | ----------- |
| Primary   | 민트 배경, 화이트 텍스트 | 주요 CTA    |
| Secondary | 투명, 민트 테두리        | 보조 액션   |
| Ghost     | 투명, 민트 텍스트        | 링크형      |
| Module    | 모듈 색상 배경           | 모듈별 액션 |

### 5.2 카드

```
기본 카드:
- Background: #FFFFFF
- Border-radius: 16px
- Shadow: 0 2px 8px rgba(0,0,0,0.06)
- Padding: 16px

모듈 카드:
- Background: 모듈 light 색상
- Border-left: 4px solid 모듈 색상
```

### 5.3 입력 필드

```
기본 상태:
- Background: #F8F7F5
- Border: 1px solid #E8E7E5
- Border-radius: 12px

포커스 상태:
- Border: 1.5px solid mint
- Shadow: 0 0 0 3px mint/10%
```
