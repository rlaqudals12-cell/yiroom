## 1. 페르소나 워크스루

### 1.1 페르소나 정의

| ID  | 역할       | 특성                     | 핵심 질문                                    |
| --- | ---------- | ------------------------ | -------------------------------------------- |
| P1  | **탐색자** | 앱 설치 직후, 분석 0~1개 | "이 앱 뭐하는 거지? 뭘 먼저 해야 해?"        |
| P2  | **학습자** | 분석 3~4개, 일주일째     | "다음엔 뭘 해볼까? 내 결과는 어디서 보지?"   |
| P3  | **일상화** | 분석 6개+, 매일 접속     | "오늘 새로운 인사이트가 있나?"               |
| P4  | **복귀자** | 2주 미접속 후 복귀       | "그때 결과가 뭐였더라? 다시 볼 이유가 있나?" |

### 1.2 2-Track 워크스루

#### Track A: Quick (5분) — 매 UI 변경 시

**대상 페르소나**: P1(탐색자) + P3(일상화)만
**검증 단계**: Step 1(진입) + Step 3(판단)만

각 페르소나별로:

**Step 1: 진입 순간 (0~3초)**

- 시각적 초점(focal point)이 명확한가?
- 첫 눈에 보이는 정보가 이 페르소나에게 가치 있는가?
- "뭐지 이게?" 반응이 아니라 "아, 이거구나" 반응인가?

**Step 3: 판단 (10~30초)**

- CTA가 1~2개로 명확한가?
- 가장 중요한 행동이 시각적으로 가장 돋보이는가?
- 이 페르소나의 목표와 제시된 CTA가 일치하는가?

#### Track B: Full (30분) — Cycle 2 또는 리디자인 시

**대상 페르소나**: 4명 전체
**검증 단계**: Step 0 + 5단계 전체

**Step 0: 궁극의 형태 정의 (워크스루 전)**

> P1 원칙: "목적지를 모르면 어떤 길도 옳다"

- 이 화면의 **이상적 최종 상태**는 무엇인가? (제약 없는 100점)
- 현재 구현은 궁극의 **몇 %**인가?
- 부족한 부분이 **의도적 제외**인지 **누락**인지 기록
- 이 정의를 기준으로 Step 1~5 워크스루를 진행한다

**Step 1: 진입 (0~3초)**

> "화면이 로드됐다. 사용자의 눈이 가장 먼저 어디로 가는가?"

- 시각적 초점이 명확한가?
- 첫 눈에 보이는 정보가 가치 있는가?

**Step 2: 이해 (3~10초)**

> "화면 내용을 파악하고 있다. 혼란스러운 부분은?"

- 모든 텍스트를 소리 내어 읽었을 때 자연스러운가?
- 숫자/점수가 맥락 없이도 좋은지 나쁜지 알 수 있는가?
- 아이콘만으로 의미 파악이 어려운 곳에 라벨이 있는가?

**Step 3: 판단 (10~30초)**

> "다음 행동을 결정한다. 선택지가 명확한가?"

- CTA가 1~2개로 명확한가?
- "이걸 누르면 뭐가 되지?" 불안감 없이 탭할 수 있는가?

**Step 4: 행동 후**

> "사용자가 행동했다. 기대한 결과가 나왔는가?"

- 탭 → 이동한 화면이 기대와 일치하는가?
- 뒤로가기 했을 때 원래 상태가 보존되는가?

**Step 5: 감정 검증**

> "이 경험 후 사용자의 감정은?"

- P1: "재밌겠다, 더 해보고 싶다" (호기심)
- P2: "점점 나를 알아가고 있다" (성장감)
- P3: "역시 매일 볼 가치가 있다" (만족감)
- P4: "다시 써봐야겠다" (재동기)

### 1.4 플로우 시뮬레이션 (화면 간 전환)

> Jobs T4 (End-to-End): "기능 모음이 아닌 경험의 흐름"

단일 화면이 아닌 **2~3 화면 연속 플로우**를 시뮬레이션한다.
각 전환에서 검증:

- 이전 화면의 맥락이 **자연스럽게 이어지는가?** (맥락 단절 없음)
- 전환 후 사용자가 **"이제 뭐 하지?"** 가 되지 않는가?
- 뒤로가기 시 **원래 상태가 보존**되는가?

#### 핵심 플로우 3가지 (반드시 검증)

**Flow A: 첫 분석 완료**

```
홈(빈 상태) → 분석 선택 → 카메라/입력 → 분석 중(로딩) → 결과
```

- P1 관점: 각 전환에서 "다음엔 뭘 하면 돼?"가 명확한가?
- 결과 화면에서 "또 해보고 싶다" 감정이 드는가?

**Flow B: 교차 인사이트 발견 (Aha Moment)**

```
2번째 분석 결과 → 교차 인사이트 카드 → 관련 모듈 탐색
```

- P2 관점: "피부랑 컬러가 이렇게 연결되네!" 반응이 나오는가?
- 교차 인사이트에서 다음 모듈로의 유도가 자연스러운가?

**Flow C: 일상 복귀**

```
앱 열기 → 홈(Daily Capsule) → 오늘의 인사이트 확인 → 체크
```

- P3 관점: 10초 안에 "오늘 볼 가치가 있었다" 느낌인가?
- P4 관점: 2주 만에 돌아와도 "다시 쓰고 싶다" 느낌인가?

### 1.5 First Launch → Aha 전체 여정

> Jobs T5 (Dent): "사용자 삶에 의미 있는 변화를 만드는가?"

설치부터 Aha Moment까지의 **전체 경험**을 하나의 이야기로 검증한다.

```
설치 → 회원가입 → 온보딩 → 홈(빈 상태)
  → 첫 분석(Flow A) → "나를 알게 됐다!" (첫 Delight)
  → 두 번째 분석(Flow B) → "이렇게 연결되다니!" (Aha Moment)
  → 다음 날 재방문(Flow C) → "매일 올 이유가 있네" (습관 형성)
```

| 구간           | 목표 감정                    | 실패 징후                        |
| -------------- | ---------------------------- | -------------------------------- |
| 설치→온보딩    | "쉽고 빠르다"                | 3단계 이상, 1분 초과             |
| 첫 분석 결과   | "이 앱 나를 아네?" (Delight) | 숫자만 나열, "그래서 뭐?" 반응   |
| 교차 인사이트  | "이렇게 연결되다니!" (Aha)   | 관련성 불명확, "억지스럽다" 반응 |
| 다음 날 재방문 | "오늘도 볼 게 있네"          | 어제와 동일한 화면, 새로움 없음  |

## 2. 발견 기록

워크스루 중 발견한 문제 기록:

```
## 발견: [제목]
- 페르소나: P?
- 단계: Step ?
- 심각도: Critical / High / Medium / Low
- 현재: [사용자가 실제로 경험하는 것]
- 기대: [사용자가 기대하는 것]
- 원인: [기술적/디자인적 원인]
- 제안: [개선 방향]
```

## 3. 비즈니스 가치 검증

- 이 기능이 **리텐션**에 기여하는가? (재방문 이유가 되는가?)
- 추가 분석 완료를 유도하는가? (미완료 → 완료 전환)
- 불필요한 공간 차지 없이 **가치 밀도**가 높은가?

---

## 4. 철학 정렬 심층

### ConnectionAwareness 검증

- 반복 노출 시 설명 깊이가 줄어드는가? (full → brief → minimal → none)
- "확인했어요" 등 사용자 확인 인터랙션이 자연스러운가?

### Aha Moment 기여도

> Aha Moment = 2번째 분석 완료 시 교차 인사이트를 처음 경험하는 순간

- 이 기능이 사용자를 Aha Moment로 이끄는가?
- 경쟁 해자(모듈 수 x 교차 인사이트 품질)에 기여하는가?

---

## 검증 순서

```
Track A (5분):
  P1 탐색자: Step 1 + Step 3
  P3 일상화: Step 1 + Step 3
  → 발견 기록

Track B (30분):
  Step 0: 궁극의 형태 정의
  4 페르소나 × 5단계 워크스루
  → 플로우 시뮬레이션 (1.4 — 화면 간 전환 3가지)
  → First Launch→Aha 여정 (1.5 — 해당 시)
  → 발견 기록
  → 비즈니스 가치 검증
  → 철학 정렬 심층
```
