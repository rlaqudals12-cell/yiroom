## 0. 궁극의 형태 (P1)

```
"완벽한 법적 컴플라이언스 시스템"

- 100% 법률 준수: 개인정보보호법, 전자상거래법, AI기본법 완전 준수
```

## 2. 연령 제한 (만 14세 미만)

### 2.1 법적 근거

**개인정보보호법 제22조의2 (아동의 개인정보 보호)**

> ① 개인정보처리자는 만 14세 미만 아동의 개인정보를 처리하기 위하여 이 법에 따른 동의를 받아야 할 때에는 그 **법정대리인의 동의**를 받아야 하며, 법정대리인이 동의하였는지를 **확인**하여야 한다.

### 2.2 위반 시 제재

| 위반 내용 | 제재 |
|----------|------|
| 법정대리인 동의 없이 아동 개인정보 수집 | **5년 이하 징역** 또는 **5천만원 이하 벌금** |
| 법정대리인 동의 확인 미이행 | **매출액 3% 이하 과징금** |
| 개인정보 파기 의무 위반 | **3천만원 이하 과태료** |

### 2.3 구현 원칙

**권장 방식**: 만 14세 미만 완전 차단 (업계 표준: 올리브영, 무신사, 화해)

```
회원가입 → 생년월일 입력 → 만 14세 미만?
                              ├─ Yes → 가입 차단 + 안내
                              └─ No → 정상 진행
```

### 2.4 필수 문구

**이용약관 조항**:
```
제○조 (이용자 자격 및 연령 제한)
① 본 서비스는 만 14세 이상인 자에 한하여 이용이 가능합니다.
② 「개인정보 보호법」 제22조의2에 따라 만 14세 미만 아동은
   법정대리인의 동의가 필요하므로, 만 14세 미만의 아동은
   본 서비스에 가입할 수 없습니다.
```

**차단 안내 메시지**:
```
죄송합니다. Yiroom은 만 14세 이상만 이용하실 수 있습니다.

개인정보보호법에 따라 만 14세 미만의 경우 법정대리인의 동의가
필요합니다.

[확인]
```

## 3. 민감정보 처리 (얼굴 이미지)

### 3.1 법적 분류

| 처리 목적 | 분류 | 동의 요건 |
|----------|------|----------|
| 일반 사진 | 개인정보 | 표준 동의 (제15조) |
| **특정 개인 식별 목적** | **민감정보** (생체인식정보) | **별도 동의** (제23조) |

**이룸 플랫폼**: 퍼스널 컬러 분석을 위한 얼굴 인식 → **민감정보 처리**

### 3.2 필수 요건

**동의 수집**:
- 다른 개인정보 처리 동의와 **별도 체크박스**
- 민감정보 수집 목적/항목/보유기간 명확 고지

**기술적 보호조치**:
- 저장 및 전송 시 **암호화 필수**
- 원본 이미지와 기타 개인정보 **분리 저장**
- 특징정보 추출 후 원본 **즉시 삭제** 권장

### 3.3 개인정보처리방침 문구

```
제○조 (민감정보의 처리)
① 회사는 이용자의 얼굴 이미지를 수집하여 AI 기반 피부톤 분석에 활용합니다.
② 얼굴 이미지 및 분석 데이터는 「개인정보 보호법」 제23조에 따른
   민감정보에 해당하며, 별도의 동의를 받아 처리합니다.
③ 회사는 얼굴 이미지를 암호화하여 저장 및 전송하며,
   분석 목적 달성 후 지체 없이 파기합니다.
④ 이용자는 민감정보 처리에 대한 동의를 거부할 권리가 있으며,
   동의 거부 시 AI 기반 컬러 분석 서비스 이용이 제한됩니다.
```

## 4. 데이터 보유 및 삭제

### 4.1 보유기간 기준

**PIPA 제21조**: 보유기간 경과 또는 처리 목적 달성 시 **5일 이내** 파기

| 데이터 유형 | 보유기간 | 근거 |
|------------|---------|------|
| 계약/청약철회 기록 | 5년 | 전자상거래법 |
| 대금결제/재화공급 기록 | 5년 | 전자상거래법 |
| 소비자 불만/분쟁처리 | 3년 | 전자상거래법 |
| 접속 로그 (민감정보) | **2년** | 안전성확보조치 기준 |
| 얼굴 이미지 | **분석 완료 즉시~7일** | 최소화 원칙 |

### 4.2 삭제 방법

| 구분 | GDPR | PIPA |
|-----|------|------|
| 삭제 시한 | 최대 1개월 | **5일 이내** |
| 삭제 방법 | 복구 불가능 | **복구 또는 재생 불가능** |
| Soft Delete | 임시 조치만 허용 | **원칙적 불인정** |
| Hard Delete | 권장 | **필수** |

### 4.3 권장 구현

```
1단계: Soft Delete (즉시) → deleted_at 기록, 사용자 접근 차단
2단계: Grace Period (30일) → 복구 가능 기간
3단계: Hard Delete (자동) → pg_cron으로 영구 삭제
4단계: 법정 보존 데이터 → 별도 DB로 분리 저장
```

### 4.4 동의 철회 처리

| 법률 | 처리 기한 |
|------|----------|
| GDPR | 최대 **1개월** |
| PIPA | **10일 이내** |

## 5. 약관 및 정책

### 5.1 이용약관 필수 항목

**전자상거래법 제10조, 제13조**:
- 상호, 대표자 성명, 주소, 연락처
- 사업자등록번호, 통신판매업 신고번호
- 서비스 내용, 가격, 결제 방법
- 청약철회 기한(7일), 방법, 효과
- 환불 조건 및 절차

### 5.2 약관 변경 고지

| 변경 유형 | 최소 고지 기간 | 고지 방법 |
|-----------|---------------|----------|
| 일반 변경 | **7일 전** | 홈페이지 게시 |
| **불리한 변경** | **30일 전** | 개별 통지 필수 (이메일, 푸시 등) |

**위반 시**: 과태료 **최대 500만원**, 해당 약관 주장 불가

### 5.3 무효 면책 조항 (약관규제법 제7조)

| 무효 조항 | 예시 |
|----------|------|
| 고의/중과실 책임 배제 | "어떠한 경우에도 책임지지 않습니다" |
| 손해배상 부당 제한 | "최대 배상 한도 1만원" |
| 위험의 고객 전가 | "모든 위험은 이용자가 부담합니다" |

**유효한 AI 면책 조항**:
```
본 AI 서비스가 제공하는 추천 및 분석 결과는 참고 목적으로만 제공되며,
최종 결정은 이용자의 판단에 따릅니다. 회사는 AI 추천의 정확성을
보증하지 않으나, 서비스 제공에 있어 고의 또는 중대한 과실이 있는
경우에는 관계 법령에 따라 책임을 부담합니다.
```

## 6. AI 투명성 (AI 기본법)

### 6.1 법적 요건 (제31조, 2026.1.22 시행)

**사전 고지 의무**:
- 제품/서비스가 AI 기반으로 작동함을 **사용 전** 알려야 함

**AI 생성물 표시 의무**:
- AI가 생성한 콘텐츠에 인지 가능한 방법으로 표시
- 가시적 워터마크, 텍스트, 또는 비가시적 워터마크 + 텍스트 알림

### 6.2 권장 고지 문구

**서비스 안내**:
```
본 서비스는 인공지능(AI) 기술을 활용하여 맞춤형 퍼스널 컬러 및
뷰티 추천을 제공합니다.
```

**결과물 표시**:
```
이 결과물은 인공지능에 의해 생성되었습니다.
```

### 6.3 위반 시 제재

과태료 **최대 3,000만원** (단, 2026~2027년 계도기간)

## 7. 자동화된 결정 (제37조의2)

### 7.1 정보주체 권리 (2024.3.15 시행)

| 권리 | 설명 |
|------|------|
| **거부권** | 자동화된 결정 적용 거부 |
| **설명요구권** | 자동화된 결정에 대한 설명 요구 |
| **검토요구권** | 인간에 의한 재검토 요구 |

### 7.2 공개 의무

개인정보처리방침에 다음 사항 공개:
- 자동화된 결정의 기준
- 자동화된 결정의 절차
- 개인정보가 처리되는 방식

### 7.3 구현 예시

```
제○조 (자동화된 결정에 대한 권리)
① 회사의 AI 시스템은 이용자의 개인정보를 분석하여
   퍼스널 컬러 진단 및 제품 추천을 제공합니다.
② 이용자는 다음의 권리를 행사할 수 있습니다:
   1. 자동화된 결정의 적용을 거부할 권리
   2. 자동화된 결정에 대해 설명을 요구할 권리
   3. 설명을 받은 후 인간에 의한 재검토를 요구할 권리
③ 자동화된 결정의 기준:
   - 피부톤 분석: 얼굴 이미지에서 추출한 색상 데이터 기반
   - 제품 추천: 분석된 컬러 유형과 이용자 선호도 기반
④ 권리 행사 방법: 개인정보 보호책임자에게 이메일 또는 서면으로 요청
```

## 9. 의료기기법 경계 (AI 피부분석)

### 9.1 법적 배경

**디지털의료제품법** (2024년 제정, 2025년 시행)이 본격 시행되면서 AI 기반 제품의 규제 체계가 명확해졌다.

| 분류 | 정의 | 규제 수준 |
|------|------|----------|
| **SaMD** | Software as a Medical Device | 의료기기 허가 필요 |
| **SiMD** | Software in Medical Device | 의료기기 허가 필요 |
| **디지털 건강지원기기** | 비의료 목적 건강관리 | 자율신고제 |

### 9.2 의료기기 등급

| 등급 | 위험도 | 예시 |
|------|--------|------|
| 1등급 | 최저 | 건강정보 기록/조회 |
| 2등급 | 낮음 | 비침습적 측정 보조 |
| 3등급 | 중간 | 진단 보조 소프트웨어 |
| 4등급 | 높음 | 치료 결정 지원 |

### 9.3 이룸 = 비의료기기 판단

| 판단 기준 | 이룸 서비스 | 의료기기 요건 |
|----------|------------|--------------|
| 목적 | 뷰티/스타일 추천 | 질병 진단/치료 |
| 출력 | 제품/색상 추천 | 진단 결과/치료 방향 |
| 사용자 | 일반 소비자 | 환자/의료진 |
| 결과 활용 | 구매 참고 | 의료 의사결정 |

**결론**: 이룸의 AI 피부분석 서비스는 **의료기기에 해당하지 않음** (디지털 건강지원기기)

### 9.4 금지 표현 (의료기기 오인 방지)

```
❌ 사용 금지:
- "피부 질환 진단"
- "피부암 조기 발견"
- "의료급 분석"
- "피부과 수준 진단"
- "병원용 피부 분석"
- "의사 대체 가능"

✅ 권장 표현:
- "피부 타입 분석"
- "피부 상태 체크"
- "스킨케어 추천을 위한 분석"
- "뷰티 컨설팅 보조"
- "참고용 피부 정보"
```

### 9.5 면책 조항 (필수 표시)

모든 분석 결과 화면에 다음 문구 표시:

```
본 분석 결과는 뷰티 제품 추천을 위한 참고 정보이며,
의학적 진단이나 치료를 대체하지 않습니다.
피부 건강에 대한 우려가 있으시면 전문의와 상담하시기 바랍니다.
```

## 11. 전자상거래법 (제휴 마케팅)

### 11.1 2025년 주요 규제 변경

| 변경 사항 | 내용 |
|----------|------|
| 다크패턴 규제 | 6개 유형 금지 (2025.02.14 시행) |
| 사업자 신원 표시 | 광고 시 필수 표시 강화 |
| 부당 표시·광고 | 사실 과장/왜곡 금지 강화 |

### 11.2 경제적 이해관계 표시 의무

**표시 대상** (모든 경우에 해당):

| 유형 | 예시 | 표시 필요 |
|------|------|----------|
| 금전적 대가 | 현금, 상품권, 적립금 | ✅ |
| 상품 제공 | 무료 제공, 무료 대여 | ✅ |
| 할인 혜택 | 특별 할인 코드 | ✅ |
| 수익 배분 | 어필리에이트 커미션 | ✅ |
| 협업 수익 | 공동구매 수익 | ✅ |

**4대 원칙**:

| 원칙 | 요건 | 예시 |
|------|------|------|
| **접근성** | 쉽게 찾을 수 있는 위치 | 콘텐츠 상단 또는 제목 |
| **인식가능성** | 눈에 띄는 형태 | 명확한 폰트, 색상 |
| **명확성** | 이해하기 쉬운 표현 | "광고", "제휴", "협찬" |
| **언어 동일성** | 콘텐츠와 같은 언어 | 한국어 콘텐츠 → 한국어 표시 |

### 11.3 다크패턴 6가지 금지 (2025.02.14 시행)

| 유형 | 설명 | 이룸 대응 |
|------|------|----------|
| **위장 광고** | 광고를 콘텐츠로 위장 | 제품 추천에 AD 표시 |
| **거짓 희소성** | 가짜 재고 부족 알림 | 사용 금지 |
| **거짓 사회적 증거** | 조작된 리뷰/평점 | 실제 데이터만 표시 |
| **강제 행동** | 구매 강요 팝업 | 사용 금지 |
| **구독 함정** | 해지 어렵게 설계 | 쉬운 해지 제공 |
| **개인정보 착취** | 과도한 정보 수집 | 최소 수집 원칙 |

### 11.4 구현 가이드라인

**제품 추천 카드 UI**:

```
┌─────────────────────────────────────┐
│ [AD] 🏷️                             │  ← 제휴 상품인 경우
├─────────────────────────────────────┤
│ [제품 이미지]                        │
│                                     │
│ 제품명                              │
│ 브랜드                              │
│                                     │
│ 📊 성분 매칭도: 92%                  │  ← 객관적 지표
│ ⭐ 사용자 평점: 4.5                  │  ← 실제 리뷰 기반
│                                     │
│ [상세 보기]    [구매하기]            │
│                                     │
│ 💡 이 추천은 피부 타입 분석 기반입니다 │  ← 추천 근거 명시
└─────────────────────────────────────┘
```

**제품 추천 인터페이스**:

```typescript
interface ProductRecommendation {
  productId: string;
  matchScore: number;
  matchReasons: string[];  // "건성 피부에 적합한 보습 성분"
  isSponsored: boolean;    // 제휴 여부 명시 (필수)
  sponsorLabel?: string;   // "AD" 또는 "제휴"
}
```

**제휴 링크 클릭 시 처리**:

```typescript
function handleAffiliateClick(product: ProductCard) {
```
