# 감정 설계 가이드

> 상위 철학: [yiroom-philosophy.md](./yiroom-philosophy.md) Section 6 (감정 설계)
> "사용자가 이룸을 쓸 때 느끼는 감정을 설계한다"

---

## 1. 핵심 원칙

```
이룸이 만드는 감정 = 호기심 → 발견 → 성장 → 자신감
이룸이 만들지 않는 감정 = 불안, 열등감, 강박, 비교
```

---

## 2. 허용 감정 (4가지)

### 2.1 호기심 (Curiosity)

| 트리거             | UX 패턴       | 예시                                             |
| ------------------ | ------------- | ------------------------------------------------ |
| 미지의 자기 정보   | 분석 시작 CTA | "내 퍼스널컬러가 뭘까?"                          |
| 교차 인사이트 예고 | 연결 힌트     | "피부 분석을 추가하면 더 정확한 추천이 가능해요" |
| 새 모듈 소개       | 탐색 유도     | "헤어 분석도 시작해볼까요?"                      |

**UX 라이팅 톤**: 질문형, 가벼운 초대

```
✅ "내 체형에 어울리는 스타일이 궁금하지 않으세요?"
✅ "피부 분석을 추가하면 어떤 인사이트가 나올까요?"
❌ "체형 분석을 안 하면 정확한 추천이 불가능합니다"
❌ "아직 3개 분석이 미완료입니다"
```

### 2.2 발견의 기쁨 (Discovery)

| 트리거              | UX 패턴          | 예시                                  |
| ------------------- | ---------------- | ------------------------------------- |
| 분석 결과 최초 확인 | 결과 카드        | "당신은 봄 웜톤이에요!"               |
| 교차 인사이트 수신  | 연결 카드        | "봄 웜톤이라서 코랄 립이 잘 어울려요" |
| 새로운 자기 이해    | "몰랐던 나" 표현 | "의외로 이런 스타일도 잘 맞아요"      |

**UX 라이팅 톤**: 긍정적 놀람, 새로운 발견

```
✅ "오, 이 조합이 정말 잘 어울려요!"
✅ "웜톤 + 건성 피부에는 이런 특별한 팁이 있어요"
❌ "분석 결과를 확인하세요" (무미건조)
❌ "결과가 나왔습니다" (사무적)
```

### 2.3 성장 실감 (Growth)

| 트리거              | UX 패턴     | 예시                                           |
| ------------------- | ----------- | ---------------------------------------------- |
| 시간 경과 후 재분석 | 변화 비교   | "3개월 전보다 수분 점수가 올랐어요"            |
| 내재화 진행         | 진행도 표시 | "3개 연결을 내재화했어요"                      |
| 자립 달성           | 마일스톤    | "이제 퍼스널컬러 조합을 직접 판단할 수 있어요" |

**UX 라이팅 톤**: 진심어린 축하, 구체적 변화 언급

```
✅ "수분 점수가 58 → 72로 올랐어요! 관리가 잘 되고 있네요"
✅ "지난달보다 직접 선택을 3번 더 했어요. 판단력이 늘고 있어요!"
❌ "점수가 올랐습니다" (구체성 부족)
❌ "더 노력하세요" (압박)
```

### 2.4 자신감 강화 (Confidence)

| 트리거                | UX 패턴     | 예시                                      |
| --------------------- | ----------- | ----------------------------------------- |
| 자립적 판단 성공      | 확인 피드백 | "직접 고른 컬러가 정확했어요!"            |
| 추천 없이 올바른 선택 | 긍정 확인   | "이 조합, 정말 잘 고르셨어요"             |
| 졸업 마일스톤         | 축하        | "이제 메이크업 컬러는 전문가 수준이에요!" |

**UX 라이팅 톤**: 진정한 인정, 사용자 역량 강조

```
✅ "직접 판단한 이 조합, 전문가 수준이에요!"
✅ "AI 추천 없이도 잘 고르셨네요 — 이해가 깊어졌어요"
❌ "잘했어요!" (어린이 대하듯)
❌ "정답입니다" (시험 분위기)
```

## 3. 금지 감정 (4가지)

### 3.1 불안 (Anxiety)

```
트리거 패턴:
❌ "이 성분이 피부에 해로울 수 있어요" (공포 유발)
❌ "지금 관리하지 않으면..." (시간 압박)
❌ "점수가 떨어졌어요!" (위기감)

대체 패턴:
✅ "이 성분 대신 이런 대안이 있어요" (대안 제시)
✅ "여유 있을 때 해보면 좋아요" (자율성 존중)
✅ "변화가 있네요. 이런 관리를 추가해볼까요?" (긍정적 액션)
```

### 3.2 열등감 (Inferiority)

```
절대 금지:
❌ "평균보다 낮아요"
❌ "대부분의 사용자는..."
❌ "점수가 __점으로 하위 __% 입니다"
❌ 리더보드에서 타인과 점수 비교

대체:
✅ "현재 상태에서 이런 개선이 가능해요"
✅ "지난 기록 대비 이렇게 변화했어요" (자기 비교만)
✅ 절대 점수 표시 (상대 순위 아님)
```

### 3.3 강박 (Obsession)

```
절대 금지:
❌ "오늘의 분석을 놓쳤어요!"
❌ "연속 기록이 끊어졌어요" (스트릭 압박)
❌ "매일 분석해야 정확해요"
❌ 빈번한 알림 (하루 2회 초과)

대체:
✅ "다시 와주셔서 반가워요" (무조건 환영)
✅ "기록이 없는 날도 괜찮아요" (명시적 허용)
✅ 알림은 가치 있는 인사이트가 있을 때만
```

### 3.4 비교 (Comparison)

```
절대 금지:
❌ "친구 __님은 __점이에요"
❌ "같은 나이대 평균은..."
❌ "이상적인 비율은..."
❌ 소셜 피드에서 분석 점수 공개

대체:
✅ 오직 자기 자신의 과거 데이터와 비교
✅ 점수가 아닌 변화 방향에 집중
✅ "당신만의 특징"으로 프레이밍
```

## 4. 신체 이미지 보호 (Body Image Protection)

### 4.1 4대 원칙

| #   | 원칙                      | 설명                                         |
| --- | ------------------------- | -------------------------------------------- |
| 1   | **결핍 프레이밍 금지**    | "부족하다", "안 좋다" 표현 사용 안 함        |
| 2   | **타인 비교 금지**        | "평균", "이상적", "표준" 체형 언급 안 함     |
| 3   | **부정적 신체 언어 금지** | "문제 부위", "약점", "결점" 표현 사용 안 함  |
| 4   | **긍정적 행동 마무리**    | 모든 분석 결과 후 실행 가능한 긍정 액션 제시 |

### 4.2 리프레이밍 예시

| 원래 표현                | 리프레이밍                                                 |
| ------------------------ | ---------------------------------------------------------- |
| "엉덩이가 좁은 편이에요" | "직선적인 실루엣이 특징이에요. A라인 스커트가 잘 어울려요" |
| "피부 상태가 안 좋아요"  | "수분 보충에 집중하면 더 좋아질 수 있어요"                 |
| "얼굴이 넓어 보여요"     | "이목구비가 또렷하게 보이는 얼굴형이에요"                  |
| "다리가 짧아요"          | "하이웨이스트로 비율을 더 살릴 수 있어요"                  |
| "이마가 넓어요"          | "앞머리 스타일링으로 분위기를 다양하게 연출할 수 있어요"   |
| "잇몸이 안 좋아요"       | "잇몸 건강을 개선할 수 있는 방법이 있어요"                 |

### 4.3 점수 표시 원칙

```
점수는 "상태 파악"에만 사용하고 "판정"에 사용하지 않는다.

✅ 허용:
- "수분 점수 58/100 — 보습에 좀 더 신경 쓰면 좋아요"
- "지난번 52에서 58로 올랐어요!"

❌ 금지:
- "수분 점수 58/100 — 낮은 편이에요"
- "목표 80에 한참 못 미쳐요"
- "F등급입니다"
```

## 5. UX 라이팅 가이드라인

### 5.1 톤 앤 보이스

```
이룸의 톤 = "아는 친구가 조언해주는 느낌"

특성:
- 해요체 (합니다체 아님)
- 전문적이지만 쉬운 용어
- 단정이 아닌 제안
- 지시가 아닌 초대

예:
✅ "이런 스타일은 어때요?"
❌ "이 스타일을 추천합니다."
✅ "코랄 계열이 잘 어울릴 것 같아요"
❌ "코랄 계열을 사용하십시오"
```

### 5.2 상황별 톤

| 상황          | 톤                 | 예시                                              |
| ------------- | ------------------ | ------------------------------------------------- |
| 첫 만남       | 반갑고 호기심 유발 | "나에 대해 알아볼 준비 됐나요?"                   |
| 분석 결과     | 긍정적 발견        | "당신은 봄 웜톤이에요! 화사한 컬러가 잘 어울려요" |
| 교차 인사이트 | 연결의 기쁨        | "봄 웜톤이라서 이런 립 컬러가 특히 빛나요"        |
| 성장 확인     | 진심어린 인정      | "지난달보다 판단력이 많이 늘었어요!"              |
| 에러 발생     | 미안하고 해결 지향 | "잠깐 문제가 생겼어요. 다시 해볼까요?"            |
| 빈 상태       | 가볍게 초대        | "아직 기록이 없어요. 첫 분석을 시작해볼까요?"     |

### 5.3 금지 표현 사전

| 금지 표현       | 대체 표현                    |
| --------------- | ---------------------------- |
| ~해야 합니다    | ~해보면 좋아요               |
| 반드시          | (삭제 또는 "특히")           |
| 위험합니다      | 주의가 필요해요              |
| 부족합니다      | 더 좋아질 수 있어요          |
| 문제가 있습니다 | 개선할 수 있는 부분이 있어요 |
| 하위/상위       | (삭제 — 순위 표현 금지)      |
| 평균 이하       | 현재 상태에서                |
| 실패            | 다시 시도해볼까요            |
| ~하지 마세요    | ~대신 이건 어때요            |

## 6. 감정 검증 체크리스트

모든 UI 텍스트에 적용:

```
[ ] 이 텍스트가 유발하는 감정은? (호기심/발견/성장/자신감 중 하나인가?)
[ ] 불안, 열등감, 강박, 비교를 유발하지 않는가?
[ ] 신체 이미지 보호 4대 원칙을 준수하는가?
[ ] "~해야" 대신 "~해보면"을 사용하는가?
[ ] 타인 비교 데이터가 포함되지 않았는가?
[ ] 부정적 표현 대신 긍정적 행동으로 마무리하는가?
[ ] 해요체를 사용하고 있는가?
[ ] 전문 용어 없이 이해 가능한가?
```
