# 의사결정 과학 원리 (Decision Science Principles)

> 이 문서는 이룸 캡슐 에코시스템의 UX 설계 기반이 되는 의사결정 과학 원리를 설명합니다.
> 사용자가 더 적은 노력으로, 더 좋은 선택을 하도록 돕는 과학적 근거를 제공합니다.

---

## 0. 궁극의 형태 (P1)

### 이상적 최종 상태

```
"사용자가 의식하지 못하는 사이에 최적의 결정을 내리는 시스템"

- 의사결정 피로 0: 모든 선택에 인지적 부담 없음
- 최적 기본값: 아무것도 안 해도 80점짜리 결정
- 자율성 보장: 원하면 커스터마이징 가능하되 강제하지 않음
- 후회 최소화: 선택 후 만족도 4.5/5.0 이상
```

### 물리적 한계

| 한계               | 설명                                  |
| ------------------ | ------------------------------------- |
| **인간 인지 한계** | Working memory 7±2 (Miller's Law)     |
| **주관적 선호**    | 객관적 최적 ≠ 사용자 만족             |
| **맥락 의존성**    | 동일 선택지도 맥락에 따라 최적이 다름 |
| **시간 압박**      | 아침 준비 시간 = 제한적 의사결정 자원 |

### 100점 기준

| 지표              | 100점 기준                  |
| ----------------- | --------------------------- |
| **의사결정 시간** | 캡슐 미사용 대비 50% 감소   |
| **선택 만족도**   | 4.5/5.0 이상                |
| **기본값 수용률** | 60% 이상 (좋은 기본값 증거) |
| **선택 후회율**   | 5% 미만                     |
| **재방문율**      | 선택 기능 7일 리텐션 70%+   |

### 현재 목표

**20%** — 의사결정 과학 미적용 상태

- ❌ 선택지 과잉 (모듈별 추천 목록 무제한)
- ❌ 기본값 부재 (모든 것을 사용자가 선택)
- ❌ Progressive Disclosure 미적용
- ❌ 의사결정 피로 측정 없음
- ⏳ 캡슐 시스템으로 C1-C5 적용 예정

### 의도적 제외

| 제외 항목       | 이유                        | 재검토 시점  |
| --------------- | --------------------------- | ------------ |
| 다크 패턴       | 윤리적 문제, 이룸 철학 위배 | 영구 제외    |
| 무의식 넛지     | 사용자 자율성 침해 우려     | 영구 제외    |
| A/B 테스트 과잉 | 사용자 일관성 저해          | Phase 2 이후 |

## 1. 핵심 이론

### 1.1 의사결정 피로 (Decision Fatigue)

**이론**: Baumeister et al. (1998) — 의지력(Willpower)은 유한한 자원

```
의사결정 피로 모델:

  결정 품질
  ↑
  │████████
  │        ████████
  │                ████████
  │                        ████████
  │                                ████████ ← 질 저하
  └─────────────────────────────────────────→ 결정 횟수

  아침: 의지력 최대 → 좋은 결정
  저녁: 의지력 고갈 → 나쁜 결정 또는 결정 회피
```

**학술 근거**:

- Danziger et al. (2011): 이스라엘 판사 연구 — 아침 가석방 승인 65%, 점심 전 0%
- Vohs et al. (2014): 선택 과부하 후 자제력 저하

**이룸 적용**:

- 사용자는 하루에 평균 35,000개 결정을 내림 (Research by Cornell)
- 뷰티/웰니스 결정: 스킨케어 제품 선택, 오늘 입을 옷, 운동 종류, 식단
- 캡슐 시스템: 35,000개 중 뷰티/웰니스 결정을 **자동화**하여 인지 자원 보존

### 1.2 선택의 역설 (Paradox of Choice)

**이론**: Barry Schwartz (2004) — 선택지가 많을수록 만족도가 낮아짐

```
만족도
↑
│      ●●●●
│    ●●    ●●
│   ●        ●●
│  ●           ●●●
│ ●                ●●●●●●  ← 선택지 과잉 → 만족도 하락
│●
└────────────────────────→ 선택지 수

  최적점: 3-7개 선택지 (Miller's Law와 일치)
```

**학술 근거**:

- Iyengar & Lepper (2000): 잼 실험 — 24종류 시식대 구매율 3%, 6종류 시식대 구매율 30%
- Scheibehenne et al. (2010): 메타분석 — 선택 과부하 효과 확인 (특히 전문성 낮을 때)

**이룸 적용**:
| 모듈 | 전체 선택지 | 캡슐 후 선택지 | 감소율 |
|------|-----------|-------------|--------|
| S-1 스킨케어 | 50+ 제품 | 5개 (5-Step) | 90% |
| C-1 의류 | 100+ 벌 | 33개 (캡슐) | 67% |
| W-1 운동 | 100+ 종류 | 7개 (Mag-7) | 93% |
| N-1 식재료 | 무한 | 7개 (주간) | 99%+ |

### 1.3 기본값 효과 (Default Effect)

**이론**: Johnson & Goldstein (2003) — 기본값이 선택을 지배함

```
장기 기증 동의율:

  Opt-in 국가 (기본값 = 비동의):
  ├── 독일: 12%
  ├── 덴마크: 4.25%
  └── 네덜란드: 27.5%

  Opt-out 국가 (기본값 = 동의):
  ├── 오스트리아: 99.98%
  ├── 프랑스: 99.91%
  └── 벨기에: 98%
```

**이룸 적용**:

- **스마트 기본값**: AI 분석 결과 기반으로 최적 캡슐 자동 구성
- **Opt-out 설계**: 기본 캡슐 제공 → 사용자가 원하면 수정
- **점진적 노출**: 처음엔 기본값, 숙련되면 커스터마이징 옵션 노출

```
기본값 설계 원칙:
1. 80% 사용자에게 적합한 기본값 선택
2. 변경 비용을 0으로 (원클릭 수정)
3. 기본값 선택 이유를 설명 가능하게
4. 안전 관련은 반드시 Opt-in (SafetyProfile)
```

### 1.4 만족화 vs 극대화 (Satisficing vs Maximizing)

**이론**: Herbert Simon (1956) — 합리적 인간은 "충분히 좋은" 선택을 함

```
Maximizer (극대화자):
- 모든 옵션 비교 → 최적 선택 추구
- 결과: 객관적으로 더 좋은 선택, 주관적으로 덜 만족
- 더 많은 후회, 우울 경향

Satisficer (만족화자):
- "충분히 좋은" 옵션 발견 → 선택 종료
- 결과: 객관적으로 비슷한 선택, 주관적으로 더 만족
- 적은 후회, 높은 행복감
```

**학술 근거**:

- Schwartz et al. (2002): Maximization Scale — 극대화자는 우울, 후회, 사회 비교 높음

**이룸 적용**:

- 캡슐 = 만족화 지원 시스템
- "이 5개면 충분해요" → Satisficing 유도
- 비교 정보 최소화 (전체 순위 대신 "당신에게 맞는 Top 3")

## 2. Hick's Law — 반응 시간과 선택지 수

### 2.1 공식

```
T = a + b × log₂(n + 1)

T: 반응 시간 (ms)
a: 기본 반응 시간 (상수, ~200ms)
b: 정보 처리 시간 (상수, ~150ms/bit)
n: 등확률 선택지 수

예시:
  n = 2: T = 200 + 150 × log₂(3) = 200 + 238 = 438ms
  n = 7: T = 200 + 150 × log₂(8) = 200 + 450 = 650ms
  n = 30: T = 200 + 150 × log₂(31) = 200 + 743 = 943ms
  n = 100: T = 200 + 150 × log₂(101) = 200 + 999 = 1199ms
```

### 2.2 이룸 적용 — 선택지 최적화

```typescript
/**
 * 모듈별 최적 선택지 수 (Hick's Law 기반)
 *
 * 목표 반응 시간: 700ms 이하 (3-7개 선택지)
 * Miller's Law (7±2)와 교차 검증
 */
const OPTIMAL_CHOICE_COUNT: Record<string, number> = {
  // 일상 결정 (빠른 결정 필요): 3-5개
  dailyOutfit: 3, // 오늘의 코디 추천
  dailySkincare: 5, // 5-Step 고정
  dailyWorkout: 3, // 오늘의 운동 추천
  dailyMeal: 3, // 오늘의 식단 추천

  // 구성 결정 (숙고 허용): 5-7개
  capsuleItems: 7, // 캡슐 구성 아이템
  productRecommend: 5, // 제품 추천 목록
  workoutPlan: 7, // 주간 운동 계획

  // 탐색 결정 (브라우징): 7-12개
  productBrowse: 12, // 상품 탐색 목록
  ingredientList: 10, // 성분 분석 결과
};
```

## 3. Progressive Disclosure (점진적 노출)

### 3.1 원리

```
정보 노출 수준:

Level 1 (즉시): 핵심 결과 + 추천 액션
  └── "오늘의 캡슐: 3개 루틴 준비됨 ✓"

Level 2 (탭 시): 상세 정보
  └── "스킨케어 5-Step / 운동 3세트 / 식단 3끼"

Level 3 (스크롤): 근거 및 대안
  └── "이 제품을 추천한 이유: 피부타입 매칭 92%"

Level 4 (설정): 고급 커스터마이징
  └── "캡슐 구성 직접 수정 / 선호 조건 변경"
```

### 3.2 방문 횟수 기반 노출

```
초보자 (1-7회 방문):
  → 기본값 강조, 선택지 최소화
  → "추천 루틴으로 시작하기" 단일 CTA

중급자 (8-30회 방문):
  → 커스터마이징 옵션 점진적 노출
  → "루틴 수정하기" 보조 CTA 추가

숙련자 (30회+ 방문):
  → 전체 옵션 노출
  → 고급 설정, 수동 캡슐 구성 허용
```

### 3.3 인지 부하 예산 (Cognitive Load Budget)

```
한 화면의 인지 부하 예산: 7 단위 (Miller's Law)

나쁜 예 (인지 부하 초과):
┌─────────────────────────┐
│ 피부분석 결과 (4항목)    │  4
│ 추천 제품 (6개)          │  6
│ 주의사항 (3개)           │  3
│ 관련 분석 (2개)          │  2
│ 총 인지 부하: 15 단위    │  ← 초과!
└─────────────────────────┘

좋은 예 (인지 부하 준수):
┌─────────────────────────┐
│ 핵심 결과 (1개 문장)     │  1
│ 오늘의 캡슐 (3개)       │  3
│ 시작 버튼 (1개)          │  1
│ 총 인지 부하: 5 단위     │  ← 안전
│ [더 알아보기 →]          │  ← Level 2로 이동
└─────────────────────────┘
```

## 4. 넛지 이론과 윤리적 설계

### 4.1 Thaler & Sunstein (2008) — 넛지

```
넛지 = 선택의 자유를 제한하지 않으면서
       예측 가능한 방향으로 행동을 유도

이룸의 넛지 원칙:
1. 투명성: 왜 이것을 추천하는지 항상 설명
2. 자율성: 무시/변경 가능, 강제하지 않음
3. 이익 일치: 사용자 이익 = 시스템 목표
4. 다크 패턴 금지: 조급함/공포 유발 금지
```

### 4.2 허용되는 넛지 vs 금지되는 넛지

| 유형            | 허용                                | 금지                                |
| --------------- | ----------------------------------- | ----------------------------------- |
| **기본값**      | 최적 캡슐 자동 구성                 | "이 제품 안 사면 손해"              |
| **프레이밍**    | "피부에 95% 적합" (긍정)            | "이걸 안 쓰면 피부 나빠져요" (공포) |
| **사회적 증거** | "비슷한 피부타입 사용자 78%가 선택" | "모든 사람이 이걸 사요" (과장)      |
| **적시 정보**   | 계절 전환 알림                      | 매일 반복 구매 알림 (스팸)          |
| **단순화**      | 5-Step으로 줄이기                   | 쉬운 것만 보여주고 위험 숨기기      |

### 4.3 어필리에이트와 넛지의 경계

```
핵심 갈등: 캡슐 미니멀리즘 vs 커머스 매출

윤리적 해결:
1. 추천은 "교체/보충"으로만 (캡슐 총량 불변)
2. 추천 이유 투명 공개 ("매칭 92%" vs "광고")
3. 어필리에이트 수수료가 추천 순위에 영향 주지 않음
4. "추천 제품 없이도 캡슐 완성 가능" 항상 표시
```

## 5. 후회 최소화 프레임워크 (Regret Minimization)

### 5.1 Zeelenberg & Pieters (2007) — 후회 이론

```
후회 유형:

1. 행동 후회 (Action Regret): "이걸 사지 말걸"
   → 대응: 교환/반품 안내, "어울리지 않으면 캡슐에서 빼세요"

2. 비행동 후회 (Inaction Regret): "이걸 살걸"
   → 대응: 위시리스트 저장, "다음 로테이션 때 추가할 수 있어요"

3. 비교 후회 (Comparative Regret): "저게 더 나았을 텐데"
   → 대응: 비교 정보 최소화, Top 3만 노출
```

### 5.2 이룸 후회 최소화 전략

```
1. 되돌리기 쉽게 (Reversibility)
   - 캡슐 아이템 변경: 원클릭
   - 분석 재실행: 언제든 가능
   - 루틴 리셋: "기본값으로 돌아가기"

2. 작은 실험 유도 (Small Experiments)
   - "이번 주만 이 제품 써보기" → 전면 교체가 아닌 테스트
   - 운동: "오늘 하나만 바꿔보기"

3. 사회적 검증 (Social Proof)
   - "비슷한 프로필 사용자 만족도 4.2/5.0"
   - 단, 과장/강제 금지
```

## 7. 검증 방법

### 7.1 측정 지표

| 지표          | 측정 방법                      | 목표                  |
| ------------- | ------------------------------ | --------------------- |
| 의사결정 시간 | 추천→선택 소요 시간            | 캡슐 미사용 대비 -50% |
| 선택 만족도   | 선택 직후 + 7일 후 만족도 조사 | 4.0/5.0+              |
| 기본값 수용률 | 기본값 변경 없이 사용한 비율   | 60%+                  |
| 후회율        | "다시 선택한다면?" 질문        | < 10%                 |
| 결정 회피율   | 추천 화면에서 이탈한 비율      | < 20%                 |

### 7.2 A/B 테스트 설계

```
실험 1: 선택지 수 최적화
  Control: 10개 제품 추천
  Treatment: 3개 제품 추천
  측정: 구매율, 만족도, 이탈률

실험 2: 기본값 효과
  Control: 빈 캡슐 (사용자 직접 구성)
  Treatment: AI 기본값 캡슐 (수정 가능)
  측정: 완성률, 만족도, 사용 지속률

실험 3: Progressive Disclosure
  Control: 전체 정보 한 화면
  Treatment: 3단계 점진적 노출
  측정: 정보 소비량, 행동 전환율
```
