# 퍼스널 대비 (Personal Contrast) 원리

> 퍼스널컬러(시즌)와 직교하는 두 번째 조화 축 — "어떤 색"이 아니라 "얼마나 세게".
> 리서치: [2026-07-personal-contrast.md](../research/claude-ai-research/2026-07-personal-contrast.md)

## 1. 핵심 개념

### 1.1 시즌이 답하지 못하는 질문

12톤 퍼스널컬러는 색상(hue)·언더톤·채도·명도 계열("어떤 색이 어울리나")을 판정한다. 그러나 "헤어와 옷이 각각 어울려도 조합이 안 어울리는" 현상의 상당수는 **명도 대비(value contrast) 불일치**다: 착장의 명암 대비가 얼굴의 자연 대비와 어긋날 때 옷이 얼굴을 이기거나 얼굴이 묻힌다.

### 1.2 퍼스널 대비의 정의

**피부·모발(·눈) 사이의 명도(L\*) 격차.** 흑발+밝은 피부 = 고대비, 갈발+중간 피부 = 중대비. 원칙: **착장·헤어·메이크업의 대비 강도가 자연 대비를 반향(echo)할 때 조화롭다** (David Zyla, Inside Out Style 등 컨설팅 체계 공통).

- 고대비인 사람: 명확한 배색·강한 패턴·진한 립이 어울림. 톤온톤 파스텔은 얼굴이 묻힘.
- 저대비인 사람: 톤온톤·부드러운 그라데이션. 흑백 강대비는 옷이 얼굴을 이김.
- **어두운 피부 ≠ 자동 고대비** — 대비는 절대 명도가 아니라 피부-모발 간 상대 격차. (12톤 deep 판정과 별개 축)

## 2. 측정

```
primaryContrast = |L*_hair − L*_skin|      (CIE Lab 명도)
판정: low < 25 ≤ medium < 45 ≤ high       (0-10 value scale 3/6 경계의 L* 환산)
```

- 피부 L\*: 기존 `zone-sampler`의 가중 평균 Lab(이미 실측 중) 재사용.
- 모발 L\*: 얼굴 상단 외곽 영역에서 비피부·저채도 픽셀 샘플링(피부 샘플러 패턴 복제, 필터만 변경).
- 눈(홍채) L\*: Face Mesh iris refine 필요 — v2로 보류. 모발-피부 대비만으로 실용적.
- **한국인 캘리브레이션**: 흑발 지배 분포라 절대 임계 대신 분포 백분위 재보정 여지 — v1은 고정 임계 + 데이터 축적 후 조정(임계는 상수로 격리).

### 2.1 신뢰도 조건

염색·저조도·모자 착용 시 모발 L\* 왜곡 → 샘플 픽셀 수·분산이 기준 미달이면 **contrast 미판정**(추정 금지, 정직성 원칙). 기존 `contrastLevel` 스키마 필드는 존재하나 값이 추측(Mock 하드코딩/VLM 출력)이었다 — 실측으로 대체하고, 실측 불가 시 필드 생략.

## 3. 적용 규칙 (대비 → 표현 강도)

| 대비   | 배색                                       | 패턴           | 립/메이크업      | 헤어 컬러                |
| ------ | ------------------------------------------ | -------------- | ---------------- | ------------------------ |
| high   | 명확한 명암 배색(밝은 상의+어두운 하의 등) | 뚜렷한 패턴 OK | 진한 립 잘 받음  | 뿌리-끝 대비 스타일 가능 |
| medium | 중간 대비, 유연                            | 중간 스케일    | 중간 발색        | 자연스러운 변화          |
| low    | 톤온톤·인접 명도                           | 잔잔한 패턴    | MLBB·은은한 발색 | 피부와 가까운 명도 유지  |

시즌(색상)과 대비(강도)는 곱해진다: "여름 쿨 뮤트 × 저대비" = 뮤트 팔레트를 톤온톤으로.

## 4. 구현 도출

1. `lib/analysis/personal-color-v2/`에 모발 샘플러 + `computePersonalContrast()` (순수 함수, 임계 상수).
2. PC 분석 파이프라인에서 실측값으로 `contrastLevel` 저장(스키마 기존재 — 마이그레이션 불필요). Mock/VLM 추측 경로 제거.
3. 소비: PC 결과 페이지 1줄("대비가 낮은 편이라 톤온톤이 잘 어울려요" + 근거), 브리핑 배색 카드(daily-outfit)의 명도 격차 조절, 코디/메이크업 규칙에 intensity 파라미터.

## 5. 검증

- 동일 이미지 → 동일 대비(결정론). 합성 테스트 픽셀(흑발+밝은 피부 → high 등).
- 신뢰도 미달 → 미판정(생략) 테스트.
- 사용자 문구에 "대비" 풀이 병기(초보자: "얼굴의 밝고 어두움 차이").
