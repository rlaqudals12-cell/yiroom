## 0. 궁극의 형태 (P1)

### 이상적 최종 상태

```
"내 체형을 있는 그대로 이해하게 해주는 3D 거울"

- 프로필 체형 축을 열면 회전 가능한 나의 체형 아바타가 보인다
- 재분석 시 이전 아바타와 오버레이 비교 — 숫자가 아니라 변화가 보인다
- 체형 타입(S/W/N)과 비율이 실루엣으로 직관 전달된다
- 아바타 위에 스타일링 조언이 시각적으로 겹쳐진다 (어깨 강조, 허리 라인 등)
- 옷장 아이템의 가상 착장(long-term)
```

### 물리적/논리적 한계

| 항목               | 한계                                                                        |
| ------------------ | --------------------------------------------------------------------------- |
| 단안 사진 스케일   | 절대 치수(cm) 복원 불가 — **비율만 신뢰 가능** (스케일 모호성)              |
| 입력 데이터 밀도   | 현행 저장 데이터 = body_type(S/W/N) + 어깨-허리 비율 1개 (프로드 실측 확인) |
| 사실적 재현        | 사실적(photoreal) 아바타는 "나랑 다른데?" 기대-실망 갭 유발 (ZOZOSUIT 교훈) |
| SMPL 에셋 라이선스 | SMPL 모델 파일은 비상업 연구 전용 — 상용 사용은 Meshcapade 유료 라이선스    |
| 모바일 렌더링      | WebGL/expo-gl 성능·번들 제약 — 저폴리곤 필수                                |

### 100점 기준

- 체형 타입 5종(측정) + S/W/N 3종(골격)이 실루엣으로 구분 가능
- 동일 입력 → 동일 아바타 (결정론, 재현성 100%)
- 비율 파라미터 변화가 시각적으로 단조(monotonic) 반영
- 초기 로드 추가 비용 0 (dynamic import, 결과/프로필 페이지에서만 로드)
- 재분석 오버레이 비교 시 변화 방향이 직관적으로 보임

### 현재 목표: 60% (v1)

- 웹: 회전 가능한 저폴리곤 스타일라이즈드 아바타 (Three.js)
- 입력: body_type(S/W/N) 프리셋 + ratio 있으면 어깨/허리 모프 적용
- 표면: 체형 결과 페이지 + 프로필 체형 축 카드

### 의도적 제외 (v1)

| 제외 항목            | 이유                                         | 재검토 시점              |
| -------------------- | -------------------------------------------- | ------------------------ |
| 치수(cm) 표시        | 스케일 모호성 + ZOZOSUIT 교훈 (기대-실망 갭) | 재검토 안 함 (원칙)      |
| 사실적(photoreal)    | 기대-실망 갭, 에셋 라이선스, 용량            | 사용자 요구 실증 시      |
| SMPL 런타임 탑재     | 상용 라이선스 필요 + v1 요구에 과잉          | 이미지→체형 피팅 도입 시 |
| 가상 착장(옷 입히기) | 의류 시뮬레이션은 별도 대형 과제             | Phase 5 이후             |
| 모바일 3D 패리티     | expo-gl 검증 필요                            | 웹 v1 검증 후            |

## 1. 핵심 개념

### 1.1 파라메트릭 바디 모델 (Parametric Body Model)

인체 형상을 소수의 연속 파라미터로 표현하는 모델. 대표가 **SMPL** (Skinned Multi-Person Linear Model, SIGGRAPH Asia 2015):

```
정점 위치 = 기본 메시(template) + Σ βᵢ · 형상기저(shape basis)ᵢ + 포즈 보정
```

- **β (shape parameters)**: 10~300차원 벡터. 각 성분이 "키 방향", "체격", "허리 굵기" 같은 주성분(PCA) 축에 대응
- **핵심 통찰**: 체형의 다양성은 고차원처럼 보이지만, 실제로는 **소수의 선형 축의 조합**으로 대부분 설명된다 (PCA 상위 2~10개 성분이 분산의 대부분)

이룸이 차용하는 것은 이 **"형상 = 기본형 + 축별 선형 변형의 합"이라는 원리**이며, SMPL의 에셋(메시·기저 데이터)이 아니다.

### 1.2 모프 타겟 (Morph Target / Blend Shape)

파라메트릭 원리의 실용 구현. 그래픽스 표준 기법:

```
V(w) = V_base + Σ wᵢ · (V_targetᵢ − V_base),  wᵢ ∈ [0, 1]
```

- `V_base`: 중립 체형 메시의 정점 배열
- `V_targetᵢ`: 극단 체형(예: 어깨 최대) 메시
- `wᵢ`: 모프 가중치 — **분석 데이터에서 도출**

Three.js가 `morphTargetInfluences`로 네이티브 지원. 정점 수만 동일하면 절차적 생성 메시에도 적용 가능.

### 1.3 스케일 모호성과 비율 파라미터화

단안 사진에서 절대 크기는 복원 불가(같은 사진 = 큰 사람이 멀리 있는 것과 작은 사람이 가까이 있는 것 구분 불가). 따라서:

```
아바타 입력은 무차원 비율만 사용한다.
  shoulderToWaistRatio = 어깨너비 / 허리너비   (스케일 소거)
  waistToHipRatio      = 허리너비 / 힙너비     (스케일 소거)
```

이는 현행 체형 분석이 비율 기반 분류(ASTM D5585 계열)를 쓰는 이유와 동일하며, 아바타도 같은 데이터 위에 선다. **키·몸무게가 없어도 아바타는 성립한다** (전체 스케일은 화면 정규화로 고정).

### 1.4 지각적 정직성 (Perceptual Honesty) — 스타일라이즈드 우선

ZOZOSUIT 실패의 원인은 정밀도 부족이 아니라 **기대-실망 갭**이었다. 시각화 스타일이 곧 기대 수준을 설정한다:

| 스타일                | 사용자 기대                 | 갭 리스크 |
| --------------------- | --------------------------- | --------- |
| 사실적 (photoreal)    | "이게 나여야 한다"          | 높음      |
| 스타일라이즈드 (중립) | "내 체형의 특징을 보여준다" | 낮음      |

→ **아바타는 의도적으로 중립·저폴리곤 스타일라이즈드로 만든다.** 얼굴 디테일 없음, 피부색 중립(브랜드 톤), 실루엣과 비율만 정확. 이는 기존 "익명 공유 실루엣" 시각 언어와도 정합.

### 1.5 변화 시각화 (오버레이 비교)

리서치 결론 "숫자보다 변화를 보여줘라"의 구현 원리:

```
이전 아바타(반투명 고스트) + 현재 아바타(불투명) 동일 카메라·스케일로 중첩
→ 비율 변화가 실루엣 차이로 지각됨
```

두 아바타가 같은 파라미터 공간에서 생성되므로(결정론) 오버레이는 항상 정렬된다. 이는 피부 축의 `computeSkinTrend` "직전 대비" 패턴의 체형판 확장.

## 2. 수학적 기반

### 2.1 분석 데이터 → 모프 가중치 매핑

입력(프로드 실측 기준 가용 데이터):

| 입력                       | 출처                                  | 가용성         |
| -------------------------- | ------------------------------------- | -------------- |
| `body_type` ∈ {S, W, N}    | body_analyses.body_type               | 전 행 (필수)   |
| `ratio` (어깨/허리)        | body_analyses.ratio                   | 통합 경로 행만 |
| `shoulder/waist/hip` 0-100 | body_analyses (V1 경로 행)            | 구형 행만      |
| BodyRatios 전체            | body-v2 클라이언트 측정 (현재 미저장) | 향후 저장 시   |

매핑 (2단계):

**① 타입 프리셋** — S/W/N별 기본 모프 가중치 벡터:

```
w_preset(S) = { shoulder: 0.65, waist: 0.45, hip: 0.50, torso: 0.55 }  // 스트레이트: 상체 존재감, 직선
w_preset(W) = { shoulder: 0.40, waist: 0.35, hip: 0.60, torso: 0.45 }  // 웨이브: 하체 중심, 곡선
w_preset(N) = { shoulder: 0.55, waist: 0.50, hip: 0.52, torso: 0.60 }  // 내추럴: 골격감, 프레임
```

(값은 SDD에서 확정 — 시각 검증으로 튜닝)

**② 비율 보정** — 실측 비율이 있으면 프리셋 위에 덮어씀:

```
w_shoulder = clamp01( (shoulderToWaistRatio − R_MIN) / (R_MAX − R_MIN) )
  R_MIN = 0.95 (허리≈어깨), R_MAX = 1.45 (어깨 뚜렷이 넓음)
```

경계값은 body-v2 `BODY_SHAPE_THRESHOLDS` (모래시계 0.75, 역삼각형 1.1)와 정합하게 설정.

### 2.2 결정론 요구

```
avatar = f(body_type, ratio?, ratios?)   — 순수 함수
```

- 동일 분석 행 → 항상 동일 아바타 (Math.random 금지, 시간 의존 금지)
- 재현성은 이룸 신뢰 전략의 1순위 조건(market-position 리서치) — 아바타도 예외 없음
- AI 호출 없음: 아바타는 **저장된 분석 결과의 파생 뷰**이며 새 추론이 아니다 (Hybrid 데이터 패턴의 "표시 데이터" 계층)

### 2.3 절차적 메시 생성 (단면 로프팅)

인체 몸통은 높이별 타원 단면의 연속으로 근사 가능:

```
단면(h) = 타원( a(h)·widthScale(h), b(h) )
  a(h): 높이 h에서의 좌우 반지름 — 어깨/허리/힙 제어점을 스플라인 보간
  b(h): 전후 반지름 — 좌우의 0.6~0.75배 (인체 단면 비)
```

- 제어점: 어깨(h=0.82), 가슴(0.72), 허리(0.58), 힙(0.48), 허벅지(0.38) 등
- 모프 가중치가 제어점 반지름을 스케일 → 스플라인이 부드러운 실루엣 생성
- 팔다리는 캡슐(capsule) 프리미티브 — 실루엣 인식에 충분, 정점 수 최소
- 결과: 에셋 파일 0KB, 전부 코드에서 생성, 정점 수 ~2-5k (모바일 안전권)

## 3. 구현 도출

### 3.1 원리 → 아키텍처

```
body_analyses 행 (body_type, ratio)
        │  (순수 함수, 서버/클라 어디서든)
        ▼
avatar-params (모프 가중치 벡터 w)      ← lib/avatar/ (도메인 로직, 테스트 대상)
        │
        ▼
절차적 메시 생성 + Three.js 렌더        ← components/avatar/ (Client Component, dynamic import)
        │
        ▼
체형 결과 페이지 / 프로필 체형 축 카드   ← 표면 (ADR-109 정합)
```

### 3.2 렌더링 제약 (performance-guidelines 준수)

- Three.js는 **dynamic import + ssr: false** 필수 (초기 번들 보호, ~150KB gzip)
- 뷰포트 진입 시 로드 (IntersectionObserver 또는 스크롤 하단 배치)
- `<Canvas>` 미지원/WebGL 실패 시 폴백: 기존 2D 실루엣 일러스트

### 3.3 데이터 경로 원칙 (유령 배선 방지)

- 읽기: **`body_analyses`만** (프로드 실측 검증 완료 — `body_assessments`는 프로드에 존재하지 않는 유령 테이블)
- v1은 새 컬럼/마이그레이션 없이 기존 컬럼만 소비
- BodyRatios 전체 저장(입력 밀도 향상)은 별도 결정 — ADR-110 참조

## 4. 검증 방법

| 검증        | 방법                                                                |
| ----------- | ------------------------------------------------------------------- |
| 결정론      | 동일 입력 5회 → 정점 배열 해시 동일 (단위 테스트)                   |
| 단조성      | ratio 증가 → 어깨 모프 가중치 단조 증가 (property 테스트)           |
| 타입 구분성 | S/W/N 프리셋 3종 실루엣이 육안 구분 가능 (스크린샷 스냅샷)          |
| 경계 정합   | BODY_SHAPE_THRESHOLDS 경계 입력에서 아바타가 급변하지 않음 (연속성) |
| 성능        | 결과 페이지 LCP 영향 0 (dynamic import 검증), 60fps 회전            |
| 폴백        | WebGL 미지원 환경에서 2D 실루엣 표시                                |
