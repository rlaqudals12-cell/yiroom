# 측정 프레임워크

> 상위 철학: [yiroom-philosophy.md](./yiroom-philosophy.md) Section 11 (측정 프레임워크)
> "성장을 측정하되, 의존을 측정하지 마라"

---

## 1. 핵심 원칙

```
측정의 목적 = 사용자의 자립 진행도 파악
측정의 목적 ≠ 앱 의존도 극대화

모든 지표는 이 질문을 통과해야 한다:
"이 지표가 올라갈 때, 사용자는 더 자립적인가?"
```

---

## 2. North Star Metric

### 2.1 정의

```
North Star: 월간 자립적 판단 횟수 (Monthly Independent Decisions)

= AI 설명/추천 없이 사용자가 직접 선택한 횟수

측정 방법:
1. 추천 목록에서 "직접 선택" 행동 (추천 순위와 무관한 선택)
2. 캡슐에서 AI 추천 수정/교체
3. 인벤토리에서 자율 조합
4. "이미 알고 있어요" 버튼 클릭
```

### 2.2 왜 이것인가

```
검토한 대안들:

| 대안 | 기각 이유 |
|------|-----------|
| DAU/MAU | 의존도 높아도 올라감 |
| 세션 시간 | 길다고 좋은 것 아님 (방황 가능) |
| 분석 완료 수 | Layer 1 고착 가능 |
| 추천 수락률 | 맹목적 수락 구분 불가 |
| NPS | 결과 지표 (선행 아님) |

"자립적 판단"은:
- 올라가면 → 사용자가 성장 중 (좋은 신호)
- 높은데 재방문 → 새 도메인 탐구 (자연스러운 확장)
- 높은데 이탈 → 자립 완료 (성공적 졸업)
```

## 3. Layer별 지표

### 3.1 Layer 1 (즉각 해결)

| 지표               | 목표  | 측정        | 구현              |
| ------------------ | ----- | ----------- | ----------------- |
| **첫 분석 완료율** | 80%+  | 온보딩 퍼널 | 분석 완료 이벤트  |
| 분석 결과 체류시간 | 30초+ | 페이지 체류 | 클라이언트 타이머 |
| 결과 공유율        | 10%+  | 공유 클릭   | 이벤트 로그       |

### 3.2 Layer 1→2 전환 (Aha Moment)

| 지표                    | 목표 | 측정               | 구현                |
| ----------------------- | ---- | ------------------ | ------------------- |
| **2번째 분석 완료율**   | 50%+ | 퍼널               | 분석 카운트 추적    |
| 첫 교차 인사이트 수신율 | 70%+ | 인사이트 생성 로그 | insight 생성 이벤트 |
| Aha→3번째 분석 전환율   | 40%+ | 퍼널               | 분석 카운트         |

### 3.3 Layer 2 (판단력 성장)

| 지표                      | 목표 | 측정        | 구현                               |
| ------------------------- | ---- | ----------- | ---------------------------------- |
| **교차 인사이트 확인율**  | 60%+ | 클릭/체류   | ConnectionAwareness expose+confirm |
| 연결 인식률 (recognized+) | 40%+ | status 통계 | connection_awareness 테이블        |
| "왜" 설명 열람률          | 50%+ | 클릭        | 이벤트 로그                        |

### 3.4 Layer 2→3 전환

| 지표              | 목표      | 측정                 | 구현                      |
| ----------------- | --------- | -------------------- | ------------------------- |
| **연결 내재화율** | 30%+      | internalized / total | connection_awareness 통계 |
| 추천 수정률       | 증가 추세 | 캡슐 수정 횟수       | 캡슐 이벤트 로그          |

### 3.5 Layer 3 (자기 이해)

| 지표                 | 목표      | 측정              | 구현                 |
| -------------------- | --------- | ----------------- | -------------------- |
| **자립적 판단 빈도** | 월 5회+   | 비추천 선택 로그  | 이벤트 분석          |
| independent 연결 수  | 증가 추세 | status 통계       | connection_awareness |
| 설명 생략 비율       | 증가 추세 | depth='none' 비율 | 로그 분석            |

## 4. 반지표 (Anti-Metrics)

### 4.1 경계 지표

```
이 지표가 올라가면 위험 신호:

1. "설명 없이 수락" 비율
   → 이해 없는 맹목적 수락 = 우리 목적과 반대
   → 해결: "왜" 설명 더 노출

2. 동일 연결 10회+ 노출인데 exposed 유지
   → 설명이 효과적이지 않음
   → 해결: 설명 방식 변경, Gemini로 개인화

3. DAU는 높은데 자립적 판단 0
   → 의존 패턴 형성
   → 해결: "직접 골라볼까요?" CTA 추가

4. 알림 클릭률만 높고 인사이트 확인 낮음
   → 알림 남발 의심
   → 해결: 알림 빈도 줄이고 품질 향상
```

### 4.2 건강 지표 조합

```
건강한 성장 =
  ✅ 2번째 분석 완료율 ↑
  ✅ 교차 인사이트 확인율 ↑
  ✅ 내재화율 ↑
  ✅ 자립적 판단 ↑

불건강한 성장 =
  ⚠️ DAU ↑ BUT 자립적 판단 →
  ⚠️ 수락률 ↑ BUT 확인율 ↓
  ⚠️ 세션 시간 ↑ BUT 내재화 →
```

## 5. 측정 구현

### 5.1 이벤트 스키마

```typescript
interface AnalyticsEvent {
  eventName: string;
  userId: string;
  timestamp: string;
  properties: Record<string, unknown>;
}

// 핵심 이벤트
type CoreEvent =
  | 'analysis_completed' // 분석 완료
  | 'insight_exposed' // 인사이트 노출
  | 'insight_confirmed' // 인사이트 확인
  | 'independent_decision' // 자립적 판단
  | 'explanation_viewed' // "왜" 설명 열람
  | 'recommendation_modified' // 추천 수정
  | 'capsule_customized'; // 캡슐 커스터마이징
```

### 5.2 대시보드 (관리자)

```
/admin/metrics 페이지:

1. North Star 차트 — 월간 자립적 판단 추이
2. 퍼널 — L1 → L2 → L3 전환율
3. 내재화 분포 — exposed/recognized/internalized/independent 비율
4. 반지표 경고 — 불건강 패턴 감지 시 알림
```
