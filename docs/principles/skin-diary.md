# 피부 일기 원리 (Skin Diary Principles)

> 피부 상태의 시간축 추적 및 트렌드 분석의 과학적 근거
> **관련 모듈**: S-1 (피부 분석), S-2 (12존 분석), 피부 일기

---

## 1. 핵심 원리: 피부는 시간 함수다

피부 상태는 정적이지 않다. 다음 변수들에 의해 **일 단위**로 변동한다:

| 변수          | 영향 주기 | 피부 반응                 |
| ------------- | --------- | ------------------------- |
| 수면          | 일 단위   | 수분 보유량, 탄력         |
| 식단          | 1-3일     | 유분, 트러블              |
| 스트레스      | 1-7일     | 코르티솔 → 피지 분비 ↑    |
| 생리 주기     | 28일      | 호르몬 → 유분/트러블 주기 |
| 계절          | 90일      | 습도 → 수분/유분 밸런스   |
| 스킨케어 제품 | 14-28일   | 성분 효과 발현            |

### 1.1 턴오버 주기

표피 세포의 완전 교체(turnover) 주기는 약 **28일**.

- 스킨케어 제품 효과 판단에 최소 4주 데이터 필요
- 2주 미만의 트렌드는 **노이즈**일 가능성이 높음

### 1.2 트렌드 분석의 통계적 요건

| 분석 유형      | 최소 데이터 포인트 | 신뢰도        |
| -------------- | ------------------ | ------------- |
| 주간 변화 감지 | 3회 (7일 내)       | 낮음 (참고용) |
| 월간 트렌드    | 4회 (28일 내)      | 보통          |
| 계절 패턴      | 12회+ (90일 내)    | 높음          |

## 2. 자동 기록 원리: 마찰 최소화

### 2.1 행동과학적 근거

일기 앱의 **30일 지속률은 5-10%** (Habit Formation 연구).
높은 마찰(매일 셀카 + 설문)은 이탈의 주요 원인.

**해결 원리**: 사용자 행동을 요구하지 않고, **기존 행동의 부산물로 데이터를 축적**한다.

```
❌ 높은 마찰: "오늘 피부 일기를 작성하세요" (능동적 행동 요구)
✅ 낮은 마찰: "분석할 때마다 자동으로 기록됩니다" (기존 행동의 부산물)
```

### 2.2 데이터 소스 계층

| 우선순위 | 소스                         | 마찰       | 데이터 품질    |
| -------- | ---------------------------- | ---------- | -------------- |
| 1 (자동) | S-1/S-2 분석 결과            | 없음       | 높음 (AI 분석) |
| 2 (자동) | ConnectionAwareness 인사이트 | 없음       | 보통           |
| 3 (선택) | 사용자 한줄 메모             | 극저 (1탭) | 주관적         |
| 4 (선택) | 컨디션 이모지 선택           | 극저 (1탭) | 주관적         |

**원칙**: 1-2 자동 소스만으로도 트렌드 분석이 가능해야 한다.
3-4 선택 소스는 **부가 정보**일 뿐, 필수가 아니다.

## 3. 트렌드 감지 알고리즘

### 3.1 이동 평균 (Moving Average)

피부 점수의 노이즈를 제거하고 트렌드를 추출한다.

```
SMA(n) = Σ(score[i], i=1..n) / n

n = 3: 단기 트렌드 (주간)
n = 7: 중기 트렌드 (월간)
```

### 3.2 악화 감지 (Deterioration Detection)

연속 하락 패턴을 감지하여 사용자에게 알린다.

```
악화 조건:
  - 최근 3회 연속 점수 하락 (각 5점 이상)
  - 또는 7일 이동평균이 이전 대비 10점 이상 하락

개선 조건:
  - 최근 3회 연속 점수 상승 (각 5점 이상)
  - 또는 7일 이동평균이 이전 대비 10점 이상 상승
```

### 3.3 변화율 (Rate of Change)

```
ROC = (현재 SMA - 이전 SMA) / 이전 SMA × 100

|ROC| < 5%: 안정
|ROC| 5-15%: 주의
|ROC| > 15%: 알림 트리거
```

## 4. 크로스 도메인 상관 분석

피부 일기의 궁극적 가치는 **피부 변화의 원인을 다른 도메인에서 찾는 것**이다.

### 4.1 상관 관계 매핑

```
피부 수분 ↓  ←→  수분 섭취 ↓ (N-1 영양)
피부 트러블 ↑ ←→  수면 부족 (W-1 운동/라이프스타일)
피부 탄력 ↑  ←→  콜라겐 식품 섭취 ↑ (N-1 영양)
피부 유분 ↑  ←→  고지방 식단 (N-1 영양)
```

### 4.2 ConnectionAwareness 연동

기존 ConnectionAwareness 시스템이 이 상관 관계를 자동으로 추적한다:

- `skin` ↔ `nutrition`: 피부-영양 연결
- `skin` ↔ `workout`: 피부-운동 연결

피부 일기 트렌드와 다른 모듈 기록의 시간적 상관을 분석하여
ConnectionAwareness 인사이트를 **자동 생성**한다.

## 5. 시각화 원리

### 5.1 시간축 표현

| 뷰     | X축       | Y축             | 용도           |
| ------ | --------- | --------------- | -------------- |
| 주간   | 7일       | 바이탈리티 점수 | 단기 변동 확인 |
| 월간   | 30일      | 이동평균 + 원본 | 트렌드 확인    |
| 캘린더 | 달력 격자 | 색상 코딩       | 전체 조감      |

### 5.2 색상 코딩

```
점수 90+: 초록 (S등급)
점수 75-89: 파랑 (A등급)
점수 60-74: 노랑 (B등급)
점수 40-59: 주황 (C등급)
점수 0-39: 빨강 (D등급)
```

S-2의 기존 바이탈리티 등급 체계(S/A/B/C/D)를 그대로 활용한다.

## 6. 이룸 철학과의 정합성

### "통합 자기 이해"에서 피부 일기의 위치

```
단발 분석 (현재):  "지금 내 피부는 복합성, 바이탈리티 72점"
시간축 확장 (일기): "지난 달 대비 수분 개선, 영양 섭취 변화와 연관"
궁극의 형태:        "나의 피부는 이런 패턴으로 변하고, 이런 습관이 영향을 준다"
```

피부 일기는 단발 스냅샷을 **시간축으로 연결**하여
사용자가 자신의 피부 패턴을 **스스로 이해**하게 돕는 도구다.
