# 👀 Code Review 프롬프트 템플릿

> 코드 품질 검토 및 개선 제안용

---

## 기본 프롬프트

```
[파일 경로]의 코드를 리뷰해주세요.

검토 항목:
- 코드 품질
- 성능
- 보안
- 테스트 커버리지

개선 사항이 있으면 구체적으로 제안해주세요.
```

---

## 상세 프롬프트

```
다음 파일들을 코드 리뷰해주세요:

파일:
- [파일 경로 1]
- [파일 경로 2]

검토 관점:
1. **가독성**: 코드가 명확하고 이해하기 쉬운가?
2. **유지보수성**: 향후 수정이 용이한가?
3. **성능**: 불필요한 연산이나 리렌더링이 있는가?
4. **보안**: 취약점이 있는가?
5. **일관성**: 프로젝트 컨벤션을 따르는가?

문제가 있으면 구체적인 개선 코드를 제안해주세요.
```

---

## 예시: 컴포넌트 리뷰

```
components/workout/WorkoutCard.tsx를 리뷰해주세요.

검토 관점:
1. React 최적화 (useMemo, useCallback 적절히 사용?)
2. props 타입 정의
3. 접근성 (a11y)
4. 스타일링 일관성

개선 사항 있으면 수정된 코드를 보여주세요.
```

---

## 예시: API 리뷰

```
app/api/workout/analyze/route.ts를 리뷰해주세요.

검토 관점:
1. 인증 처리
2. 입력 검증
3. 에러 핸들링
4. 응답 형식 일관성
5. 타임아웃 처리

보안 취약점이나 개선점이 있으면 알려주세요.
```

---

## 예시: 전체 모듈 리뷰

```
W-1 운동 모듈 전체를 리뷰해주세요.

대상 디렉토리:
- app/workout/
- components/workout/
- lib/workout/
- stores/workoutStore.ts

검토 관점:
1. 아키텍처 적절성
2. 코드 중복
3. 타입 안정성
4. 테스트 커버리지
5. 성능 병목

주요 개선 포인트 Top 5를 정리해주세요.
```

---

## 체크리스트 형식

```
다음 체크리스트로 [파일 경로]를 검토해주세요:

□ TypeScript 타입이 명확한가?
□ 에러 핸들링이 적절한가?
□ 불필요한 리렌더링이 없는가?
□ API 호출에 로딩/에러 상태가 있는가?
□ 접근성(a11y)이 고려되었는가?
□ 테스트가 작성되었는가?
□ 주석이 필요한 복잡한 로직에 주석이 있는가?
□ 하드코딩된 값이 없는가?

각 항목에 대해 Pass/Fail과 개선 제안을 해주세요.
```

---

## 팁

- 구체적인 파일 경로 명시
- 검토 관점 우선순위 제공
- 개선 코드 요청 시 명시
- 전체 리뷰 시 Top N 형식 요청
