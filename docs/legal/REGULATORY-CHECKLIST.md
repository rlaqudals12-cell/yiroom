## 1. 약사법 제44조 (의약품 판매 행위)

### 1.1 준수 사항

| #   | 체크 | 항목                                  | 기준                          | 검증 방법                                |
| --- | ---- | ------------------------------------- | ----------------------------- | ---------------------------------------- |
| 1   | [ ]  | 특정 의약품 용법/용량 지시 없음       | "하루 2회 복용하세요" 등 금지 | `scripts/check-forbidden-expressions.sh` |
| 2   | [ ]  | 의약품/의약외품 직접 추천 없음        | "XX 연고를 바르세요" 등 금지  | 코드베이스 금지 표현 스캔                |
| 3   | [ ]  | 성분 정보는 "일반 특성 설명"으로 한정 | "XX 성분이 포함되어 있어요"   | UI 텍스트 리뷰                           |
| 4   | [ ]  | "추천" 대신 "고려" 표현 사용          | "~를 고려해볼 수 있어요"      | `grep -rn "추천합니다" apps/`            |

### 1.2 허용/금지 경계

```
허용:
  ✅ "비타민 C가 풍부한 식품 목록이에요"
  ✅ "건조한 피부에 보습 크림을 사용해볼 수 있어요"
  ✅ "XX 성분이 포함된 제품을 찾아보세요"

금지:
  ❌ "비타민 C 1000mg 보충제를 드세요"
  ❌ "이 약을 하루 2회 복용하세요"
  ❌ "피부 트러블에 XX 연고를 바르세요"
```

## 3. 의료법 제27조 (무면허 의료행위)

### 3.1 준수 사항

| #   | 체크 | 항목                       | 기준                                  |
| --- | ---- | -------------------------- | ------------------------------------- |
| 1   | [ ]  | 질병 진단 표현 없음        | "접촉피부염입니다", "습진입니다" 금지 |
| 2   | [ ]  | AI 분석 = "참고 정보" 명시 | 모든 분석 결과에 면책 포함            |
| 3   | [ ]  | 질환 관리 지시 없음        | "당뇨 식단을 따르세요" 금지           |
| 4   | [ ]  | 전문의 상담 권고 포함      | "증상이 있으면 전문의와 상담해주세요" |

### 3.2 이룸 기능별 분류

| 기능                          | 분류       | 근거                  |
| ----------------------------- | ---------- | --------------------- |
| AI 피부 타입 분석 (건성/지성) | **비의료** | 일반적 피부 상태 분류 |
| 성분 알레르기 경고            | **비의료** | 기존 정보 기반 필터링 |
| 칼로리/영양소 계산            | **비의료** | 일반 건강 정보        |
| 체형 타입 분석                | **비의료** | 일반적 신체 특성 분류 |
| 퍼스널컬러 분석               | **비의료** | 색상 특성 분류        |
| 운동 추천                     | **비의료** | 일반 건강 정보        |

### 3.3 절대 금지 기능

```
이룸이 절대 제공하지 않는 기능:
  ❌ 피부 질환 진단 ("피부암이 의심됩니다")
  ❌ 질병 진단 ("습진입니다", "아토피입니다")
  ❌ 처방/용법 지시 ("이 약을 복용하세요")
  ❌ 의약품 효능 표현 ("이 약은 부작용이 있어요")
  ❌ 특정 질환 식단 지시 ("당뇨 식단을 따르세요")
```

| 7   | [ ]  | AES-256 암호화 저장     | safety_profiles 테이블                |

| 체형 타입      | 개인정보     | 일반 동의 | RLS           |

## 5. 디지털의료제품법 (2025년 시행)

### 5.1 이룸 분류: 비의료 건강관리 앱

```
디지털의료제품법 분류:
  - 의료기기: 질병 진단/치료/예방 → 이룸 해당 없음
  - 디지털치료기기: 질환 치료 소프트웨어 → 이룸 해당 없음
  - 건강관리 앱: 일반 건강 정보 제공 → ✅ 이룸 해당

비의료 건강관리 앱 조건:
  1. 질병 진단 기능 없음
  2. 치료 지시 기능 없음
  3. 일반적 건강 정보 제공만
  4. "의료 조언 아님" 면책 표시
```

### 5.2 이룸 "알레르겐 검사"의 법적 성격

```
이룸의 "알레르겐 검사"는:
- 입력: 사용자의 자가 입력 알레르기 정보
- 처리: 미리 정의된 알레르겐 목록과 단순 문자열 매칭
- 출력: 일치 성분 필터링 (차단/경고 표시)
→ "의료적 진단/판정"이 아닌 "정보 필터링 서비스"로 분류
→ 디지털의료제품법 비해당 (비의료 건강관리 앱)
```

### 5.3 준수 사항

| #   | 체크 | 항목                                       |
| --- | ---- | ------------------------------------------ |
| 1   | [ ]  | 앱 설명에 "의료기기 아님" 명시             |
| 2   | [ ]  | 진단/치료 관련 기능 없음 확인              |
| 3   | [ ]  | "건강 정보 참고용" 면책 앱 내 표시         |
| 4   | [ ]  | 스토어 등록 시 "건강 및 피트니스" 카테고리 |

## 8. 종합 체크리스트

### 8.1 출시 전 필수 (법적 최소 요건)

```
약사법:
  □ 의약품 용법/용량 지시 없음
  □ 의약품 직접 추천 없음

화장품법:
  □ 금지 표현 스캔 완료 (0건)
  □ "개인차" 안내 포함

의료법:
  □ 질병 진단 기능 없음
  □ 모든 분석 결과에 면책 문구

개인정보보호법:
  □ 민감정보 별도 동의 UI
  □ AES-256 암호화 저장
  □ 동의 철회 → 즉시 삭제
  □ 로그에 PII 미포함

디지털의료제품법:
  □ 비의료 건강관리 앱 분류 확인
  □ "의료기기 아님" 면책

전자상거래법:
  □ 어필리에이트 공시
  □ 추천 기준 명시
```

### 8.2 정기 검토 (분기별)

```
□ 금지 표현 코드베이스 스캔
□ 면책 문구 법령 변경 반영
□ 개인정보처리방침 최신화
□ 관련 법령 개정 사항 확인
□ Safety DB (알레르겐/금기사항) 최신화
```
