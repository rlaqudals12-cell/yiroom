## 0. 정체성 재정의 v2 (2026-04-22)

> **이 섹션은 v2.0에서 추가된 최상위 정체성 선언이며, 하위 모든 섹션보다 우선한다.**

### 0.1 한 문장 정체성

> **"거울 + 옷장을 이해해주는 앱 — 당신의 시각적 정체성 5축을 해석하고, 당장 실행할 수 있게 세상과 연결한다."**

### 0.2 5축 모델 (Visual Identity Axes)

이룸은 사용자의 **시각적 정체성**을 5개 축으로 분해해 해석한다. 이것이 이룸의 본질이며, 나머지 기능은 이 5축을 받쳐주는 인프라/응용 레이어다.

|      축      | 모듈 | 역할                                                           | 입력                                       |
| :----------: | ---- | -------------------------------------------------------------- | ------------------------------------------ |
|    **색**    | PC-1 | 나의 색 정체성 (언더톤/계절)                                   | 얼굴 사진 (1회 측정, 거의 불변)            |
|   **피부**   | S-1  | 나의 피부 상태                                                 | 얼굴 사진 (주기적 재측정)                  |
|   **체형**   | C-1  | 나의 몸 비율 — "개선"이 아닌 "이해와 표현"                     | 전신 사진 + 자가 입력                      |
|   **헤어**   | H-1  | 나의 헤어 — 얼굴형 기반 + 모발 타입/굵기/선호 (하이브리드)     | 얼굴 사진 + 자가 입력                      |
| **메이크업** | M-1  | **실행 레이어** (독립 분석 아님) — PC + S + 상황 입력으로 제안 | PC-1/S-1 결과 + 상황(데일리/오피스/데이트) |

**핵심 통찰**: 얼굴 사진 1장 → PC-1 + S-1 + M-1(실행). 전신 사진 1장 + 자가 입력 2분 → C-1 + H-1. **총 2분 30초에 4축 분석 + M 실행 레이어 완성**.

### 0.3 제거된 모듈

| 모듈                | 상태          | 근거                                                                         |
| ------------------- | ------------- | ---------------------------------------------------------------------------- |
| **OH-1 (구강건강)** | **완전 제거** | 뷰티 포지셔닝과 이질, "나를 잘 아는 존재" 맥락에서 빠짐, 쇼핑 전환 연결 없음 |

### 0.4 Phase 2 보류 모듈

| 모듈           | 상태                  | 재검토 시점                                                         |
| -------------- | --------------------- | ------------------------------------------------------------------- |
| **W-1 (운동)** | UI 숨김, 코드/DB 유지 | MAU 10만+ 또는 "동반자 단계" 진입 시                                |
| **N-1 (영양)** | UI 숨김, 코드/DB 유지 | 동일. 단, S-1 하위의 "피부에 좋은 음식" 형태로 흡수 가능성은 열어둠 |

**보류 근거**: 운동·영양은 삼성헬스/마이피트니스팔 등 전문 앱이 이미 잘 하고 있음. 이룸이 대체 불가능한 영역 아님. 수익 모델(어필리에이트)과 궁합 나쁨(헬스 앱은 구독 모델). Phase 1의 "시각 정체성"에 집중하는 것이 경쟁 우위.

### 0.5 갈래 B 확정 (뷰티 정체성 vs 통합 웰니스)

3갈래 중 **갈래 B (시각 정체성 앱)** 확정:

- 갈래 A (얼굴 앱만) — 경쟁사(Oh My Skin) 직접 경쟁, 차별화 부족
- **갈래 B (5축, 얼굴+몸+스타일)** — 경쟁사 없음, 쇼핑 전환 2배 (화장품 + 의류)
- 갈래 C (웰니스 통합) — 포지셔닝 희석, 삼성헬스와 직접 경쟁

### 0.6 C-1 출력 구조 (확정)

C-1 결과 페이지는 **시간축이 다른 3종 세트**로 구성:

| 섹션                        | 내용                                          | 목적                             |
| --------------------------- | --------------------------------------------- | -------------------------------- |
| **섹션 1: 스타일링 원칙**   | 체형별 3~5개 원칙 문장 (`STYLING_PRINCIPLES`) | 쇼핑할 때 기억하는 **장기 기준** |
| **섹션 2: 추천 코디 3세트** | 기존 `getOutfitExamples()` 재활용             | 오늘 따라 입는 **단기 실행**     |
| **섹션 3: 옷장 조합**       | 인벤토리 연동 (Phase 1.5, 출시 후)            | 돈 없이 실행하는 **무료 경로**   |

### 0.7 0 → 1 → 궁극 로드맵

```
[현재: 분석기]
  5축 각자 분석, 결과가 개별 카드로 나열됨
         ↓
[다음: 조언자]
  5축 → 하나의 "나 프로필"로 통합 + 상황별 조언
  ("오늘 이 메이크업", "이번 주 이 제품")
         ↓
[궁극: 동반자]
  5축 변화 추적, 선제적 제안, 정체성의 시간 여행
```

## 1. 이상적 최종 상태

### 1.1 핵심 가치

```
"사진 한 장으로 온전한 나를 이해하고, 변화를 돕는다"

사용자가 이룸을 통해:
├── 자신의 피부, 체형, 컬러를 과학적으로 이해한다
├── 개인화된 추천을 신뢰하고 행동한다
└── 변화를 추적하고 동기부여를 받는다
```

### 1.2 100점의 사용자 경험

| 단계       | 이상적 상태             | 측정 기준          |
| ---------- | ----------------------- | ------------------ |
| **온보딩** | 2분 내 첫 분석 완료     | 온보딩 완료율 95%+ |
| **분석**   | 3초 내 결과 표시        | p95 응답시간 < 3s  |
| **결과**   | 즉시 이해 가능한 시각화 | 이해도 점수 4.5+/5 |
| **추천**   | "이거 나한테 맞네" 체감 | 추천 수용률 70%+   |
| **재방문** | 습관적으로 확인         | MAU 유지율 60%+    |

### 1.3 100점의 기술적 상태

| 영역          | 이상적 상태      | 측정 기준                  |
| ------------- | ---------------- | -------------------------- |
| **AI 정확도** | 전문가 수준 분석 | 피부과 의사 대비 90%+ 일치 |
| **성능**      | 즉각적 반응      | LCP < 2.5s, FID < 100ms    |

| **테스트**    | 완전 자동화      | 커버리지 80%+, E2E 완비    |

□ P3: 원자 단위로 분해되고 테스트됨
□ P4: 더 단순화할 수 없음이 확인됨
□ G0-G5: 모든 Gate 통과됨
```

```
