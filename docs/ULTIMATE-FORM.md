# 이룸 궁극의 형태 (P1)

> "목적지를 모르면 어떤 길도 옳다"
>
> 이 문서는 이룸 프로젝트의 100점 상태를 정의한다.
> 모든 구현은 이 문서를 기준으로 달성도를 측정한다.

---

## 1. 이상적 최종 상태

### 1.1 핵심 가치

```
"사진 한 장으로 온전한 나를 이해하고, 변화를 돕는다"

사용자가 이룸을 통해:
├── 자신의 피부, 체형, 컬러를 과학적으로 이해한다
├── 개인화된 추천을 신뢰하고 행동한다
└── 변화를 추적하고 동기부여를 받는다
```

### 1.2 100점의 사용자 경험

| 단계 | 이상적 상태 | 측정 기준 |
|------|------------|----------|
| **온보딩** | 2분 내 첫 분석 완료 | 온보딩 완료율 95%+ |
| **분석** | 3초 내 결과 표시 | p95 응답시간 < 3s |
| **결과** | 즉시 이해 가능한 시각화 | 이해도 점수 4.5+/5 |
| **추천** | "이거 나한테 맞네" 체감 | 추천 수용률 70%+ |
| **재방문** | 습관적으로 확인 | MAU 유지율 60%+ |

### 1.3 100점의 기술적 상태

| 영역 | 이상적 상태 | 측정 기준 |
|------|------------|----------|
| **AI 정확도** | 전문가 수준 분석 | 피부과 의사 대비 90%+ 일치 |
| **성능** | 즉각적 반응 | LCP < 2.5s, FID < 100ms |
| **안정성** | 무중단 운영 | 가용성 99.9%+ |
| **보안** | 제로 트러스트 | OWASP Top 10 완전 대응 |
| **테스트** | 완전 자동화 | 커버리지 80%+, E2E 완비 |

## 2. 물리적/논리적 한계

### 2.1 기술적 한계

| 한계 | 설명 | 대응 |
|------|------|------|
| **2D 이미지 한계** | 사진에서 3D 정보 추출 불가 | 다각도 촬영 안내, 오차 명시 |
| **AI 정확도 상한** | 100% 정확도는 불가능 | 신뢰도 점수 표시, 전문가 상담 권장 |
| **조명 의존성** | 조명에 따라 분석 결과 변동 | 조명 보정 알고리즘, 재촬영 안내 |
| **개인차** | 동일 분석도 개인마다 다름 | 개인화 학습, 피드백 반영 |

### 2.2 비용/시간 제약

| 제약 | 현실 | 대응 |
|------|------|------|
| **Gemini API 비용** | 분석당 ~$0.01 | Rate Limiting, 캐싱 |
| **개발 리소스** | 1인 개발 | 우선순위 집중, 자동화 |
| **법적 검토** | 전문 법률 자문 필요 | 보수적 접근, 단계적 출시 |

## 3. 모듈별 궁극의 형태

### 3.1 Phase 1: Core Image Engine (CIE)

```
## CIE 궁극의 형태

### 이상적 최종 상태
- 모든 조명 조건에서 일관된 분석
- 실시간 품질 피드백 (촬영 중 안내)
- 자동 색온도/화이트밸런스 보정

### 100점 기준
- Sharpness 검증: 99% 정확도
- 색온도 추정: ±200K 오차
- 얼굴 감지: 99.5%+ 성공률

### 현재 목표: 85%
- 기본 sharpness + 색온도 검증
- 정면 얼굴 감지

### 의도적 제외
- 실시간 촬영 안내 (v2에서)
- 다각도 합성 (v3에서)
```

```
## PC-2 (퍼스널컬러 v2) 궁극의 형태

### 이상적 최종 상태

- 웜톤/쿨톤 명확 구분

### 의도적 제외
```

## 4. Phase별 달성 목표

| Phase | 목표 달성도 | 핵심 결과물 | 완전성 체크 |
|-------|-----------|------------|------------|

| **-1: 보안** | 100% | RLS 완성, Rate Limiting | G0-G5 완료 |

## 5. 달성도 측정 기준

### 5.1 각 모듈 완전성 체크리스트

```markdown
□ P0: 요구사항이 검증됨 (왜? 3회)
□ P1: 궁극의 형태가 이 문서에 정의됨
□ P1: 현재 달성도 %가 명시됨
□ P1: 의도적 제외 항목이 기록됨
□ P2: 원리 문서가 docs/principles/에 존재
□ P3: 원자 단위로 분해되고 테스트됨
□ P4: 더 단순화할 수 없음이 확인됨
□ G0-G5: 모든 Gate 통과됨
```

### 5.2 전체 프로젝트 달성도

```
현재 상태: 약 55%

├── 문서 (원리 + ADR + SDD): 70%
├── 법적 준수: 30% (문서만, 구현 미완)
├── 보안: 50% (RLS 일부, Rate Limiting 일부)
├── Core 기능: 60% (v1 완료, v2 미완)
├── 확장 기능: 20% (Phase 3 미착수)
├── 테스트: 40% (추정)
└── 성능/최적화: 50%
```

## 6. 의도적 제외 (Not Now)

| 항목 | 제외 이유 | 재검토 시점 |
|------|----------|------------|
| **3D 스캔 연동** | 하드웨어 의존, 보급률 낮음 | 2027년 이후 |
| **실시간 영상 분석** | 연산 부담, 배터리 소모 | 성능 개선 후 |
| **AI 개인화 학습** | 데이터 축적 필요 | 사용자 100K+ 후 |
| **다국어 지원** | 초기 집중 필요 | 한국 시장 안정화 후 |
| **SNS 공유 기능** | 핵심 가치와 거리 | Phase 5 이후 |
