# 이룸 용어 설명집 (Glossary)

> 개발 문서에서 자주 등장하는 용어들을 쉽게 이해할 수 있도록 정리한 가이드

## 📖 이 문서 사용법

- **알파벳순**으로 정렬되어 있습니다
- 각 용어는 **한 줄 설명** + **쉬운 비유** + **사용 예시**로 구성
- 관련 용어는 `→ 참고:` 로 연결

---

## A

### API (Application Programming Interface)

**한 줄 설명**: 프로그램들이 서로 대화하는 방법

**쉬운 비유**: 식당의 메뉴판과 웨이터. 손님(프로그램)이 메뉴판(API 문서)을 보고 주문하면, 웨이터(API)가 주방(서버)에 전달하고 음식(데이터)을 가져다줍니다.

**이룸에서 사용**: `/api/analyze/skin` - 피부 분석 요청을 받는 API

→ 참고: REST API, Route Handler

### App Router

**한 줄 설명**: Next.js에서 페이지를 구성하는 최신 방식

**쉬운 비유**: 건물의 층별 안내도. `app/` 폴더 안의 폴더 구조가 곧 웹사이트의 주소(URL) 구조가 됩니다.

**이룸에서 사용**: `app/(main)/analysis/skin/page.tsx` → `/analysis/skin` 페이지

→ 참고: Route Group

---

## B

### Backend (백엔드)

**한 줄 설명**: 사용자 눈에 보이지 않는 서버 측 프로그램

**쉬운 비유**: 식당의 주방. 손님은 주방을 못 보지만, 주방에서 음식(데이터)을 만들어 내보냅니다.

→ 참고: Frontend, API

### Build (빌드)

**한 줄 설명**: 개발 코드를 실제 서비스용으로 변환하는 과정

**쉬운 비유**: 원고를 책으로 인쇄하는 과정. 개발자가 쓴 코드(원고)를 사용자가 볼 수 있는 형태(책)로 만듭니다.

**명령어**: `npm run build`

---

## C

### Client Component

**한 줄 설명**: 사용자의 브라우저에서 실행되는 React 컴포넌트

**쉬운 비유**: 스마트폰 앱. 버튼 클릭, 입력 등 사용자와 직접 상호작용하는 부분입니다.

**이룸에서 사용**: `'use client'`로 시작하는 파일

→ 참고: Server Component

### Clerk

**한 줄 설명**: 로그인/회원가입을 쉽게 구현해주는 서비스

**쉬운 비유**: 아파트 관리실. 입주민(사용자) 확인, 출입 카드(토큰) 발급, 보안 관리를 대신 해줍니다.

**이룸에서 사용**: Google/카카오 로그인, 사용자 인증

→ 참고: clerk_user_id, 인증

### clerk_user_id

**한 줄 설명**: Clerk에서 발급한 사용자 고유 번호

**쉬운 비유**: 주민등록번호. 각 사용자를 유일하게 식별하는 ID입니다.

**이룸에서 사용**: 모든 테이블에서 "이 데이터가 누구 것인지" 확인하는 기준

### Component (컴포넌트)

**한 줄 설명**: 화면을 구성하는 재사용 가능한 조각

**쉬운 비유**: 레고 블록. 버튼, 카드, 입력창 등을 블록처럼 조립해서 페이지를 만듭니다.

**이룸에서 사용**: `components/` 폴더의 모든 파일

### CORS (Cross-Origin Resource Sharing)

**한 줄 설명**: 다른 도메인의 데이터를 가져올 수 있게 허용하는 설정

**쉬운 비유**: 여권과 비자. 다른 나라(도메인)의 자원을 가져오려면 허가(CORS 설정)가 필요합니다.

### CSS Variables (CSS 변수)

**한 줄 설명**: 색상, 크기 등을 한 곳에서 관리하는 방법

**쉬운 비유**: 회사 로고 색상 코드표. "우리 회사 메인 색상"을 정해두면 모든 문서에서 같은 색을 씁니다.

**이룸에서 사용**: `--module-skin`, `--primary` 등 globals.css의 변수

→ 참고: Design Tokens

---

## D

### Database (데이터베이스, DB)

**한 줄 설명**: 데이터를 체계적으로 저장하는 저장소

**쉬운 비유**: 엑셀 파일들이 모인 거대한 서랍장. 각 엑셀 파일이 "테이블", 각 행이 "레코드"입니다.

**이룸에서 사용**: Supabase (PostgreSQL)

### Design Tokens (디자인 토큰)

**한 줄 설명**: 디자인 요소(색상, 폰트 등)를 코드로 정의한 것

**쉬운 비유**: 패션 브랜드의 시그니처 컬러. "이룸 핑크"라고 정해두면 모든 곳에서 같은 색을 씁니다.

**이룸에서 사용**: `--module-skin: oklch(0.65 0.18 250)` 같은 CSS 변수

### Dynamic Import (동적 불러오기)

**한 줄 설명**: 필요할 때만 코드를 불러오는 방식

**쉬운 비유**: 주문형 배송. 모든 물건을 미리 사지 않고, 필요할 때만 주문합니다.

**이룸에서 사용**: 차트, 모달 등 무거운 컴포넌트에 적용

---

## E

### Environment Variables (환경 변수)

**한 줄 설명**: 비밀번호, API 키 등 민감한 설정값

**쉬운 비유**: 금고 비밀번호. 코드에 직접 쓰지 않고 별도 파일(`.env`)에 보관합니다.

**이룸에서 사용**: `GOOGLE_GENERATIVE_AI_API_KEY`, `SUPABASE_URL` 등

### Expo

**한 줄 설명**: React Native 앱을 쉽게 만들 수 있게 해주는 도구

**쉬운 비유**: 스마트폰 앱 만들기 키트. 복잡한 설정 없이 앱을 개발할 수 있습니다.

**이룸에서 사용**: `apps/mobile/` 모바일 앱

---

## F

### Fallback (폴백)

**한 줄 설명**: 원래 방법이 실패했을 때 대신 사용하는 방법

**쉬운 비유**: 비상 출구. 정문이 막히면 비상구로 나가듯, AI가 실패하면 Mock 데이터를 사용합니다.

**이룸에서 사용**: Gemini API 실패 시 Mock 데이터로 대체

### Frontend (프론트엔드)

**한 줄 설명**: 사용자 눈에 보이는 화면 부분

**쉬운 비유**: 식당의 홀. 손님이 직접 보고 사용하는 모든 것(메뉴판, 테이블, 인테리어)입니다.

→ 참고: Backend

---

## G

### Gemini

**한 줄 설명**: Google이 만든 AI 모델

**쉬운 비유**: 똑똑한 비서. 사진을 보고 피부 상태를 분석하거나, 체형을 판단해줍니다.

**이룸에서 사용**: PC-1, S-1, C-1 등 모든 AI 분석

### Git

**한 줄 설명**: 코드 변경 이력을 관리하는 도구

**쉬운 비유**: 문서의 "변경 내역 추적" 기능. 누가, 언제, 무엇을 바꿨는지 기록합니다.

### Glob Pattern (글롭 패턴)

**한 줄 설명**: 여러 파일을 한 번에 선택하는 패턴

**쉬운 비유**: "모든 .jpg 파일"처럼 조건으로 파일을 찾는 방법

**예시**: `**/*.tsx` - 모든 폴더의 .tsx 파일

---

## H

### Hook (훅)

**한 줄 설명**: React에서 기능을 추가하는 함수

**쉬운 비유**: 플러그인. `useState`는 데이터 저장, `useEffect`는 부가 작업 실행 기능을 추가합니다.

**이룸에서 사용**: `useClerkSupabaseClient()`, `useUserMatching()` 등

### Hydration (하이드레이션)

**한 줄 설명**: 서버에서 만든 HTML에 JavaScript 기능을 연결하는 과정

**쉬운 비유**: 마네킹에 생명을 불어넣는 과정. 정적인 HTML(마네킹)이 동적인 앱(사람)이 됩니다.

→ 참고: SSR

---

## I

### i18n (Internationalization, 국제화)

**한 줄 설명**: 앱을 여러 언어로 지원하도록 만드는 것

**쉬운 비유**: 다국어 메뉴판. 한국어, 영어, 일본어 메뉴판을 준비하는 것과 같습니다.

**이룸에서 사용**: next-intl 라이브러리 (도입 예정)

---

## J

### JSON (JavaScript Object Notation)

**한 줄 설명**: 데이터를 저장하고 전송하는 형식

**쉬운 비유**: 국제 우편 양식. 어떤 나라(프로그램)에서든 읽을 수 있는 표준 형식입니다.

**예시**:

```json
{
  "name": "홍길동",
  "age": 25
}
```

### JSONB

**한 줄 설명**: PostgreSQL에서 JSON을 효율적으로 저장하는 방식

**쉬운 비유**: 압축 JSON. 빠른 검색이 가능한 JSON 저장 방식입니다.

**이룸에서 사용**: `concerns`, `recommendations` 등 복잡한 데이터 저장

---

## L

### Lint (린트)

**한 줄 설명**: 코드 스타일과 오류를 자동으로 검사하는 도구

**쉬운 비유**: 맞춤법 검사기. 코드의 오타, 잘못된 패턴을 찾아줍니다.

**명령어**: `npm run lint`

---

## M

### Migration (마이그레이션)

**한 줄 설명**: 데이터베이스 구조를 변경하는 명령

**쉬운 비유**: 건물 리모델링 설계도. "2층에 방 하나 추가"처럼 DB 변경 내용을 기록합니다.

**이룸에서 사용**: `supabase/migrations/` 폴더의 SQL 파일

### Mock Data (목 데이터)

**한 줄 설명**: 테스트나 개발용 가짜 데이터

**쉬운 비유**: 전시용 음식 모형. 실제 음식은 아니지만 보여주기용으로 사용합니다.

**이룸에서 사용**: `lib/mock/` 폴더 - AI 실패 시 대체 데이터

### Monorepo (모노레포)

**한 줄 설명**: 여러 프로젝트를 하나의 저장소에서 관리하는 방식

**쉬운 비유**: 대형 쇼핑몰. 여러 매장(앱)이 한 건물(저장소)에 모여있습니다.

**이룸에서 사용**: `apps/web` + `apps/mobile` + `packages/shared`

---

## N

### Next.js

**한 줄 설명**: React 기반의 웹 프레임워크

**쉬운 비유**: 집 짓기 키트. React(벽돌)를 가지고 집(웹사이트)을 빠르게 지을 수 있게 해줍니다.

**이룸에서 사용**: 웹 앱의 기반 프레임워크

### npm (Node Package Manager)

**한 줄 설명**: JavaScript 패키지(라이브러리)를 설치/관리하는 도구

**쉬운 비유**: 앱스토어. 필요한 도구(패키지)를 검색하고 설치합니다.

---

## O

### OKLCH

**한 줄 설명**: 최신 색상 표현 방식 (CSS Color Level 4)

**쉬운 비유**: 더 정확한 색상 레시피. RGB보다 사람 눈에 자연스러운 색상 변화를 표현합니다.

**이룸에서 사용**: `--module-skin: oklch(0.65 0.18 250)`

**구조**: `oklch(밝기 채도 색상각도)`

- 밝기: 0(검정)~1(흰색)
- 채도: 0(무채색)~0.4(선명)
- 색상: 0~360도 (색상환)

---

## P

### PostgreSQL

**한 줄 설명**: 오픈소스 관계형 데이터베이스

**쉬운 비유**: 엑셀의 프로 버전. 대용량 데이터를 안전하고 빠르게 관리합니다.

**이룸에서 사용**: Supabase가 PostgreSQL 기반

### Props (프롭스)

**한 줄 설명**: 컴포넌트에 전달하는 데이터

**쉬운 비유**: 함수의 재료. `<Button color="blue" />`에서 `color`가 props입니다.

---

## R

### RAG (Retrieval-Augmented Generation)

**한 줄 설명**: AI가 외부 데이터를 참고해서 답변하는 방식

**쉬운 비유**: 오픈북 시험. AI가 자기 기억만 쓰지 않고 참고 자료(제품 DB)를 보고 답합니다.

**이룸에서 사용**: 제품 Q&A에서 실제 제품 데이터 기반 응답

### React

**한 줄 설명**: 사용자 인터페이스를 만드는 JavaScript 라이브러리

**쉬운 비유**: 레고 시스템. 작은 블록(컴포넌트)을 조립해서 화면을 만듭니다.

### Repository Pattern (레포지토리 패턴)

**한 줄 설명**: 데이터 접근 코드를 한 곳에 모아두는 설계 방식

**쉬운 비유**: 도서관 사서. 책(데이터)을 직접 찾지 않고 사서(Repository)에게 요청합니다.

**이룸에서 사용**: `lib/api/`, `lib/products/repositories/`

### REST API

**한 줄 설명**: 웹에서 데이터를 주고받는 표준 방식

**쉬운 비유**: 우체국 시스템. 정해진 형식(URL, 메서드)으로 요청하면 응답을 받습니다.

### RLS (Row Level Security)

**한 줄 설명**: 데이터베이스에서 행(row) 단위로 접근을 제어하는 보안 기능

**쉬운 비유**: 아파트 열쇠. 내 집(내 데이터)만 열 수 있고, 남의 집(남의 데이터)은 못 엽니다.

**이룸에서 사용**: 모든 테이블에 `clerk_user_id` 기반 RLS 적용

### Route Group

**한 줄 설명**: URL에 영향 없이 폴더를 그룹화하는 방법

**쉬운 비유**: 서류 정리용 폴더. 실제 주소에는 안 나타나지만 파일을 정리할 수 있습니다.

**이룸에서 사용**: `(main)`, `(auth)` 등 괄호로 감싼 폴더

---

## S

### Server Component

**한 줄 설명**: 서버에서 실행되는 React 컴포넌트

**쉬운 비유**: 주방에서 미리 만들어 나오는 음식. 브라우저가 아닌 서버에서 HTML을 만듭니다.

→ 참고: Client Component

### SSR (Server-Side Rendering)

**한 줄 설명**: 서버에서 HTML을 만들어 보내는 방식

**쉬운 비유**: 포장 음식. 집(브라우저)에서 요리하지 않고, 가게(서버)에서 다 만들어 보냅니다.

### State (상태)

**한 줄 설명**: 컴포넌트가 기억하는 데이터

**쉬운 비유**: 메모장. 사용자가 입력한 내용, 버튼 클릭 여부 등을 기억합니다.

### Supabase

**한 줄 설명**: Firebase 대안 오픈소스 백엔드 서비스

**쉬운 비유**: 올인원 서버 키트. 데이터베이스, 인증, 저장소를 한 번에 제공합니다.

**이룸에서 사용**: 모든 데이터 저장, 사용자 인증 연동

---

## T

### Tailwind CSS

**한 줄 설명**: 미리 정의된 클래스로 스타일링하는 CSS 프레임워크

**쉬운 비유**: 옷 태그 스타일링. `bg-blue-500`(파란 배경), `text-lg`(큰 글씨) 같은 태그를 붙입니다.

### Token (토큰)

**한 줄 설명**: 인증 정보가 담긴 암호화된 문자열

**쉬운 비유**: 놀이공원 입장 팔찌. 한 번 받으면 계속 보여주며 입장할 수 있습니다.

→ 참고: Design Tokens (다른 의미)

### TypeScript

**한 줄 설명**: JavaScript에 타입 검사를 추가한 언어

**쉬운 비유**: 레시피의 재료 명시. "설탕 100g"처럼 변수의 타입을 명확히 지정합니다.

### Turbopack

**쉬운 비유**: 고속 컴파일러. 코드 변경 시 빠르게 반영합니다.

## U

### UI (User Interface)

**한 줄 설명**: 사용자가 보고 조작하는 화면

**쉬운 비유**: 자동차 계기판과 버튼들. 사용자가 직접 보고 만지는 모든 것입니다.

### UX (User Experience)

**한 줄 설명**: 사용자가 제품을 사용하면서 느끼는 경험

**쉬운 비유**: 자동차를 운전하는 느낌. 편한지, 직관적인지, 만족스러운지 등의 경험입니다.

## W

## 🏷️ 이룸 전용 용어

### PC-1 (Personal Color 1)

**의미**: 퍼스널 컬러 분석 모듈

**기능**: 얼굴/손목 사진으로 어울리는 색상(봄/여름/가을/겨울) 분석

### S-1 (Skin 1)

**의미**: 피부 분석 모듈

**기능**: 얼굴 사진으로 피부 상태(수분, 유분, 모공 등) 분석

### C-1 (Body/Contour 1)

**의미**: 체형 분석 모듈

**기능**: 전신 사진으로 체형 타입(직사각형, 모래시계 등) 분석

### F-1 (Face 1)

**의미**: 얼굴형 분석 모듈

**기능**: 얼굴 윤곽, 이목구비, 스타일 퍼스널리티 분석

### W-1 (Workout 1)

**의미**: 운동 모듈

**기능**: 운동 계획, 기록, 스트릭 관리

### N-1 (Nutrition 1)

**의미**: 영양 모듈

**기능**: 식단 기록, 영양 분석, 물 섭취 추적

### Monk Scale (몽크 스케일)

**의미**: Google이 2022년 발표한 피부톤 표준 (10단계)

**용도**: 다양한 인종의 피부톤을 공정하게 표현

**범위**: monk_01(가장 밝음) ~ monk_10(가장 어두움)

### Cross-module (크로스 모듈)

**의미**: 분석 모듈 간 데이터 연동

**예시**: PC-1 결과를 S-1에서 드레이핑 탭으로 활용

### Hybrid Pattern (하이브리드 패턴)

**의미**: DB 핵심 데이터 + Mock 표시 데이터를 조합하는 전략

**장점**: Mock 수정만으로 모든 사용자에게 개선 사항 적용
