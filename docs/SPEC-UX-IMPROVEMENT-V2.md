# 이룸 UX 개선 스펙 V2

> 버전: 2.0 | 작성일: 2026-01-02 | 상태: 검토 중
> 기반: 경쟁사 분석 (잼페이스, 화해, 닥터지, 페이스보드)

---

## 1. 개요

### 1.1 배경

현재 이룸 앱/웹의 UX를 경쟁사와 비교 분석한 결과:

| 영역           | 현재 상태                   | 경쟁사 벤치마크                    |
| -------------- | --------------------------- | ---------------------------------- |
| 온보딩         | 운동 7단계, 영양 7단계      | 잼페이스: 사진 1장 → 즉시 결과     |
| 분석 결과      | 텍스트 설명 위주            | 닮은 유튜버 매칭, 제품 즉시 연결   |
| 제품 연결      | 별도 페이지 이동 필요       | 결과 화면에서 바로 스와이프        |
| 설명 품질      | "O형: 풍만한 실루엣" (모호) | 자가진단 팁, 구체적 특징 ✅ 개선됨 |
| 게이미피케이션 | streak 숫자만 표시          | 애니메이션, 햅틱, 공유 카드        |

### 1.2 목표

1. **전환율 향상**: 분석 완료 → 제품 클릭률 +30%
2. **온보딩 완료율**: 70% → 90%
3. **일일 활성 사용자(DAU)**: 리텐션 +20%
4. **사용자 만족도**: NPS 점수 +15

### 1.3 경쟁사 상세 분석

#### 잼페이스 (ZamFace)

- **강점**: AR 퍼스널컬러 진단, 닮은 유튜버 매칭, 화장품 호환성 체크
- **MAU**: 280만+ 퍼컬 매칭 이용
- **UX 특징**: 사진 1장으로 즉시 결과, 결과에서 바로 제품 연결
- **참고**: https://zamface.fun/

#### 화해 (Hwahae)

- **강점**: 9억 리뷰 DB, 피부타입별 랭킹, 성분 분석
- **MAU**: 200만+ 설치
- **UX 교훈**: "너무 많은 이벤트 문구로 시선 분산" - 정보 밀도 최적화 필요
- **참고**: https://www.hwahae.com/

#### 닥터지 Ai옵티미

- **강점**: 50만 피부 데이터, OCR 성분 분석, 96.5% 만족도
- **UX 특징**: "3분 진단" 강조, 매칭률 수치로 표시
- **누적**: 10만+ 이용

#### 페이스보드 (FaceBoard)

- **강점**: 특허 시스템, 높은 정확도, 무료 진단
- **UX 특징**: 촬영 → 4계절 분류 즉시 결과

## 2. 개선 항목 상세

### 2.1 P0: 분석 결과 → 제품 바로 연결

#### 2.1.1 현재 문제

```
[분석 결과 페이지]
   ↓ 스크롤
[추천 아이템 텍스트]
   ↓ 별도 버튼 클릭
[제품 페이지 이동]
   ↓ 카테고리 선택
[제품 목록]
```

- 4단계 이동 필요
- 이탈률 높음

#### 2.1.2 개선안

```
[분석 결과 페이지]
   ↓ 스와이프/스크롤
[맞춤 추천 제품 카드 3개] ← 바로 표시
   ↓ 탭
[제품 상세 모달/페이지]
```

- 2단계로 축소
- 결과 페이지에서 바로 제품 확인

#### 2.1.3 UI 설계

```
┌─────────────────────────────────────────┐
│  🎨 당신은 O자형이에요!                  │
│  부드러운 곡선과 풍성한 매력             │
├─────────────────────────────────────────┤
│                                         │
│  📦 이 체형에 딱 맞는 아이템             │
│  ┌───────┐ ┌───────┐ ┌───────┐         │
│  │ 제품1 │ │ 제품2 │ │ 제품3 │  →      │
│  │ V넥   │ │ A라인 │ │ 롱카디│         │
│  │₩29,000│ │₩45,000│ │₩39,000│         │
│  └───────┘ └───────┘ └───────┘         │
│                                         │
│  [전체 추천 제품 보기]                   │
└─────────────────────────────────────────┘
```

#### 2.1.4 구현 범위

| 분석 타입  | 연결 제품   | 필터 조건               |
| ---------- | ----------- | ----------------------- |
| 퍼스널컬러 | 화장품      | personalColorSeasons    |
| 피부 분석  | 스킨케어    | skinTypes, concerns     |
| 체형 분석  | 패션 아이템 | 체형별 추천 아이템 매핑 |

#### 2.1.5 기술 구현

### 2.2 P0: 온보딩 단계 축소

#### 2.2.1 현재 상태

| 모듈        | 단계 수 | 예상 시간 |
| ----------- | ------- | --------- |
| 운동 온보딩 | 7단계   | 3-5분     |
| 영양 온보딩 | 7단계   | 3-5분     |
| PC-1 분석   | 3단계   | 1-2분     |
| S-1 분석    | 3단계   | 1-2분     |
| C-1 분석    | 3단계   | 1-2분     |

**문제**: 운동/영양 온보딩이 너무 길어 이탈률 높음

#### 2.2.2 개선안

**Phase 1: 필수 질문만 (3단계)**

```
Step 1: 목표 선택 (다이어트/근력/유지/체력)
Step 2: 운동 경험 (초보/중급/고급)
Step 3: 선호 운동 (유산소/근력/요가/복합)
→ 즉시 플랜 생성
```

**Phase 2: 선택적 상세 설정**

- 프로필 페이지에서 추가 정보 입력 가능
- 기록 축적 후 AI가 자동 조정

#### 2.2.3 질문 우선순위

| 현재 질문 | 필수 | 선택 | 삭제           |
| --------- | ---- | ---- | -------------- |
| 운동 목표 | ✅   |      |                |
| 운동 경험 | ✅   |      |                |
| 선호 운동 | ✅   |      |                |
| 가용 시간 |      | ✅   |                |
| 운동 장소 |      | ✅   |                |
| 부상 이력 |      | ✅   |                |
| 장비 보유 |      |      | ✅ (자동 추론) |

- 공유 카드: `html2canvas` + Web Share API

- 햅틱: `expo-haptics`
- 애니메이션: `lottie-react-native` + `react-native-reanimated`

- 이미지 저장: Supabase Storage
