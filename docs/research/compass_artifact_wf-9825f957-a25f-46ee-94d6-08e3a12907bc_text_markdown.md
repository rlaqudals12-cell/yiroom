# 이룸(Yiroom) 개인정보보호법 준수 가이드

한국 뷰티테크 스타트업 이룸이 AI 기반 퍼스널 컬러 분석 서비스를 운영함에 있어 **개인정보보호법 준수가 핵심 법적 리스크 관리 요소**입니다. 얼굴 이미지 수집 시 그 처리 목적에 따라 일반 개인정보 또는 민감정보(생체인식정보)로 분류되며, 민감정보 해당 시 **별도 동의**가 필수입니다. Google Gemini 등 해외 AI API 사용 시에는 **국외이전 동의 또는 처리방침 공개**가 요구됩니다. 위반 시 매출액의 **3% 이하 과징금**, **5년 이하 징역** 등 중대한 제재가 부과될 수 있어 철저한 준비가 필요합니다.

---

## 1. 얼굴 이미지의 법적 분류와 수집 요건

### 생체정보와 생체인식정보의 핵심 구분

개인정보보호법 시행령 제18조 제3호는 민감정보로서 생체인식정보를 **"개인의 신체적, 생리적, 행동적 특징에 관한 정보로서 특정 개인을 알아볼 목적으로 일정한 기술적 수단을 통해 생성한 정보"**로 정의합니다.

개인정보보호위원회의 2021년 「생체정보 보호 가이드라인」에 따르면, 얼굴 이미지의 법적 분류는 **처리 목적**에 따라 달라집니다:

| 구분 | 정의 | 법적 지위 | 이룸 서비스 해당 |
|------|------|----------|-----------------|
| **생체인식정보** | 특정 개인을 **인증·식별할 목적**으로 처리되는 정보 | **민감정보** (별도 동의) | 얼굴 인식 로그인 시 |
| **일반적 생체정보** | 연령·성별·피부톤 등 **개인 특징 파악 목적**으로 처리 | 일반 개인정보 | 퍼스널 컬러 분석 시 |

**퍼스널 컬러 분석의 경우**, 피부톤·눈동자색 등 색상 특성 분석이 목적이므로 원칙적으로 일반 개인정보로 분류됩니다. 그러나 **AI가 얼굴 특징점을 추출·분석하는 기술적 수단을 사용**한다면 민감정보(생체인식정보)에 해당할 가능성이 있습니다.

> ⚠️ **실무 권고**: 서비스 특성상 법적 분류가 불명확하므로, **민감정보 처리 동의를 별도로 획득**하는 것이 안전합니다.

### 수집 단계별 법적 요건

**일반 개인정보로 처리 시** (개인정보보호법 제15조):
- 수집·이용 목적, 수집 항목, 보유기간, 동의 거부권 및 불이익 내용 고지
- 일반 동의 획득

**민감정보(생체인식정보)로 처리 시** (제23조):
- 다른 개인정보 처리 동의와 **별도로 동의** 획득 필수
- 전송·저장 시 **암호화 의무**
- 위반 시 **5년 이하 징역 또는 5천만원 이하 벌금**

### 저장 및 파기 의무

개인정보보호법 제21조에 따라 보유기간 경과 또는 처리 목적 달성 시 **지체 없이 파기**해야 합니다.

| 처리 유형 | 권장 파기 시점 | 파기 방법 |
|----------|--------------|----------|
| 실시간 분석 후 미저장 | 분석 즉시 자동 삭제 | 최선의 방법 |
| 일시 저장 후 분석 | 분석 완료 즉시 (5일 이내) | 복원 불가능하게 삭제 |
| AI 학습용 활용 | 별도 동의받은 기간 경과 시 | 덮어쓰기 3회 이상 권고 |

파기 의무 위반 시 **3천만원 이하 과태료**가 부과됩니다.

## 2. 개인정보처리방침 필수 기재 사항

### 제30조 법정 필수 항목

개인정보보호법 제30조 및 시행령 제31조에 따른 **필수 기재 사항**:

1. **개인정보의 처리 목적** - AI 퍼스널 컬러 분석 서비스 제공 명시
2. **처리 및 보유 기간** - 이미지 즉시 파기/결과 보존 기간 구분
3. **처리하는 개인정보 항목** - 얼굴 이미지, 분석 결과 등
4. **파기절차 및 방법** - 즉시 삭제, 복구 불가능 방법
5. **정보주체 권리·의무 및 행사방법** - 열람·정정·삭제·처리정지 요구
6. **개인정보 보호책임자** - 성명, 부서명, 연락처
7. **제3자 제공 사항** (해당 시)
8. **처리 위탁 사항** (해당 시)
9. **국외이전 사항** (AI API 사용 시 필수)
10. **안전성 확보조치**

### AI 프로파일링 고지 의무 (제37조의2, 2024.3.15 시행)

이룸의 AI 퍼스널 컬러 분석이 사용자에게 제품 추천 등 **의사결정에 영향**을 미친다면 다음을 공개해야 합니다:

| 공개 사항 | 설명 | 샘플 문구 |
|-----------|------|----------|
| **결정의 기준** | 어떤 기준으로 결정되는지 | "피부톤, 눈동자색, 머리카락색을 분석하여 컬러 타입 결정" |
| **결정의 절차** | 결정 과정 | "이미지 업로드 → AI 특징 추출 → 타입 분류 → 결과 제공" |
| **처리 방식** | 데이터 처리 방법 | "이미지는 분석 후 즉시 파기, 결과만 저장" |

**정보주체 권리**: 설명 요구권, 거부권, 검토 요구권을 안내해야 하며, 위반 시 **5백만원 이하 과태료**가 부과됩니다.

### 게시 위치 및 방법 (시행령 제31조 제2항)

- **웹사이트**: 첫 화면 또는 연결화면에 지속 게재
- **모바일 앱**: 첫 화면, '설정', '회원가입' 등 확인 용이 위치
- **명칭**: 반드시 **"개인정보 처리방침"** 사용
- **구분**: 글자 크기, 색상 등으로 다른 고지사항과 구분
- **변경 이력**: 이전 버전 확인 가능하도록 공개

미공개 또는 변경 미공개 시 **1천만원 이하 과태료**입니다.

## 3. 동의 획득 UX 법적 요건

### 동의의 유효 요건 (시행령 제17조)

개인정보보호법 제22조 및 시행령 제17조에 따라 동의는 다음 조건을 **모두 충족**해야 합니다:

1. 정보주체가 **자유로운 의사**에 따라 동의 여부 결정
2. 동의 내용이 **구체적이고 명확**
3. **쉽게 읽고 이해할 수 있는 문구** 사용
4. **동의 여부를 명확하게 표시**할 수 있는 방법 제공

### 필수/선택 동의 구분

| 구분 | 동의 항목 | 서비스 거부 가능 여부 |
|------|----------|---------------------|
| **필수** | 서비스 제공에 필수적인 최소한의 정보 | 거부 시 서비스 이용 불가 |
| **필수(별도)** | 민감정보(얼굴 이미지) 수집 | 거부 시 해당 기능 이용 불가 |
| **선택** | 마케팅, 광고성 정보 수신 | **거부해도 서비스 제공 의무** (제22조 제5항) |

선택 동의 거부를 이유로 서비스 제공 거부 시 **3천만원 이하 과태료**입니다.

### UI/UX 시각적 요건

| 요건 | 세부 기준 | 법적 근거 |
|------|----------|----------|
| **글자 크기** | 최소 **9포인트 이상**, 중요 내용은 **20% 이상 크게** | 시행령 제17조 |
| **강조 방법** | **굵은 글씨**, **다른 색상**, **밑줄** 등 사용 | 시행령 제17조 |
| **체크박스 기본값** | 반드시 **미체크(unchecked)** 상태 | 시행령 제17조 제4항 |
| **약관 전문 확인** | 클릭/탭으로 전문 열람 가능 기능 필수 | - |

위반 시 **1천만원 이하 과태료**가 부과됩니다.

### 민감정보 별도 동의의 형식

개인정보보호법 제23조 제1항에 따라 민감정보(생체인식정보) 처리 시 **"다른 개인정보의 처리에 대한 동의와 별도로"** 동의를 받아야 합니다. 이는 **물리적으로 분리된 별도의 체크박스**를 의미합니다.

### 동의 철회 요건

제39조의7에 따라 동의 철회는 **"개인정보 수집방법보다 쉽게"** 제공해야 합니다:

- 앱 내 설정 메뉴에서 쉽게 접근 가능한 철회 기능
- 개별 동의 항목별 철회 기능
- 철회 시 지체 없이 **복구·재생 불가능하게 파기**

위반 시 **3천만원 이하 과태료**입니다.

### 미성년자 법정대리인 동의 (제22조의2)

**만 14세 미만 아동**의 개인정보 수집 시 **법정대리인의 동의** 필수:

| 확인 방법 | 설명 |
|-----------|------|
| 휴대전화 본인인증 | 법정대리인 휴대전화로 본인인증 |
| 신용카드/직불카드 | 법정대리인 카드정보 제공 |
| 서면 동의 | 서명날인 후 제출 |
| 전자우편 | 동의 의사표시 전자우편 수신 |

미준수 시 **5년 이하 징역 또는 5천만원 이하 벌금**, **매출액 3% 이하 과징금**입니다.

## 4. 개인정보 국외이전 (AI API 사용 시)

### Google Gemini 등 해외 AI API 사용 = 국외이전

개인정보보호법 제28조의8에 따라 개인정보의 국외 **제공·처리위탁·보관**은 원칙적으로 금지되며, 예외 사유 충족 시에만 허용됩니다.

Google Gemini API로 얼굴 이미지를 전송하면 **국외 처리위탁**에 해당합니다. 미국은 현재 한국의 **적정성 결정국으로 지정되지 않아** 별도 조치가 필요합니다.

### 국외이전 허용 방법

| 방법 | 요건 | 이룸 적용 |
|------|------|----------|
| **방법 1: 별도 동의** | 법정 고지사항 5가지 고지 후 동의 | 권장 |
| **방법 2: 계약 이행 목적** | 개인정보 처리방침에 공개 또는 정보주체에게 통지 | 가능 |
| **방법 3: 인증받은 자** | 개인정보보호위원회 지정 인증 취득 | Google 미취득 |

### 국외이전 동의 시 법정 고지사항 (제28조의8 제2항)

1. 이전되는 **개인정보 항목** (얼굴 이미지)
2. 개인정보가 **이전되는 국가** (미국 등), 이전일시 및 방법
3. 개인정보를 **이전받는 자의 명칭과 연락처** (Google LLC)
4. 이전받는 자의 **이용목적 및 보유·이용 기간**
5. 개인정보 이전을 **거부하는 방법, 절차 및 효과**

> ⚠️ **주의**: 고지사항을 충실히 알리지 않고 형식적 동의만 받은 경우 **과징금 부과 대상**입니다 (개인정보보호위원회 2024년 결정례).

### AI API 사용 시 필수 확인 사항

- API 호출 시 데이터 **처리 서버 위치** 확인
- 데이터 **보관 여부** 및 보관 기간
- **AI 모델 학습에 활용 여부** 확인
- **DPA(Data Processing Agreement)** 체결
- AI 학습 목적 활용 **금지 조항** 확인

## 5. 실무 준수 체크리스트

### 개인정보 보호책임자(CPO) 지정

| 기업 규모 | 지정 기준 |
|-----------|----------|
| 5인 미만(소상공인) | 대표자가 자동 지정 |
| 일반 기업 | 임원 또는 개인정보 담당 부서장 |
| 대규모 기업* | 전문 CPO 요건 충족 필요 (2026.3.14까지 유예) |

*전문 CPO 의무 대상: 연간 매출 1,500억원 이상 + 5만명 이상 민감정보 또는 100만명 이상 정보주체

미지정 시 **1천만원 이하 과태료**입니다.

### 내부 관리계획 수립 의무

**정보통신서비스 제공자(앱 운영)는 규모와 관계없이 반드시 수립** 필요:

- 개인정보취급자 관리·감독 및 교육
- 개인정보 보호책임자 지정 및 조직 운영
- 접근권한 부여·변경·말소 기준
- 접근통제 시스템 운영
- 접속기록 보관·점검 (최소 1년, 민감정보 5만명 이상 시 2년)
- 개인정보 암호화
- 악성프로그램 방지
- 물리적 안전조치

### 개인정보 영향평가

법 제33조에 따라 **공공기관**은 의무, **민간기업은 권고사항**입니다.

그러나 이룸처럼 얼굴 이미지(생체정보) 대량 처리, 해외 AI API 연동 시에는 **자발적 영향평가 수행을 권장**합니다.
