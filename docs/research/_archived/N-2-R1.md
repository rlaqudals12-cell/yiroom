# N-2-R1: 미성년자 차단 UI/UX 및 법적 고지 종합 리서치

## 1. 핵심 요약

이룸(Yiroom) 서비스가 만 14세 미만 이용을 제한하려면 **개인정보보호법 제22조의2**에 따른 연령 확인 절차가 필수이며, 이를 위반 시 **5년 이하 징역 또는 5천만원 이하 벌금**, 매출액 **3% 이하 과징금**이 부과될 수 있다. 연령 확인 방법으로는 생년월일 입력(최소 요건), 휴대폰 본인인증(권장), 또는 "만 14세 이상입니다" 체크박스가 있으며, 얼굴 이미지 분석 서비스 특성상 **AI 기본법(2026.1.22 시행) 제31조**에 따라 AI 사용 사실 고지도 필수다. 차단 페이지에는 법적 근거, 차단 사유, 법정대리인 동의 안내, 고객센터 정보를 반드시 포함해야 하며, 우회 방지를 위해 Device Fingerprinting과 서버 세션 관리를 병행해야 한다.

---

## 2. 상세 내용

### 2.1 법적 고지 의무 문구

개인정보보호법 제22조의2는 만 14세 미만 아동의 개인정보 처리 시 **법정대리인 동의**를 의무화한다. 서비스가 만 14세 미만 이용을 완전히 차단하는 경우 법정대리인 동의 절차 자체를 생략할 수 있으나, 이 경우에도 **연령 확인 절차**와 **이용 제한 사실 고지**는 필수적으로 구현해야 한다.

**법적 고지 문구의 요건**은 개인정보보호법 제22조의2 제3항에서 규정하며, 만 14세 미만 아동에게 고지할 때는 "**이해하기 쉬운 양식과 명확하고 알기 쉬운 언어**"를 사용해야 한다. 개인정보보호위원회의 아동·청소년 개인정보보호 가이드라인(2022.7)은 복잡한 법률 용어 대신 쉬운 표현 사용과 시각적 이해도를 높이는 양식 활용을 권장한다.

정보통신서비스 제공자에게 적용되는 개인정보보호법 제39조의3 제4항 역시 동일하게 만 14세 미만 아동의 개인정보 수집 시 법정대리인 동의와 동의 확인 절차를 의무화한다. 법정대리인 동의 확인 방법은 시행령 제17조의2에서 **6가지 방법**(휴대폰 문자 확인, 카드정보 제공, 휴대폰 본인인증, 서면 동의, 이메일, 전화)을 규정하고 있다.

**위반 시 제재**는 상당히 엄격하다. 법정대리인 동의 없이 만 14세 미만 아동의 개인정보를 처리하면 개인정보보호법 제71조 제3호에 따라 **5년 이하 징역 또는 5천만원 이하 벌금**에 처해지며, 제64조의2에 따라 전체 매출액의 **3% 이하 과징금**이 부과될 수 있다. 법정대리인 동의 확인 절차 미이행은 **1,000만원~1,500만원 과태료** 대상이다.

### 2.2 차단 페이지 필수 요소

차단 페이지는 법적 의무 준수와 사용자 경험을 동시에 고려해야 한다. 필수 요소로는 **차단 사유 명시**(만 14세 미만 이용 불가 이유), **법적 근거 표시**(개인정보보호법 제22조의2), **대안 제시**(법정대리인 동의 절차 안내 또는 홈페이지 링크), **고객센터 정보**(전화번호, 이메일, 운영시간)가 포함되어야 한다.

**웹접근성 요건(KWCAG 2.2)**도 충족해야 한다. 핵심 항목으로는 모든 이미지에 적절한 대체 텍스트(alt) 제공, 텍스트와 배경의 명도 대비 **4.5:1 이상** 유지, 키보드만으로 모든 기능 접근 가능, 스크린 리더 호환 마크업(ARIA 라벨), 기본 언어 명시(`lang="ko"`)가 있다. 차단 페이지의 핵심 메시지는 `role="alert"` 또는 `aria-live="polite"`로 스크린 리더에 알림 처리해야 한다.

틱톡과 인스타그램의 사례를 참고하면, 만 14세 미만 입력 시 **다른 생년월일로 재가입을 차단**하는 기술적 조치가 효과적이다. 네이버와 카카오는 만 14세 미만 확인 시 자동으로 **법정대리인 동의 절차로 전환**하는 방식을 사용하며, 이룸 서비스도 법정대리인 동의 옵션 제공 여부에 따라 유사한 플로우를 설계할 수 있다.

### 2.3 데이터 처리 방법

부주의로 수집된 미성년자 데이터는 **표준 개인정보보호지침 제13조**에 따라 처리해야 한다. 법정대리인 동의가 거부되었거나 동의 의사를 확인할 수 없는 경우, 수집일로부터 **5일 이내 파기**해야 한다. 개인정보가 불필요해진 경우에는 개인정보보호법 제21조에 따라 **지체 없이** 파기해야 한다.

파기 방법은 시행령 제16조에서 규정한다. 전자적 파일은 **복원 불가능한 방법**으로 영구 삭제해야 하며, 서면 기록물은 파쇄 또는 소각해야 한다. 삭제 완료 통지에 대한 명시적 법적 의무는 없으나, 정보주체(법정대리인 포함)가 요청 시 삭제 사실을 확인해줄 수 있도록 삭제 로그를 보관하는 것이 권장된다.

**면책 가능 사유**도 있다. 아동이 허위로 연령을 기재한 "기망 동의"의 경우, 사업자가 합리적인 연령 확인 조치(생년월일 입력, 체크박스, 본인인증 등)를 취했다면 면책 사유가 될 수 있다. 다만 감독기관 조사 시 사실 확인서 제출이 필요할 수 있어, 연령 확인 절차 기록을 보관해야 한다.

법정대리인 동의를 받은 경우 동의 이력(동의서, 이메일, 녹취 등)은 최소한의 정보만 회원 탈퇴 시까지 보유할 수 있다. 별도의 감독기관 신고 의무는 없으나, 개인정보 유출 사고 발생 시에는 개인정보보호법 제34조에 따라 정보주체 통지 및 개인정보보호위원회 신고가 필수다.

### 2.4 연령 재확인 플로우

연령 확인 방법은 신뢰도, 비용, 사용자 경험을 종합적으로 고려해 선택해야 한다. **생년월일 직접 입력**은 비용이 들지 않고 구현이 간단하지만 허위 입력이 가능하여 법적 효력이 약하다. **휴대폰 본인인증(PASS/통신사)**은 신뢰도가 높고 만 14세 미만을 정확히 확인할 수 있으나 건당 **300~500원** 또는 월정액 비용이 발생한다. **간편인증(카카오/네이버/PASS앱)**은 사용자 친화적이며 건당 **200~450원** 수준이다.

연령 오입력 시 재입력 정책은 우회 방지와 사용자 편의 사이에서 균형을 잡아야 한다. 권장되는 방식은 **3~5회 재시도 제한** 후 **15~30분 대기 시간** 부여이며, 실패 횟수에 따라 점진적으로 대기시간을 증가시키는 exponential backoff 방식이 효과적이다. 틱톡은 만 14세 미만 입력 시 해당 기기에서 다른 생년월일로 재가입 자체를 차단한다.

우회 방지를 위한 기술적 조치로는 **Device Fingerprinting**(브라우저, OS, 화면 해상도, 폰트 등 조합으로 기기 고유 식별), **서버 세션 기반 추적**, **IP 기반 Rate Limiting**, **쿠키 정보 저장**(HttpOnly, Secure, SameSite=Strict 플래그 필수)이 있다. 단일 신호(쿠키만, IP만) 변경으로는 우회가 불가능하도록 **다중 신호 검증**을 구현해야 한다.

세션 및 데이터 처리는 보안을 최우선으로 해야 한다. 연령 인증 정보는 **서버 세션에 저장**하고 클라이언트에는 세션 ID만 HttpOnly 쿠키로 전달하는 방식을 권장한다. 쿠키 설정은 `Set-Cookie: __Host-AGE_VERIFIED=true; Path=/; Secure; HttpOnly; SameSite=Strict; Max-Age=86400` 형태가 이상적이며, 세션 유효기간은 보안상 **24시간 이내**로 설정해야 한다.

### 2.5 국내 서비스 사례

**네이버**는 생년월일 입력 후 만 14세 미만 여부를 자동 판별하고, 미만인 경우 **법정대리인 본인인증**을 필수로 요구한다. 보호자의 이름, 생년월일, 휴대전화 번호 입력 후 휴대폰 인증 또는 아이핀 인증을 진행하며, 보호자 본인인증 정보가 성인이 아니면 14세 미만 회원가입 자체가 불가하다.

**카카오**는 카카오계정 가입 시 생년월일 입력으로 연령을 필터링하고, 카카오싱크 연동 서비스에서 "만 14세 이상 연령 동의" 약관을 추가할 수 있다. 만 14세 미만 사용자가 동의 시도 시 보호자 휴대폰 본인인증 프로세스가 자동 진행되며, 보호자 동의 미완료 시 "access_denied Not allowed under age 14" 에러를 반환한다. 법정대리인과 아동의 나이 차이가 **만 18세 이상**이어야 하는 검증도 포함된다.

**게임사(넥슨, 넷마블)**는 휴대폰 본인인증 또는 아이핀 인증을 사용하며, **1일 3회** 인증 실패 시 당일 서비스 이용이 불가하다. 2022년 1월 강제적 셧다운제 폐지 이후에는 게임시간 선택제로 전환되어, 만 18세 미만 청소년 또는 법정대리인 요청 시 게임 이용 시간 제한을 설정할 수 있다.

**틱톡 한국**은 글로벌 기준(13세)과 달리 한국에서는 **만 14세 이상**만 가입 가능하며, 가입 시 생년월일 전체를 직접 입력해야 한다. 만 14세 미만 입력 시 다른 생년월일 사용 재가입이 차단되며, 안전 심사팀이 키워드 및 커뮤니티 신고로 미성년자 의심 계정을 파악하여 정지 처리한다.

**인스타그램/메타 한국**은 2025년 1월 22일부터 '10대 계정(Teen Account)'을 국내에 적용하여 만 14~18세 계정의 기본 설정을 비공개로 하고, **60분** 초과 시 앱 닫기 알림, 오후 10시~오전 7시 알림 해제 등의 보호 기능을 제공한다. 만 14세 미만으로 잘못 입력 시 계정이 차단되며, 30일 이내 연령 확인 못하면 계정이 비활성화된다.

## 3. 구현 시 필수 사항

### 법적 필수 요건
- [ ] **연령 확인 절차 구현**: 생년월일 입력, "만 14세 이상입니다" 체크박스, 또는 휴대폰 본인인증 중 최소 1개
- [ ] **이용약관에 연령 제한 명시**: "본 서비스는 만 14세 이상만 이용 가능합니다" 문구 포함
- [ ] **개인정보처리방침에 아동 정책 명시**: 만 14세 미만 개인정보 수집 제한 사항 기재
- [ ] **AI 사용 사실 고지**: AI 기본법 제31조에 따라 "본 서비스는 AI 기반 얼굴 분석 기술을 사용합니다" 고지 (위반 시 3,000만원 이하 과태료)
- [ ] **얼굴 이미지 처리 동의**: 개인정보보호법 제15조에 따른 수집 목적, 항목, 보유기간, 동의 거부권 고지

### 차단 페이지 필수 요소
- [ ] **차단 사유 명확한 안내**: "만 14세 미만은 서비스 이용이 제한됩니다"
- [ ] **법적 근거 명시**: 「개인정보 보호법」 제22조의2 조항 표시
- [ ] **법정대리인 동의 절차 안내**: (제공 시) 보호자 동의 방법 설명 및 링크
- [ ] **고객센터 연락처**: 전화번호, 이메일, 운영시간 표시
- [ ] **웹접근성 준수**: KWCAG 2.2 기준 충족 (명도 대비 4.5:1, 키보드 접근성, 스크린 리더 호환)

### 기술적 필수 요건
- [ ] **우회 방지 장치**: Device Fingerprinting + 서버 세션 + 쿠키 조합
- [ ] **재시도 제한**: 연령 확인 실패 시 3~5회 제한, 15~30분 대기시간 적용
- [ ] **보안 쿠키 설정**: HttpOnly, Secure, SameSite=Strict 플래그 적용
- [ ] **얼굴 이미지 보호**: 암호화 저장, 분석 완료 후 즉시 삭제 정책

### 데이터 처리 필수 요건
- [ ] **미성년자 데이터 즉시 삭제**: 부주의로 수집 시 5일 이내 파기
- [ ] **삭제 로그 보관**: 삭제 사실 확인 요청 대응용
- [ ] **연령 확인 기록 보관**: 기망 동의 면책 입증용

export const AgeVerification: React.FC<AgeVerificationProps> = ({
  onVerified,
  onBlocked
}) => {
  const [birthYear, setBirthYear] = useState<number | null>(null);
  const [birthMonth, setBirthMonth] = useState<number | null>(null);
  const [birthDay, setBirthDay] = useState<number | null>(null);
  const [attempts, setAttempts] = useState(0);
  const MAX_ATTEMPTS = 3;

  const calculateAge = (): number => {
    if (!birthYear || !birthMonth || !birthDay) return 0;
    const today = new Date();
    const birthDate = new Date(birthYear, birthMonth - 1, birthDay);
    let age = today.getFullYear() - birthDate.getFullYear();
    const monthDiff = today.getMonth() - birthDate.getMonth();
    if (monthDiff < 0 || (monthDiff === 0 && today.getDate() < birthDate.getDate())) {
      age--;
    }
    return age;
  };

  const handleSubmit = async () => {
    if (attempts >= MAX_ATTEMPTS) {
      // 15분 대기 후 재시도 가능
      return;
    }

    const age = calculateAge();
    
    if (age >= 14) {
      // 서버에 인증 결과 저장
      await fetch('/api/age-verify', {
        method: 'POST',
        body: JSON.stringify({ verified: true, age }),
        headers: { 'Content-Type': 'application/json' }
      });
      onVerified();
    } else {
      setAttempts(prev => prev + 1);
      // Device fingerprint 저장
      await fetch('/api/age-verify', {
        method: 'POST',
        body: JSON.stringify({ verified: false, blocked: true }),
        headers: { 'Content-Type': 'application/json' }
      });
      onBlocked();
    }
  };

      {/* ... 생년월일 입력 필드 */}

  );
};
