# claude.ai 리서치 요청 템플릿

> 이 템플릿을 claude.ai에 복사하여 딥 리서치 요청 시 사용

---

## 표준 형식

```
[티켓ID] 리서치 제목
────────────────────────

리서치 질문:
1. [구체적 질문]
2. [비교/검토 항목]
3. [기술적 제약 사항]

참고 맥락:
- 프로젝트: 이룸 (한국어 웰니스 앱)
- 기술 스택: Next.js 16, React 19, Supabase, Gemini 3 Flash
- 제약: 클라이언트 번들 500KB 미만, 응답 3초 이내

예상 출력:
- 알고리즘/라이브러리 비교표
- 권장 선택 및 근거
- 구현 코드 스니펫 (TypeScript)
```

---

## 예시: CIE-1-R1

```
[CIE-1-R1] 이미지 품질 평가 알고리즘
────────────────────────────────────

리서치 질문:
1. Gemini Vision API 없이 클라이언트/서버에서 이미지 sharpness를 측정하는 방법은?
   - Laplacian variance vs Sobel gradient 비교
2. 조명 색온도(color temperature) 추정 알고리즘은?
   - Von Kries, CAT02 등 비교
3. 얼굴 메이크업 감지에 사용되는 feature는?
   - skin texture variance, lip color saturation 등

참고 맥락:
- 프로젝트: 이룸 (한국어 웰니스 앱)
- 기술 스택: Next.js 16, React 19, TypeScript
- 제약: 
  - 클라이언트 측 처리 가능해야 함 (Canvas API)
  - 서버 측 대안도 검토 (Sharp 라이브러리)
  - 응답 200ms 이내

예상 출력:
- 각 알고리즘의 정확도/속도 비교표
- 클라이언트(Canvas API) vs 서버(Sharp) 구현 차이
- 참고 논문/라이브러리 링크
- TypeScript 구현 코드 스니펫
```

## 예시: CIE-2-R1

```
[CIE-2-R1] 얼굴 랜드마크 라이브러리 선정
────────────────────────────────────────

리서치 질문:
1. 브라우저에서 실행 가능한 얼굴 랜드마크 라이브러리 비교:
   - face-api.js (TensorFlow.js 기반)
   - mediapipe/face_mesh (Google)
   - @vladmandic/face-api (face-api.js 포크)
2. 각 라이브러리의 정확도/속도/번들 크기 비교
3. 68점 vs 478점(mediapipe) 랜드마크의 trade-off
4. React/Next.js SSR 환경에서의 호환성 이슈

참고 맥락:
- 프로젝트: 이룸 (한국어 웰니스 앱)
- 기술 스택: Next.js 16, React 19
- 용도: 피부존 추출, 혈관 ROI 추출
- 제약:
  - 번들 크기 < 5MB (모델 포함)
  - 초기화 시간 < 3초
  - 추론 시간 < 500ms

예상 출력:
- 라이브러리 비교표 (정확도, 속도, 크기, 라이선스)
- 이룸 요구사항에 최적 선택 및 근거
- 설치/초기화 코드 스니펫
```

## 예시: CIE-3-R1

```
[CIE-3-R1] 피부톤 측정을 위한 Lab 색공간 분석
─────────────────────────────────────────────

리서치 질문:
1. 피부톤 측정에 Lab 색공간이 RGB/HSV보다 적합한 이유는?
2. 아시아인 피부톤의 Lab a* 범위는? (문헌 기반)
3. Lab a* 값으로 웜톤/쿨톤 판정 시 임계값은?
   - PC-1 현재: 혈관 blue → cool, green → warm (AI 판정)
   - 개선안: Lab a* > X → warm, Lab a* < Y → cool (수치 기반)
4. 조명 색온도 변화에 따른 Lab a* 오차 범위는?

참고 맥락:
- 프로젝트: 이룸 퍼스널컬러 분석
- 현재 방식: Gemini AI가 텍스트로 판정
- 개선 목표: 수치 기반 판정으로 일관성 확보
- 제약: 조명 보정 후에도 ±3 Lab a* 오차 허용

예상 출력:
- 아시아인 피부톤 Lab 분포 데이터 (논문 인용)
- 웜톤/쿨톤 Lab a* 임계값 제안
- 조명 보정 후 Lab a* 안정성 검증 방법
- RGB ↔ Lab 변환 TypeScript 코드
```

## 사용 팁

1. **구체적으로 질문**: 모호한 질문은 모호한 답변을 유발
2. **제약 조건 명시**: 성능, 크기, 라이선스 등 명확히
3. **예상 출력 명시**: 비교표, 코드, 벤치마크 등 원하는 형식
4. **맥락 제공**: 프로젝트 특성, 기존 구현 방식 포함
