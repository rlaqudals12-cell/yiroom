## BUNDLE-01: Next.js 기술 스택 통합 ⚠️ P0

**포함 항목**: SEO, Rate Limiting, Performance, Data Fetching
**예상 시간**: 3시간

```markdown
# Next.js 16 통합 기술 스택 리서치 요청

## 프로젝트 컨텍스트
- 앱 이름: 이룸 (Yiroom) - 통합 웰니스 AI 플랫폼
- 기술 스택: Next.js 16, React 19, Supabase, Clerk, Gemini 3 Flash
- 배포: Vercel
- 현재 상태: MVP 단계, SEO 50%, 성능 최적화 80%

## 통합 리서치 요청

다음 4개 영역에 대해 **통합된 관점**에서 리서치해주세요:

### 1. SEO 최적화 (Next.js 16 App Router)
- Metadata API 완전 가이드
- generateMetadata 동적 생성
- JSON-LD 구조화 데이터 (웰니스/뷰티 앱 특화)
- Open Graph 및 Twitter Cards
- sitemap.xml, robots.txt 자동 생성
- Core Web Vitals와 SEO 연관성
- 한국어 SEO 특수 고려사항

### 2. Rate Limiting (Upstash Redis)
- @upstash/ratelimit 설정 가이드
- Sliding Window vs Token Bucket vs Fixed Window
- 사용자별, IP별, 엔드포인트별 제한
- Vercel Edge Functions와 통합
- AI 분석 API 특화 제한 (50회/일/사용자)
- 429 응답 UX 처리

### 3. 성능 최적화
- ISR vs SSG vs SSR 선택 기준 (AI 분석 앱)
- React Server Components 최적화
- 이미지 최적화 (next/image + Supabase Storage)
- 번들 분석 및 코드 스플리팅
- Core Web Vitals 목표 (LCP < 2.5s, CLS < 0.1)
- Vercel Analytics 연동

### 4. 데이터 페칭 전략
- Server Components에서 fetch 캐싱
- Client Components에서 React Query
- Supabase 실시간 구독
- 낙관적 업데이트 패턴
- 에러 처리 및 재시도

## 출력 형식

각 영역별:
1. **현재 베스트 프랙티스** (2026년 기준)
2. **구현 코드 예시** (TypeScript)
3. **설정 파일 예시** (next.config.ts 등)
4. **통합 고려사항** (4개 영역 간 상호작용)
5. **체크리스트** (구현 완료 확인용)

## 특별 요청
- Next.js 16, React 19 최신 API 사용
- Vercel 배포 환경 최적화
- AI 분석 앱 특성 반영 (이미지 업로드, 긴 응답 시간)
```

## BUNDLE-02: 수익화 및 비즈니스 모델 ⚠️ P0

**포함 항목**: Monetization, Pricing, Payment, Cost Model
**예상 시간**: 2시간

```markdown
# 웰니스 AI 앱 수익화 전략 통합 리서치

## 프로젝트 컨텍스트
- 앱: 이룸 - 퍼스널컬러, 피부분석, 체형분석, 영양/운동 추천
- 타겟: 20-30대 한국 여성 (확장: 남성, 글로벌)
- 현재: 무료, 수익 모델 없음
- 기술 비용: Gemini API, Supabase, Vercel

## 통합 리서치 요청

### 1. 수익화 모델 설계
- Freemium vs 구독 vs 크레딧 기반
- 뷰티/웰니스 앱 성공 사례 벤치마크
  - FaceApp, Noom, MyFitnessPal 수익 모델
- 무료 제공 기능 vs 유료 기능 구분 기준
- 한국 시장 가격 민감도

### 2. 가격 책정 전략
- 가격대 벤치마크 (한국 뷰티/웰니스 앱)
- 심리적 가격 포인트 (₩4,900, ₩9,900, ₩19,900)
- 연간 vs 월간 구독 할인율
- 프로모션 전략 (첫 달 무료, 친구 초대)
- A/B 테스트 설계

### 3. 결제 시스템 (한국)
- 토스페이먼츠 vs 아임포트 vs Stripe 비교
- 인앱결제 (iOS/Android) 수수료 고려
- 구독 자동결제 구현
- 환불 정책 및 처리
- PG사 선택 기준

### 4. 운영 비용 모델
- MAU별 비용 예측 (1K, 10K, 100K)
- Gemini API 비용 최적화
- Supabase 비용 스케일링
- 손익분기점 계산
- Unit Economics (CAC, LTV, LTV/CAC)

## 출력 형식

1. **수익 모델 비교표** (장단점, 추천)
2. **가격표 제안** (3-4 티어)
3. **비용 스프레드시트** (MAU별)
4. **결제 시스템 비교표**
5. **1년 재무 시나리오** (Best/Base/Worst)
6. **실행 로드맵** (MVP → 수익화 단계)
```

## BUNDLE-05: 플랫폼 전략 (웹/앱) ⚠️ P0

**포함 항목**: Web vs App, PWA, Native, Cross-platform
**예상 시간**: 2시간

```markdown
# 웰니스 AI 서비스 플랫폼 전략 리서치

## 프로젝트 컨텍스트
- 현재: Next.js 웹 앱 (apps/web)
- 계획: Expo 모바일 앱 (apps/mobile)
- 특성: 카메라 사용, AI 분석, 결과 저장
- 타겟: 20-30대 한국 모바일 사용자

## 통합 리서치 요청

### 1. 웹 vs 앱 전략
- 웹 퍼스트 vs 앱 퍼스트 결정 기준
- 뷰티/웰니스 앱 시장 데이터 (웹 vs 앱 사용 비율)

- 사용자 획득 관점 (ASO vs SEO)
- 수익화 관점 (인앱결제 vs 웹결제)

### 2. PWA 전략
- PWA 적합성 평가 (이룸 기능 기준)
- 오프라인 기능 설계
- 푸시 알림 (웹 푸시)
- 홈 화면 추가 유도
- PWA vs 네이티브 성능 비교
- 카메라 API 제한사항

### 3. 네이티브 앱 (Expo)
- Expo SDK 54 최신 기능
- 카메라 연동 (expo-camera)
- 이미지 처리 성능
- 앱스토어 배포 프로세스
- EAS Build/Submit

### 4. 크로스플랫폼 코드 공유
- 웹-모바일 코드 공유 전략
- packages/shared 구조
- UI 컴포넌트 공유 (vs 분리)
- 비즈니스 로직 공유
- 타입 공유

## 출력 형식

1. **플랫폼 의사결정 매트릭스**
2. **기능별 플랫폼 분배표**
3. **PWA 구현 가이드** (Next.js 16)
4. **Expo 앱 아키텍처**
5. **코드 공유 전략**
6. **출시 로드맵** (웹 → PWA → 네이티브)
```

## BUNDLE-06: 크로스 도메인 시너지 ⚠️ P1

**포함 항목**: Cross-Domain Synergy, PC×Skincare, Skin×Nutrition, Multi-Domain Capsule
**예상 시간**: 3시간

```markdown
# 이룸 크로스 도메인 시너지 리서치

## 프로젝트 컨텍스트
- 이룸 도메인: 퍼스널컬러, 피부분석, 체형분석, 자세분석, 영양, 운동
- 핵심 가치: "통합된 자기 이해"
- 현재: 도메인별 독립 분석
- 목표: 도메인 간 연결로 통합 인사이트

## 통합 리서치 요청

### 1. 크로스 도메인 시너지 원리
- 도메인 연결 매트릭스 (어떤 도메인이 어떻게 연결되는가?)
- 직접 연결 vs 간접 연결 vs 상호 강화
- 제약 조건 (A가 B를 제한하는 경우)
- 통합 점수 계산 방법
- 전체 웰니스 점수 설계

### 2. 퍼스널컬러 × 스킨케어/메이크업
- 웜톤/쿨톤과 파운데이션 톤 매칭
- 시즌별 베이스 메이크업 추천 로직
- 컬러 코렉팅 원리
- 립/아이 컬러 추천 연결
- 제품 DB 연동 방법

### 3. 피부 × 영양 (이너뷰티)
- 피부 타입별 영양소 매핑
- 피부 고민별 식품/보조제 추천
- 경구 섭취 → 피부 효과 과학적 근거
- 이너뷰티 제품 추천 알고리즘

### 4. 다중 도메인 캡슐 루틴
- 캡슐 워드로브 → 캡슐 루틴 확장
- 아침 루틴 = 스킨케어 + 영양 + 착장
- 하루 루틴 = 아침 + 운동 + 저녁
- 대체 옵션 다중 레벨
- 사용자 인벤토리 기반 추천

## 출력 형식

1. **도메인 연결 다이어그램** (Mermaid)
2. **시너지 매트릭스** (6×6)
```
