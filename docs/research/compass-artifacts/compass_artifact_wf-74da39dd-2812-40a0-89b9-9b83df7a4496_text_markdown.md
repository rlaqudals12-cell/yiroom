# 이룸 서비스 운영 비용 모델 분석

**사용자 1명당 월 $0.07~$0.25의 변동 비용이 발생하며, 월 1,000명 규모에서 $113의 총 운영비가 예상된다.** 이룸(Yiroom)의 AI 이미지 분석 서비스는 Google Gemini API가 예상외로 저렴하고, Supabase Image Transformations와 Vercel 인프라가 주요 비용 드라이버가 될 것으로 분석된다. 스케일업 시 **10,000명 기준 월 $700~$900**, **100,000명 기준 월 $4,000~$6,000** 수준으로 증가한다.

## AI API 비용: Gemini가 예상외로 저렴

Google Gemini API 비용은 전체 운영비의 **2~3%** 수준으로, 초기 우려와 달리 매우 저렴하다. 이미지 분석 기반 서비스임에도 비용 부담이 낮은 이유는 Gemini 2.0 Flash의 공격적인 가격 정책 때문이다.

### 모델별 가격 비교 (1M 토큰당)

| 모델 | 입력 | 출력 | 이미지 1장 비용 | 이룸 추천 |
|------|------|------|-----------------|-----------|
| **Gemini 2.0 Flash** | $0.10 | $0.40 | ~$0.00006 | ✅ 최적 |
| Gemini 2.5 Flash | $0.30 | $2.50 | ~$0.00019 | 고품질 필요 시 |
| Gemini 2.5 Pro | $1.25 | $10.00 | ~$0.00079 | 복잡 분석 시 |

**퍼스널컬러 분석 1회당 비용**: 이미지 1장(~1,000 토큰) + 프롬프트(300 토큰) + 출력(800 토큰) 기준 **$0.00045**. 사용자당 월 3회 분석 시 **월 $0.00135/사용자**에 불과하다.

### 무료 티어 활용 전략

Gemini 2.0 Flash는 일일 500회 무료 요청을 제공한다. **월 15,000회까지 무료**이므로 5,000명 × 월 3회 분석까지 무료 티어로 커버 가능하다. 단, 2025년 12월부터 무료 티어 한도가 **50~80% 감소**했으므로 프로덕션에서는 유료 Tier 1 전환이 안전하다.

## 인프라 비용: Vercel과 Supabase가 핵심

인프라 비용은 **총 운영비의 85~95%**를 차지하며, 특히 Supabase의 Image Transformations 비용이 스케일업 시 급증한다.

### Vercel 비용 구조

| 항목 | Pro 포함량 | 초과 단가 | 주요 비용 요인 |
|------|------------|-----------|----------------|
| 기본 플랜 | $20/사용자/월 | - | 상업적 사용 필수 |
| Edge Requests | 10M/월 | $0.60/1M | 낮음 |
| Bandwidth | 1TB/월 | $0.15/GB | 이미지 서비스 시 주의 |
| Image Optimization | 크레딧 기반 | $0.05/1K | **주요 비용 요인** |

**Next.js 16 특화 비용**: ISR(Incremental Static Regeneration)과 Image Optimization이 주요 비용 드라이버다. **Edge Function을 적극 활용하면 Serverless 대비 15배 저렴**하므로, AI 분석 결과 캐싱 로직을 Edge로 이전하는 것을 권장한다.

### Supabase 비용 구조

| 항목 | Pro 포함량 | 초과 단가 | 10,000명 예상 |
|------|------------|-----------|---------------|
| 기본 플랜 | $25/월 | - | $25 |
| Storage | 100GB | $0.021/GB | $1.05 |
| Auth MAUs | 100,000 | $0.00325 | $0 |
| **Image Transform** | 100개 | **$5/1,000개** | **$375** |
| Egress | 250GB | $0.09/GB | $0 |

**경고**: Image Transformations가 가장 큰 비용 함정이다. 10,000명 사용자(각 7.5장 이미지) 시 월 **$375**가 발생한다. 이 비용은 매월 리셋되어 누적된다.

### Upstash Redis 비용 구조

캐싱 및 세션 관리용 Redis는 상대적으로 저렴하다:

| 시나리오 | 월간 커맨드 | 월 비용 |
|----------|-------------|---------|
| 1,000명 DAU | 300K | $0 (무료 티어) |
| 5,000명 DAU | 1.5M | ~$3 |
| 10,000명 DAU | 3M | ~$6 |
| 50,000명 DAU | 15M | ~$30 |

Free 티어(월 500K 커맨드)로 초기 단계를 충분히 커버할 수 있으며, **Pay-as-you-go 방식이 이룸 규모에 최적**이다.

## 스토리지 비용 분석: 이미지 저장 전략이 핵심

이룸 서비스는 사용자당 평균 **5~10장의 고해상도 이미지**(각 2MB)를 저장한다고 가정했다.

### 저장 비용 시나리오

| 사용자 | 총 이미지 | 저장 용량 | Supabase 비용 | 외부 대안 비용 |
|--------|-----------|-----------|---------------|----------------|
| 1,000명 | 7,500장 | 15GB | $0 | $0.35 (S3) |
| 10,000명 | 75,000장 | 150GB | $1.05 | $3.45 (S3) |
| 100,000명 | 750,000장 | 1.5TB | $29.40 | $34.50 (S3) |

**분석**: 순수 저장 비용은 낮지만, **이미지 변환(썸네일, 리사이징) 비용이 10~50배 더 크다**. 저장소 자체보다 변환 최적화가 비용 절감의 핵심이다.

### 스토리지 비용 절감 전략

- **클라이언트 압축**: 업로드 전 80% 품질로 압축 → 용량 **50% 절감**
- **WebP 포맷 강제**: JPEG 대비 **30~50% 용량 감소**
- **사전 변환 저장**: 자주 쓰는 썸네일 크기를 업로드 시 생성 저장 → 반복 변환 방지
- **자동 삭제 정책**: 30일 이상 미사용 분석 이미지 자동 정리

**핵심 인사이트**: 고정비(Vercel Pro $20 + Supabase Pro $25)는 규모와 무관하게 **월 $45**로 고정된다. 1,000명 이상에서 변동비 중심 구조로 전환되며, 이때 **사용자당 비용이 $0.10 이하로 급감**한다.

| 규모 | 월 비용 | 월 수익 | **월 이익** | 이익률 |

**결론**: SaaS 비즈니스 특성상 **손익분기점은 매우 낮다**. 유료 전환율 5% 가정 시, **MAU 200명(유료 10명)**에서 손익분기 달성 가능하다.

## 결론 및 권장사항

이룸 서비스는 **매우 유리한 비용 구조**를 갖추고 있다. AI API 비용이 예상보다 10배 이상 저렴하고, 인프라 비용도 스케일에 따라 효율적으로 증가한다.

### 핵심 권장사항

- **즉시**: Gemini 2.0 Flash 고정 + 모든 서비스 Spend Cap 활성화
- **1개월 내**: 이미지 사전 변환 저장으로 Supabase 비용 80% 절감
- **3개월 내**: Edge Function 전환으로 Vercel 비용 최적화
- **손익분기**: MAU 122명에서 달성 가능 (₩9,900 구독, 5% 전환율)

### 예산 가이드라인

| 단계 | MAU 목표 | 월 예산 | 주의 항목 |
|------|----------|---------|-----------|
| MVP | 1,000 | $150 | Image Transforms |
| 성장 | 10,000 | $700 | Bandwidth + Transforms |
| 확장 | 100,000 | $5,000 | Enterprise 협상 검토 |

**최종 판단**: 현재 기술 스택(Vercel + Supabase + Gemini + Upstash)은 **100,000 MAU까지 월 $5,000 미만**으로 운영 가능하며, 솔로 개발자 단계에서 충분히 감당 가능한 수준이다. 이미지 변환 비용 최적화가 가장 큰 ROI를 제공하므로 이를 우선적으로 구현할 것을 권장한다.
