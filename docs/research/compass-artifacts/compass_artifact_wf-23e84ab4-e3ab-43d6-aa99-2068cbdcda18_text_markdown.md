# 이룸(Yiroom) AI 뷰티 앱 플랫폼 전략 분석

**결론: Expo 기반 App-First 전략을 권장합니다.** 한국 시장의 98% 스마트폰 보급률, 앱 전환율이 웹 대비 3배 높은 점, 그리고 AI 이미지 분석에 필수적인 카메라/GPU 접근성을 고려할 때, 네이티브 앱이 이룸의 핵심 기능인 퍼스널컬러 분석과 피부 진단에 최적입니다. 다만, Expo SDK 54의 웹 빌드 기능을 활용하여 초기 랜딩 페이지와 온보딩은 웹으로 시작하고, 핵심 AI 기능은 앱으로 유도하는 **하이브리드 접근법**이 솔로 개발자에게 가장 효율적입니다.

## 한국 사용자는 앱을 압도적으로 선호한다

한국은 세계 최고 수준의 모바일 환경을 갖추고 있으며, 사용자 행동 패턴이 네이티브 앱에 유리합니다. **스마트폰 보급률 98-99%**, 일일 평균 모바일 사용시간 **5시간 22분**, 그리고 모바일 시간의 **87%가 앱 내에서** 소비됩니다. 특히 20대 타겟 시 iOS 점유율이 **64%**에 달해 Apple 생태계 대응이 필수입니다.

뷰티 앱 시장에서 올리브영 앱은 MAU **100만 이상**, 화해 앱은 **840만 개 사용자 리뷰**를 보유하며 시장을 지배하고 있습니다. 이 두 앱이 뷰티 앱 시장의 약 **75%**를 장악하고 있으며, 모두 App-First 전략을 채택했습니다. 퍼스널컬러 분석은 현재 한국 뷰티 트렌드의 핵심 키워드로 부상했으며, 올리브영에서도 퍼스널컬러별 제품 라벨링을 적용하고 있습니다.

| 플랫폼 지표 | 한국 시장 데이터 |
|------------|-----------------|
| Android 점유율 | 75.7% |
| iOS 점유율 (20대) | 64% |
| 앱 vs 웹 전환율 | 앱이 **3배** 높음 |
| 앱 세션 길이 | 웹 대비 **2.4배** |
| 푸시 알림 opt-in | 앱 60-67% vs 웹 **5%** |

## 카메라와 AI 처리는 네이티브가 확실히 우위

퍼스널컬러 분석과 피부 진단의 핵심은 정확한 색상 재현과 실시간 이미지 처리입니다. 이 영역에서 네이티브 앱과 웹앱의 기술적 격차는 분명합니다.

**카메라 접근성 비교:** 웹(getUserMedia API)은 기본 촬영과 해상도 설정이 가능하지만, 다중 렌즈 선택, 수동 노출 제어, RAW 이미지 캡처가 불가능합니다. iOS Safari에서는 플래시/토치 제어도 지원되지 않습니다. 반면 Expo Camera SDK 54는 **2160p 해상도, 다중 렌즈 접근(iOS), 비디오 안정화, 비트레이트 제어**까지 지원합니다.

**AI/ML 처리 성능 격차:** 웹에서 TensorFlow.js + WebGPU는 네이티브 대비 약 **80% 성능**을 달성하며, MediaPipe 얼굴 감지는 CPU에서 30-70 FPS를 제공합니다. 그러나 네이티브 앱은 Core ML(iOS)과 ML Kit(Android)를 통해 Neural Engine과 GPU를 직접 활용하여 **8-14ms 지연**(웹은 14-20ms)과 더 나은 배터리 효율을 제공합니다.

| AI 뷰티 기능 | 웹 적합성 | 네이티브 적합성 | 권장 플랫폼 |
|-------------|----------|----------------|------------|
| 퍼스널컬러 실시간 분석 | ⚠️ 제한적 | ✅ 최적 | **네이티브** |
| 피부 결점/수분 감지 | ⚠️ 가능 | ✅ 최적 | **네이티브** |
| 가상 메이크업 AR | ❌ 미지원 | ✅ ARKit/ARCore | **네이티브** |
| 체형 분석 | ✅ 가능 | ✅ 최적 | 둘 다 가능 |
| 정적 이미지 분석 | ✅ 충분 | ✅ 최적 | 웹 가능 |

## 리텐션과 전환율: 앱이 압도적 우위

뷰티 앱의 성공은 사용자 리텐션에 달려 있습니다. Perfect Corp(YouCam)의 **1.1억 다운로드, 월간 16.8억 회 가상 트라이온, 평균 2일마다 재방문** 사례가 이를 증명합니다.

**리텐션 벤치마크:** 모바일 앱의 D30 리텐션 평균은 **5-6%**이며, 우수 기준은 **10% 이상**입니다. 뷰티/라이프스타일 앱은 참여 루프 활용 시 **8%**까지 도달 가능합니다. 반면 모바일 웹은 첫 방문 후 **30-50% 더 낮은** 리텐션을 보입니다.

**푸시 알림의 결정적 차이:** 앱 푸시 알림 opt-in율은 **61-67%**인 반면, 웹 푸시는 단 **5%**입니다. 푸시 알림을 수신하는 사용자의 리텐션은 미수신자 대비 **약 3배** 높으며, 푸시 미수신 사용자의 **95%가 이탈**합니다. iOS에서 PWA 푸시 알림은 iOS 16.4 이후 지원되지만, 홈화면 설치 후에만 작동하며 신뢰성이 낮습니다.

**전환율 비교:** 앱 사용자의 세션당 제품 조회는 **22개**(웹 5.7개), 전환율은 **18-21%**(웹 4-6%)로 앱이 **4.2배** 높은 참여도와 **3배** 높은 전환율을 보입니다. 인앱 결제(IAP)는 웹 결제 대비 초기 전환율이 **42% 더 높습니다**.

## 개발 비용과 출시 속도 분석

솔로 개발자로서 이룸의 기술 스택(Next.js 16, React 19, Expo SDK 54, Supabase, Gemini AI)은 최적의 선택입니다. 이 조합은 **코드의 50-90%를 웹과 앱 간 공유**할 수 있습니다.

**개발 비용 비교 (2025년 기준):**

| 플랫폼 전략 | MVP 비용 | 개발 기간 | 유지보수 복잡도 |
|------------|---------|----------|----------------|
| 웹앱 (Next.js) | $15,000-$30,000 | 4-8주 | ⭐⭐ 낮음 |
| Expo (iOS+Android+Web) | $40,000-$80,000 | 2-3개월 | ⭐⭐ 낮음 |
| 네이티브 (iOS+Android) | $100,000-$200,000 | 4-6개월 | ⭐⭐⭐⭐ 높음 |
| **웹 우선 → Expo 확장** | **$35,000-$70,000** | **2-4개월** | **⭐⭐ 낮음** |

**Expo SDK 54의 전략적 가치:** React Native 0.81.4와 React 19.1.0을 지원하며, iOS XCFrameworks 프리컴파일로 빌드 시간이 **10배 단축**(120초 → 10초)되었습니다. OTA(Over-The-Air) 업데이트로 앱스토어 심사 없이 JavaScript/에셋 변경을 **분 단위로 배포**할 수 있어, 웹 수준의 빠른 반복이 가능합니다.

**앱스토어 심사 리스크:** Apple은 90%의 앱을 24시간 내 심사하지만, 거절률은 약 **25%**입니다. Expo OTA를 활용하면 핵심 기능 업데이트 시에만 심사를 거치고, 버그 수정과 UI 개선은 즉시 배포할 수 있습니다.

## 단계별 플랫폼 확장 로드맵

### Phase 1: 빠른 시장 검증 (0-3개월)

**목표:** 핵심 가설 검증, 초기 사용자 확보

- **랜딩 페이지 (Next.js)**: 서비스 소개, 사전 등록, SEO를 통한 오가닉 트래픽 확보
- **웹 MVP**: 카카오/네이버 로그인, 간단한 퍼스널컬러 테스트(설문 기반), 결과 공유 기능
- **이메일 수집**: 앱 출시 시 알림 신청자 확보
- **비용**: $15,000-$25,000 / 4-6주
- **KPI**: 사전등록 1,000명, 퍼스널컬러 테스트 완료율 60%

### Phase 2: 핵심 앱 출시 (3-6개월)

**목표:** AI 핵심 기능 앱 출시, 리텐션 최적화

- **Expo 앱 (iOS + Android)**: 카메라 기반 퍼스널컬러 AI 분석, 피부 진단, 맞춤 추천
- **웹 → 앱 유도**: 웹에서 심화 분석은 앱 설치 유도 (Deep Link)
- **푸시 알림 시스템**: 개인화된 루틴 리마인더, 시즌별 스타일링 제안
- **인앱 결제**: 프리미엄 분석 기능 구독 모델
- **추가 비용**: $25,000-$45,000 / 6-8주
- **KPI**: D1 리텐션 35%+, D30 리텐션 10%+, 푸시 opt-in 60%+

### Phase 3: 기능 확장 및 수익화 (6-12개월)

**목표:** 풀 프로덕트 완성, 수익 모델 안정화

- **체형 분석 + 패션 추천**: AI 기반 체형 분석, 스타일링 추천
- **영양/피트니스 통합**: 캡슐 루틴 엔진 고도화
- **B2B API**: 뷰티 브랜드/리테일러 대상 화이트라벨 솔루션
- **웹앱 고도화**: PWA 기능 강화 (오프라인 캐싱, 홈화면 설치)
- **KPI**: MAU 50,000+, 유료 전환율 5%+, MRR $10,000+

### Phase 4: 스케일업 (12-24개월)

**목표:** 팀 확장, 글로벌 진출 준비

- **팀 빌딩**: React/React Native 개발자 채용 (풍부한 인재풀 활용)
- **성능 최적화**: 필요 시 특정 모듈 네이티브 전환 (Expo Development Builds)
- **글로벌 버전**: 영어, 일본어 현지화
- **고려사항**: 네이티브 전환은 성능 병목 발생 시에만, 대부분 기능은 Expo로 충분

## 한국 시장 필수 연동 요소

한국 시장 성공을 위해 반드시 통합해야 할 요소들입니다.

**소셜 로그인 (최우선):** 카카오 로그인 도입 후 신규 회원가입이 **241.9% 증가**한 사례가 있으며, 회원가입의 **75%가 카카오 로그인**을 통해 이루어집니다. 네이버 로그인도 필수입니다.

**결제 연동:** 카카오페이, 네이버페이는 한국 사용자에게 익숙한 결제 수단입니다. 앱스토어 수수료(14-26%)를 고려하여 첫 결제는 IAP, 충성 사용자는 웹 결제로 유도하는 하이브리드 전략을 권장합니다.

**개인정보보호법 준수:** 얼굴 이미지는 **개인정보**에 해당하여 수집 전 동의가 필수입니다. 생체인식 특징정보(얼굴 인식용 데이터)는 **민감정보**로 분류되어 별도 동의와 암호화가 필요합니다. 만 14세 미만은 법정대리인 동의가 필수입니다.

## 최종 권장 전략: Expo 기반 하이브리드 접근

**핵심 결정:** App-First (Expo SDK 54) + 웹 보조

| 요소 | 권장 | 근거 |
|-----|-----|-----|
| **주력 플랫폼** | Expo 앱 (iOS + Android) | 카메라/AI 필수, 리텐션/전환율 우위 |
| **보조 플랫폼** | Next.js 웹 | SEO, 사전등록, 앱 유도 |
| **개발 순서** | 웹 랜딩 → 앱 MVP → 웹 고도화 | 빠른 검증 + 핵심 기능 앱 집중 |
| **코드 공유** | 모노레포 (비즈니스 로직 90%+ 공유) | 유지보수 효율성 |
| **배포 전략** | Expo OTA + 앱스토어 | 빠른 반복 + 앱 생태계 이점 |

**이 전략의 장점:**
1. **기술 스택 시너지**: 이미 보유한 Next.js + React + Expo 스택 최대 활용
2. **비용 효율성**: 네이티브 대비 **50% 이상 비용 절감**, 단일 코드베이스
3. **빠른 반복**: OTA 업데이트로 웹 수준의 배포 속도 유지
4. **리텐션 극대화**: 푸시 알림 67% opt-in, 앱 리텐션 3배 효과
5. **팀 확장 용이**: React 생태계의 풍부한 인재풀 활용 가능

**주의사항:**
- Android 우선 개발 권장 (75.7% 점유율), iOS 동시 지원 필수 (20대 64%)
- Expo 앱 크기 약 20-30MB 추가 오버헤드 감수
- 카카오/네이버 소셜 로그인 초기부터 통합
- 개인정보 동의 프로세스 철저히 설계

이룸의 AI 퍼스널컬러 분석이라는 차별화된 핵심 기능은 네이티브 앱에서 최상의 경험을 제공할 수 있습니다. Expo SDK 54의 성숙도와 솔로 개발자에게 적합한 코드 공유 전략을 활용하면, 제한된 리소스로도 올리브영, 화해와 차별화된 AI 뷰티 플랫폼을 구축할 수 있습니다.
