## 2. 한국 법률 경계: "웰니스 추천" vs "의료 조언"

### 2.1 약사법 제44조 (의약품 판매 행위)

**조문 요지**: 약사 또는 한약사가 아닌 자는 의약품을 판매하거나 판매 목적으로 조제할 수 없다.

**이룸에 대한 적용**:

| 행위                                     | 허용 여부 | 근거                                 |
| ---------------------------------------- | --------- | ------------------------------------ |
| "비타민 C가 풍부한 식품 목록 제공"       | 허용      | 일반 영양 정보                       |
| "비타민 C 1000mg 보충제를 드세요"        | **금지**  | 특정 용량의 의약품/건강기능식품 권장 |
| "건조한 피부에 보습 크림을 사용해보세요" | 허용      | 일반적 화장품 카테고리 제안          |
| "이 약을 하루 2회 복용하세요"            | **금지**  | 의약품 용법 지시 (약사 행위)         |
| "XX 성분이 포함된 제품을 찾아보세요"     | 허용      | 성분 정보 제공                       |
| "피부 트러블에 XX 연고를 바르세요"       | **금지**  | 의약외품/의약품 권장                 |

**핵심 경계선**:

- 성분의 **일반 특성 설명** = 허용
- 특정 제품의 **용법/용량 지시** = 금지
- "~를 추천합니다" 대신 "~를 고려해볼 수 있어요" = 더 안전한 표현

### 2.2 화장품법 제13조 (표시/광고)

**금지되는 표현 (2025년 기준)**:

| 카테고리    | 금지 표현 예시               | 허용 대체 표현                               |
| ----------- | ---------------------------- | -------------------------------------------- |
| 의료적 효능 | "여드름 치료", "아토피 개선" | "여드름 피부에 적합한", "민감한 피부를 위한" |
| 의약품 오인 | "피부 치료제", "피부 재생"   | "피부 보습에 도움", "피부 컨디셔닝"          |
| 과장 효능   | "주름 제거", "탈모 방지"     | "주름 개선에 도움", "두피 건강 관리"         |
| 전문가 빙자 | "피부과 전문의 추천"         | "피부 타입별 제안"                           |

**이룸 적용 원칙**:

- 화장품 추천 시 "효능" 대신 "특성"으로 표현
- "치료/개선/제거" 대신 "관리/케어/도움"으로 표현
- 모든 추천에 "개인차가 있을 수 있어요" 포함

### 2.3 의료법 제27조 (무면허 의료행위)

**핵심**: 의료인이 아닌 자는 의료행위를 할 수 없다.

| 이룸 기능                     | 분류         | 판단 근거                         |
| ----------------------------- | ------------ | --------------------------------- |
| AI 피부 타입 분석 (건성/지성) | **비의료**   | 일반적 피부 상태 분류, 진단 아님  |
| "피부암이 의심됩니다"         | **의료행위** | 질병 진단 = 의료행위              |
| 성분 알레르기 경고            | **비의료**   | 기존 알레르기 정보 기반 필터링    |
| "이 증상은 습진입니다"        | **의료행위** | 질병 진단 = 의료행위              |
| 칼로리/영양소 계산            | **비의료**   | 일반 건강 정보 제공               |
| "당뇨 식단을 따르세요"        | **경계**     | 질환 관리 지시 → 의료 조언에 근접 |

### 2.4 "웰니스 추천"과 "의료 조언"의 경계 기준

```
                       웰니스 추천                    의료 조언
                    (이룸 허용 범위)              (이룸 금지 범위)
                  ─────────────────           ─────────────────
일반 정보 제공    "비타민 C는 항산화..."       "비타민 C 3000mg을 드세요"
피부 분석         "건성 피부 타입이에요"       "접촉피부염이 의심돼요"
제품 추천         "보습력 높은 제품이에요"     "이 약용 크림을 바르세요"
성분 정보         "EWG 3등급이에요"           "이 약은 부작용이 있어요"
건강 추적         "수분 섭취 기록"            "혈당 수치 관리 지시"
```

**3단계 판단 기준**:

1. **정보 제공인가, 지시인가?**
   - "~할 수 있어요" (정보) vs "~하세요" (지시)

2. **일반적 특성인가, 특정 질환인가?**
   - "건성 피부" (일반) vs "아토피" (질환)

3. **제품 카테고리인가, 특정 의약품인가?**
   - "보습 크림" (카테고리) vs "XX 연고 30g" (특정 의약품)

## 3. PIPA 개인정보보호법 제23조: 민감정보

### 3.1 민감정보 정의

개인정보보호법 제23조에 따르면 다음은 **민감정보**에 해당한다:

> "사상, 신념, 노동조합, 정당의 가입, 탈퇴, 정치적 견해, **건강**, **유전정보**, 범죄 경력 등"

**이룸의 민감정보 해당 항목**:

| 데이터                | 민감정보 여부 | 근거                              |
| --------------------- | ------------- | --------------------------------- |
| 알레르기 목록         | **민감정보**  | 건강 관련 정보                    |
| 복용 중인 약          | **민감정보**  | 건강 관련 정보                    |
| 임신/수유 여부        | **민감정보**  | 건강 관련 정보                    |
| 피부질환 병력         | **민감정보**  | 건강 관련 정보                    |
| 피부 타입 (건성/지성) | 비민감        | 일반적 특성 분류                  |
| 퍼스널컬러 결과       | 비민감        | 색상 선호도                       |
| 체형 타입             | **경계**      | 신체적 특징 (BMI 등 포함 시 민감) |

### 3.2 별도 동의 요건

**법적 의무**:

- 민감정보 수집 시 **일반 개인정보와 별도의 동의**를 받아야 한다 (제23조 제1항)
- 동의서에 **수집 목적, 보유 기간, 제3자 제공 여부**를 명시해야 한다
- "포괄 동의" 금지 — 각 민감정보 항목별 개별 동의 가능해야 한다

**이룸 구현 요건**:

```
일반 개인정보 동의 (회원가입 시)
├── 이름, 이메일, 생년월일
└── 피부 타입, 퍼스널컬러 결과

민감정보 별도 동의 (Safety Profile 생성 시) ← 별도 화면
├── 알레르기 정보 (선택)
├── 복용 약물 (선택)
├── 임신/수유 상태 (선택)
├── 피부질환 병력 (선택)
└── 각 항목별 개별 동의 체크박스
```

### 3.3 암호화 요건

개인정보보호법 시행령 제30조 및 KISA 가이드라인:

| 구분        | 요건                       | 구현 방법                         |
| ----------- | -------------------------- | --------------------------------- |
| 전송 암호화 | TLS 1.2+ 필수              | Vercel/Supabase 기본 적용         |
| 저장 암호화 | AES-256 이상               | Supabase Vault + 컬럼 레벨 암호화 |
| 키 관리     | 암호화 키 분리 보관        | Supabase Vault (pgsodium)         |
| 접근 로그   | 민감정보 접근 시 감사 로그 | audit_logs 테이블 기록            |

**Supabase Vault 활용 패턴**:

```sql
-- 민감정보 암호화 저장
INSERT INTO safety_profiles (
  clerk_user_id,
  allergies_encrypted,    -- vault_encrypt()
  medications_encrypted,  -- vault_encrypt()
  pregnancy_status,       -- 암호화 필요
  consent_timestamp,
  consent_version
) VALUES (...);

-- 복호화 조회 (RLS 통과 후에만)
SELECT
  vault_decrypt(allergies_encrypted) as allergies,
  vault_decrypt(medications_encrypted) as medications
FROM safety_profiles
WHERE clerk_user_id = auth.get_user_id();
```

### 3.4 삭제권 (Right to Deletion)

개인정보보호법 제36조:

- 사용자는 **언제든** 민감정보 삭제를 요청할 수 있다
- 삭제 요청 시 **5일 이내** 처리해야 한다
- 삭제 후 **복구 불가능**한 상태로 완전 삭제해야 한다
- 관련 백업/캐시에서도 삭제해야 한다

**이룸 구현**:

- 프로필 설정에서 "안전 프로필 삭제" 버튼
- 삭제 시 확인 대화상자 + 재인증 요구
- Supabase에서 해당 행 물리적 삭제 (soft delete 아님)
- 관련 캐시 즉시 무효화

### 3.5 데이터 보유 기간

| 데이터 유형                  | 보유 기간              | 근거                         |
| ---------------------------- | ---------------------- | ---------------------------- |
| 안전 프로필 (알레르기, 약물) | 회원 탈퇴 시 즉시 삭제 | 개인정보보호법 제21조        |
| 동의 기록                    | 동의 철회 후 3년 보관  | 전자상거래법 제6조           |
| 접근 감사 로그               | 3년 보관               | 개인정보보호법 시행령 제30조 |
| 안전 경고 발송 기록          | 5년 보관               | 소비자보호 증빙 목적         |

## 4. 성분 안전성 과학 (Ingredient Safety Science)

### 4.1 EWG 등급 체계

Environmental Working Group의 Skin Deep 데이터베이스 기반:

| 등급 | 위험도                 | 설명                           | 이룸 표시   |
| ---- | ---------------------- | ------------------------------ | ----------- |
| 1-2  | 낮음 (Low Hazard)      | 안전성 데이터 충분, 우려 최소  | 초록색 배지 |
| 3-6  | 중간 (Moderate Hazard) | 일부 우려 존재, 용도/농도 의존 | 노란색 배지 |
| 7-10 | 높음 (High Hazard)     | 유의미한 우려 존재             | 빨간색 배지 |

**이룸 DB 현황** (cosmetic_ingredients 테이블):

- `ewg_grade` 컬럼: 1-10 정수
- `safety_level`: 'safe' | 'moderate' | 'caution'
- `description_ko`: 한국어 성분 설명

**중요 고지사항**: EWG 등급은 절대적 위험도가 아닌 "데이터 가용성 + 우려 수준"의 복합 점수다.
이룸은 "EWG 등급은 참고 정보이며, 개인별 반응은 다를 수 있어요"를 반드시 표시해야 한다.

### 4.4 약물 상호작용 및 금기사항

**임산부/수유부 금기 성분 분류**:

| 위험 등급     | 정의                            | 이룸 대응                      |
| ------------- | ------------------------------- | ------------------------------ |
| **절대 금기** | 사용 시 태아/영아에 직접적 위험 | 빨간색 경고 + 즉시 차단        |

**절대 금기 목록 (임산부)**:

- 레티노이드 계열 (레티놀, 트레티노인, 아다팔렌)
- 고농도 살리실산 (BHA > 2%)
- 하이드로퀴논
- 화학적 자외선차단제 (옥시벤존, 아보벤존)
- 톨루엔, 포름알데히드

**약물 상호작용 (이룸 기존 ingredient_interactions 테이블)**:

## 5. 국제 비교

### 5.1 EU Cosmetics Regulation (EC No 1223/2009)

**핵심 특징**:

- 세계에서 가장 엄격한 화장품 규제
- **1,600+** 성분 사용 금지 (Annex II)
- **300+** 성분 사용 제한 (Annex III)
- CPNP (Cosmetic Products Notification Portal) 등록 의무
- Responsible Person 제도: EU 내 책임자 지정 필수

**앱 서비스에 대한 시사점**:

- 성분 안전성 데이터를 제공하는 앱은 "정보 서비스"로 분류
- 의료 기기(Medical Device)로 분류되지 않으려면 "진단" 기능 회피 필수
- EU 시장 진출 시 GDPR + Cosmetics Regulation 이중 준수

### 5.2 US FDA 화장품 규제

**MoCRA (Modernization of Cosmetics Regulation Act, 2022)**:

- 2022년 12월 발효, FDA에 화장품 규제 강화 권한 부여
- 시설 등록 및 제품 목록 등록 의무화 (2024년 7월 시행)
- 심각한 부작용(Serious Adverse Event) 15일 이내 보고 의무
- 향료 알레르겐 표시 규정 논의 중

**앱 서비스 관련**:

- 미국은 화장품 앱에 대한 직접 규제 없음
- FTC Act Section 5에 의한 "허위/기만 광고" 금지 적용
- "이 앱은 FDA의 평가를 받지 않았습니다" 면책 충분

### 5.3 유사 앱 면책 사례 분석

| 앱                | 국가   | 면책 방식                               | 특징                                     |
| ----------------- | ------ | --------------------------------------- | ---------------------------------------- |
| **Yuka**          | 프랑스 | "독립적 정보 제공, 브랜드 후원 없음"    | EWG + EU 규제 기반, 별점 제외하고 사실만 |
| **INCI Beauty**   | 프랑스 | "성분 정보만 제공, 의료 조언 아님"      | EU 규제 데이터 기반                      |
| **Think Dirty**   | 캐나다 | "교육 목적, 의료/과학적 조언 대체 불가" | 포괄적 면책                              |
| **화해 (Hwahae)** | 한국   | "성분 등급은 참고 정보, 개인차 존재"    | EWG 기반, 한국 식약처 데이터             |

**공통 면책 패턴**:

1. "의료 조언이 아님" 명시
2. "개인차 존재" 언급
3. "전문가 상담 권고" 포함
4. 특정 브랜드/제품에 대한 "추천"이 아닌 "정보 제공" 프레이밍

### 7.4 사고 대응: 안전 경고 실패 (Incident Response)

**시나리오**: 알레르기 성분이 포함된 제품에 대해 경고를 표시하지 못한 경우

**대응 절차**:

```
1. 감지 (Detection)
   ├── 사용자 신고 접수
   ├── 자동 모니터링 (경고 발송 실패 로그)
   └── 정기 감사 (분기별)

2. 평가 (Assessment)
   ├── 영향 범위 파악 (몇 명의 사용자, 어떤 성분)
   ├── 원인 분석 (DB 누락? 로직 오류? 데이터 불일치?)
   └── 심각도 분류 (높음/중간/낮음)

3. 대응 (Response)
   ├── [높음] 즉시 해당 제품 추천 중단 + 영향 사용자 개별 알림
   ├── [중간] 24시간 내 수정 배포 + 영향 사용자 알림
   └── [낮음] 다음 배포 주기에 수정 + 앱 내 공지

4. 복구 (Recovery)
   ├── 누락 데이터 보완
   ├── 재발 방지 자동 테스트 추가
   └── 사고 보고서 작성 (docs/incidents/)

5. 후속 (Follow-up)
   ├── 영향 사용자에 사과 및 안내 메시지
   ├── 개인정보보호위원회 보고 (심각 시, 72시간 이내)
   └── 내부 프로세스 개선
```

**자동 모니터링 항목**:

| 모니터링              | 주기   | 임계값                        |
| --------------------- | ------ | ----------------------------- |
| 안전 경고 발송 성공률 | 실시간 | < 99.5% 시 알림               |
| 성분 DB 커버리지      | 일간   | 신규 제품 대비 미매칭 > 5%    |
| 알레르겐 매칭 정확도  | 주간   | 랜덤 샘플 검증                |
| 동의 유효성           | 일간   | 만료된 동의로 데이터 접근 0건 |

```
□ 약사법: 특정 의약품 용법/용량 지시 없음
□ 화장품법: "치료/제거/방지" 표현 없음
□ 의료법: 질병 진단 표현 없음
□ PIPA 제23조: 민감정보 별도 동의 화면 구현
□ PIPA 제36조: 삭제권 행사 기능 구현
□ 면책 Tier 1: 모든 분석 결과 페이지에 의료 면책 표시
□ 면책 Tier 2: 성분 경고 시 전문가 상담 권고 표시
□ 면책 Tier 3: 일반 정보 페이지에 참고 정보 안내 표시
```

### 8.2 데이터 보호 체크리스트

```
□ 민감정보 암호화: AES-256 컬럼 레벨 (Supabase Vault)
□ RLS 정책: safety_profiles 테이블 clerk_user_id 기반
□ 감사 로그: 민감정보 접근 시 audit_logs 기록
□ 전송 암호화: TLS 1.2+ (Vercel/Supabase 기본)
□ 삭제 기능: 물리적 삭제 + 캐시 무효화
□ 보유 기간: 정책에 따른 자동 만료 처리
□ 동의 기록: consent_timestamp + consent_version 저장
```

### 8.3 성분 안전성 체크리스트

```
□ EWG 등급 표시 + "참고 정보" 안내
□ EU 26 알레르겐 매칭 로직
□ 주의 성분 20종 경고 로직
□ 임산부/수유부 금기 성분 차단
□ 성분 상호작용 경고 (ingredient_interactions)
□ 안전 경고 실패 모니터링
```
