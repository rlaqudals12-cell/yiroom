# 제품 스캔 · 성분 판정 · 의학적 근거 리서치 (P7 리서치 단계)

> **작성일**: 2026-07-09 | **단계**: P7 워크플로우 ① 리서치 (원리 → ADR → 스펙 → 구현 이전)
> **목적**: OASIS/Yuka식 "제품 찍으면 성분·등급·나에게 맞는지" 기능을 이룸에 도입하기 위한 코드 감사 + 도메인/법적/의학 리서치
> **핵심 결론**: 이룸은 스캔·OCR·성분 상호작용·개인화 호환성 엔진을 **이미 상당 부분 보유**. 격차는 (1) 성분 안전등급 DB의 규모/출처, (2) "며칠 후 변해요" 의학 타임라인 기능 부재, (3) 개인화 판정을 전면에 내세우는 UX/브랜딩. Yuka의 "절대 등급"이 아니라 **"나와의 적합도"**를 차별점으로 삼아야 하며, 이것이 동시에 **법적으로도 가장 안전한 표현**이다.

## 과제 B: 웹 리서치 (2025-2026)

### B-1. Yuka / OASIS 모델 분석 — 무엇을 배우고 무엇을 피할까

**점수 체계 (화장품 모드):**

- Yuka 화장품 점수는 **가장 위험한 성분 하나가 전체 점수를 지배**한다. 고위험(red) 성분이 하나라도 있으면 제품 점수는 자동으로 25/100 미만, 중위험(orange)이면 50/100 미만. ([Yuka Help — cosmetic evaluation](https://help.yuka.io/l/en/article/2t20ixn5y5-evualuation-cosmetic-products), [penalties](https://help.yuka.io/l/en/article/ih5pet4ffc-how-are-penalties-calculated-in-the-cosmetic-product-scores))
- 성분 페널티: 발암·내분비교란 의심 -12점 ~ 저위험 알레르겐/자극 -2점.
- 성분 DB ≈ **12,500개**, 자체 독성학자가 신규 성분 검토 후 추가.
- (식품 모드 참고: 영양 60% + 첨가물 30% + 유기농 10%.)

**비판점 (반드시 피할 것):**

- **농도(%) 미반영**: Yuka는 제품 내 성분 함량을 고려하지 않아 "미량이라 안전한 성분"도 감점 → 부정확. ([Beauty Independent](https://www.beautyindependent.com/yuka-beauty-industry-insiders-question-product-ratings/), [Hypebae 화장품 화학자 인터뷰](https://hypebae.com/2025/11/yuka-app-makeup-skincare-ingredients-cosmetic-chemists-interview-trend-explainer))
- **비과학적·과잉 예방주의(precautionary)**: 규제기관이 아닌 자체 알고리즘이 "위험" 딱지를 붙임 → 균형 잡힌 독성학 평가 아님.
- **"최악 성분 1개 = 전체 red"의 공포 마케팅** 구조.

**법적 논란:**

- 프랑스(2021): 육가공협회가 아질산염 감점에 대해 Yuka 제소. 1심 패소했으나 항소심(엑상프로방스 2022, 파리 2023)에서 **Yuka 승소**. 표현의 자유로 보호받았으나 소송 자체가 리스크. ([Beauty Independent](https://www.beautyindependent.com/yuka-beauty-industry-insiders-question-product-ratings/))
- 즉 **제조사와의 분쟁 가능성**이 상수. 절대 등급으로 특정 제품을 "나쁨"이라 낙인찍는 모델은 법적 표적이 된다.

> **이룸의 교훈**: Yuka의 "절대 안전등급 + 최악성분 지배 + 공포"를 **복제하지 말 것**. 대신 이룸은 이미 가진 **개인화("나에게 맞는지")**를 전면에. 제품을 "나쁘다"고 하지 않고 "**당신의 지성·민감 피부에는 이 성분이 자극될 수 있다**"고 말한다 — 이는 (a) 더 유용하고 (b) 제조사 명예훼손 소지가 낮다.

> **타임라인 기능 설계 시사점**: "예측"이 아니라 **"임상에서 보통 N주 후 개선이 보고됩니다"**라는 **문헌 인용형 안내**로 표현. 개인 결과 보장(❌) → 문헌 근거 안내(✅). 각 성분 카드에 위 논문을 근거로 링크. 이는 이룸의 "변화 추적(직전 대비 칩)" 기능과 결합하면 강력 — 사용자가 실제로 그 기간에 재분석하도록 유도하는 **리텐션 훅**.

### B-4. 법적 경계 — 한국 화장품 표시·광고 규제

식약처 「화장품 표시·광고 관리 지침(민원인 안내서)」이 2024년 개정되며 금지표현이 확대됨. ([식약처 지침 PDF](https://www.mfds.go.kr/brd/m_1060/down.do?brd_id=data0011&seq=15456&data_tp=A&file_seq=1), [코스인코리아](https://www.cosinkorea.com/news/article.html?no=54081), [코스모닝](https://cosmorning.com/mobile/article.html?no=49744))

**앱이 성분·제품을 다룰 때 핵심 위험:**

1. **의약품 오인 표현 금지**: "치료", "여드름 치료", "재생", "병원용/피부과 전용" 등. 화장품은 질병 치료를 표방할 수 없음. → 이룸은 "**개선에 도움**", "**진정에 도움을 줄 수 있어요**" 수준으로.
2. **효능 보장·수치 단정 금지**: "피부 나이 O세 감소" 같은 표현 2024 개정으로 명시 금지. → "며칠 후 변해요"를 **단정하면 위험**. "임상 문헌상 보통 8-12주에 개선 보고" 형태로.
3. **제조사 명예훼손 리스크**: 특정 제품에 "나쁨" 절대 등급 → Yuka식 소송 표적. → **개인 적합도**("당신의 민감 피부엔 자극 가능")로 프레이밍하면 사실적 의견 + 개인 맥락이라 안전.
4. **면책·정보성 성격 명시**: 현재 스캔 결과 하단 면책문("참고용, 의학적 조언 대체 아님, 알레르기 시 전문가 상담")은 **유지·강화**. 광고가 아닌 **정보 제공 도구**임을 분명히.

> **안전 표현 가이드(초안)**:
>
> - ✅ "당신의 [지성/민감] 피부 프로필과 **적합도** 82점"
> - ✅ "식약처 지정 **알레르기 유발 착향제**가 포함되어 있어요" (규제 사실)
> - ✅ "임상 연구에서는 보통 **8-12주 후** 톤 개선이 보고됩니다 (출처)"
> - ❌ "이 제품은 **나쁩니다**" / "여드름을 **치료**합니다" / "**2주면** 주름이 사라져요"

### B-5. OCR / 비전 파이프라인 — 성분표 사진 추출

- 이룸은 이미 **Gemini 멀티모달 OCR**로 성분표를 추출 (`ingredient-ocr.ts`, `gemini-2.0-flash`). 다국어(ko/en/ja/zh) + INCI 변환 + 순서(함량순) + confidence 라벨. **바코드 DB에 없는 제품을 커버하는 핵심 폴백.**
- **정확도 이슈**: 작은 폰트·곡면 용기·저조도에서 오탈자 발생 가능. 대응:
  - confidence('low')일 때 사용자에게 재촬영 유도 (현재 UI에 confidence 배지 있음 — 활용).
  - 추출 성분을 **식약처 성분 마스터와 fuzzy 매칭**해 정규화(오탈자 교정) → 판정 신뢰도↑.
  - 모델 최신화 검토: 현재 `gemini-2.0-flash`. 프로젝트 표준은 **Gemini 3 Flash**(CLAUDE.md) — OCR 경로도 상향 통일 검토(정확도·비용).
- 대안(불필요): 별도 OCR SDK(구글 Vision/네이버 CLOVA OCR)보다 이미 쓰는 Gemini 멀티모달이 성분명 의미이해+INCI 변환까지 한 번에 하므로 **현 구조 유지**가 P4(단순화)에 부합.

## 제안: 이룸식 "제품 판정" 설계 스케치

### 설계 원칙 — 등급이 아니라 "나와의 적합도"

```
Yuka:  제품 → [절대 안전등급 0-100]           (탈맥락 · 공포 · 소송 리스크)
이룸:  제품 + 내 5축 프로필 → [나와의 적합도]   (맥락 · 유용 · 법적 안전 · 차별점)
```

이미 `analyzeCompatibility()`가 이 방향으로 작동 중. 이를 **전면 브랜딩**하고 데이터·근거로 보강.

### 4-레이어 판정 구조 (기존 자산 재사용)

1. **성분 안전 레이어** (사실 기반): 식약처 배합금지/제한 여부, 알레르기 유발 착향제 25종, (참고)EWG. → "위험"이 아니라 "**규제 정보**"로 표시.
2. **개인 적합도 레이어** (차별점, 이미 구현): S-1 피부타입·민감도·고민 + PC-1 톤 → `overallScore` + 좋은점/경고. **이룸만 가능** (경쟁사는 사용자 분석 프로필이 없음).
3. **성분 궁합 레이어** (이미 구현): 함께 쓰면 안 되는 조합 경고 → "내 화장대(제품함) 전체" 궁합으로 확장 여지.
4. **효과 타임라인 레이어** (신규): 성분→기대효과→발현시기(문헌 인용). "변화 추적"과 결합해 재방문 유도.

### 데이터 소스 선택 (확정 권장)

- **1차**: 식약처 화장품 원료성분정보 API + 규제정보 API + 알레르겐 25종 고시 → `cosmetic_ingredients` 벌크 임포트 스크립트(`scripts/`).
- **2차**: 효과 타임라인용 성분-효과-논문 매핑 테이블(수동 큐레이션 ~20개 핵심 활성성분, 본 문서 B-3 표가 시드).
- **유지**: Gemini OCR, OpenBeautyFacts 폴백.
- **지양**: EWG 대량 크롤링(라이선스), 화해 데이터 복제.

### 법적 안전 표현 (제품 전반 적용)

- 절대 등급/낙인 금지 → "내 프로필 적합도" 프레이밍.
- 효능은 "도움을 줄 수 있어요" + 문헌 인용, 수치·기간 단정 금지.
- 규제 사실(배합제한·알레르겐)은 그대로 인용(출처: 식약처).
- 면책문구 유지·강화, 정보제공 도구 성격 명시.
