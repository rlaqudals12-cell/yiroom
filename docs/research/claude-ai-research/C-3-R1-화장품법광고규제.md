# C-3-R1: 화장품법 광고 규제

> 화장품 제품 추천 앱의 광고 표시 의무 및 금지 표현

## 1. 리서치 배경

### 1.1 핵심 질문

이룸(Yiroom)의 화장품 제품 추천 기능이 화장품법 광고 규제에 해당하는가?

### 1.2 관련 법령

- 화장품법
- 화장품 표시·광고 관리 지침 (2025년 1월 21일 개정)
- 화장품 광고자문 기준 및 해설서

## 2. 2025년 화장품 광고 규제 주요 변경

### 2.1 식약처 지침 개정 (2025.01.21)

식품의약품안전처가 '화장품 표시·광고 관리 지침'을 개정했습니다.

**주요 개정 내용:**

| 항목 | 금지 내용 |
|------|----------|
| 의약전문가 표현 | 병원용, 피부과 전용, 의사 추천 등 |
| 인체유래 성분 | 엑소좀, 줄기세포 등 직접 표현 |
| 사용방법 오인 | 마이크로니들, 이너케어 등 |
| 피부나이 표현 | "피부나이 n세 감소" 등 |

### 2.2 금지 표현 상세

```
❌ 절대 금지 (2025년 기준):

의료 전문가 관련:
├── "OO 의사 추천"
├── "피부과 전문의 개발"
├── "병원 전용"
├── "병원 추천"
├── "약국 전용"
└── "피부과 시술용"

성분 관련:
├── "엑소좀 화장품"
├── "줄기세포 함유"
└── "인체유래 성분"

효능 관련:
├── "피부나이 10년 감소"
├── "피부나이 5세 어려짐"
├── "피부재생"
├── "피부염증감소"
├── "항염"
└── "주름개선" (일반화장품인 경우)

사용방법 관련:
├── "마이크로니들"
└── "이너케어"
```

### 2.3 위반 시 제재

| 위반 유형 | 제재 |
|----------|------|
| 허위·과대광고 | 광고업무정지 |
| 의약품 오인 광고 | 판매업무정지 |
| 반복 위반 | 사업자 등록 취소 가능 |

## 3. 이룸 서비스 적용 분석

### 3.1 제품 추천 기능 현황

| 기능 | 설명 | 광고 해당 여부 |
|------|------|---------------|
| AI 제품 추천 | 피부 분석 기반 추천 | ⚠️ 검토 필요 |
| 성분 매칭 | 사용자 피부에 맞는 성분 | ✅ 정보 제공 |
| 제품 비교 | 여러 제품 성분 비교 | ✅ 정보 제공 |
| 구매 링크 | 제휴 쇼핑몰 연결 | ⚠️ 제휴 광고 |

### 3.2 광고 vs 정보 제공 구분

```
✅ 정보 제공 (광고 아님):
├── 객관적 성분 정보 나열
├── 사용자 리뷰 표시
├── 제품 비교 기능
└── 성분 사전 제공

⚠️ 광고로 볼 수 있음:
├── 특정 제품 "추천"
├── 제휴 브랜드 우선 노출
├── 구매 유도 문구
└── 효능 강조 표현
```

### 3.3 이룸 권장 표현 가이드

| 상황 | ❌ 금지 | ✅ 권장 |
|------|--------|--------|
| 제품 소개 | "피부과 추천템" | "성분 매칭도 높음" |
| 효능 설명 | "주름 개선" | "보습 기능" |
| 전문가 언급 | "의사 추천" | "사용자 평점 높음" |
| 피부 변화 | "피부나이 감소" | "피부 컨디션 관리" |

## 4. 제휴 마케팅 광고 표시

### 4.1 제휴 링크 표시 의무

제품 추천에 제휴(어필리에이트) 링크가 포함된 경우:

```
필수 표시 사항:

"이 제품은 제휴 링크를 포함하고 있으며,
구매 시 이룸이 일정 수수료를 받을 수 있습니다."

또는 간략히:

"제휴 광고" / "Sponsored" / "AD"
```

### 4.2 표시 위치

| 위치 | 요건 |
|------|------|
| 추천 카드 상단 | 🏷️ "제휴" 뱃지 |
| 구매 버튼 근처 | 광고 표시 문구 |
| 상세 페이지 | 제휴 관계 설명 |

## 5. 화해 사례 분석

### 5.1 화해의 광고 정책

화해(1,000만+ 사용자)의 접근 방식:

| 기능 | 구현 방식 |
|------|----------|
| 제품 정보 | 성분 기반 객관적 분석 |
| 사용자 리뷰 | 실제 사용 후기 |
| 광고 구분 | "광고" 라벨 명시 |
| 추천 기준 | 성분 매칭 알고리즘 공개 |

### 5.2 화해가 피하는 표현

- 의료 효능 암시
- 특정 브랜드 편향 추천
- 과학적 근거 없는 효능 주장

## 6. 이룸 구현 가이드라인

### 6.1 UI/UX 요구사항

```
제품 추천 카드:
┌─────────────────────────────────────┐
│ [AD] 🏷️                             │  ← 제휴 상품인 경우
├─────────────────────────────────────┤
│ [제품 이미지]                        │
│                                     │
│ 제품명                              │
│ 브랜드                              │
│                                     │
│ 📊 성분 매칭도: 92%                  │  ← 객관적 지표
│ ⭐ 사용자 평점: 4.5                  │  ← 실제 리뷰 기반
│                                     │
│ [상세 보기]    [구매하기]            │
│                                     │
│ 💡 이 추천은 피부 타입 분석 기반입니다 │  ← 추천 근거 명시
└─────────────────────────────────────┘
```

### 6.2 금지 UI 패턴

```
❌ 금지:
├── "피부과 추천 제품" 라벨
├── "의사 인증" 뱃지
├── "피부 나이 10살 감소 효과" 문구
├── 제휴 표시 없는 광고 상품 우선 노출
└── 과장된 Before/After 이미지
```

### 6.3 추천 알고리즘 투명성

```typescript
// 제품 추천 시 근거 표시 필수
interface ProductRecommendation {
  productId: string;
  matchScore: number;
  matchReasons: string[];  // "건성 피부에 적합한 보습 성분"
  isSponsored: boolean;    // 제휴 여부 명시
  sponsorLabel?: string;   // "AD" 또는 "제휴"
}
```

## 7. 체크리스트

### 7.1 출시 전 필수 확인

- [ ] 모든 제품 설명에서 금지 표현 제거
- [ ] 제휴 상품 "AD" 라벨 표시
- [ ] 추천 근거 표시 기능 구현
- [ ] 화장품 광고자문 기준 검토 완료
- [ ] 이용약관에 광고 정책 명시

### 7.2 정기 점검 사항

- [ ] 신규 금지 표현 업데이트 반영 (분기별)
- [ ] 제휴 브랜드 광고 정책 준수 확인
- [ ] 사용자 신고 모니터링
