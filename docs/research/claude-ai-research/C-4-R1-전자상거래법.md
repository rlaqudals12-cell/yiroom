# C-4-R1: 전자상거래법

> 제휴 마케팅 및 커머스 기능의 법적 준수사항

## 1. 리서치 배경

### 1.1 핵심 질문

이룸(Yiroom)의 제휴 마케팅(어필리에이트) 및 커머스 연동 기능에 적용되는 법적 의무는?

### 1.2 관련 법령

- 전자상거래 등에서의 소비자보호에 관한 법률 (전자상거래법)
- 표시·광고의 공정화에 관한 법률 (표시광고법)
- 추천·보증 등에 관한 표시·광고 심사지침
- 기만적인 표시·광고 심사지침 (2025년 개정)

## 3. 제휴 마케팅 표시 의무

### 3.1 경제적 이해관계의 범위

다음 모든 경우에 표시 의무 발생:

| 유형 | 예시 | 표시 필요 |
|------|------|----------|
| 금전적 대가 | 현금, 상품권, 적립금 | ✅ |
| 상품 제공 | 무료 제공, 무료 대여 | ✅ |
| 할인 혜택 | 특별 할인 코드 | ✅ |
| 수익 배분 | 어필리에이트 커미션 | ✅ |
| 협업 수익 | 공동구매 수익 | ✅ |
| 고용 관계 | 직원, 계약직 | ✅ |
| 미래 대가 | 조건부 보상 약속 | ✅ |

### 3.2 표시 4대 원칙

| 원칙 | 요건 | 예시 |
|------|------|------|
| **접근성** | 쉽게 찾을 수 있는 위치 | 콘텐츠 상단 또는 제목 |
| **인식가능성** | 눈에 띄는 형태 | 명확한 폰트, 색상 |
| **명확성** | 이해하기 쉬운 표현 | "광고", "제휴", "협찬" |
| **언어 동일성** | 콘텐츠와 같은 언어 | 한국어 콘텐츠 → 한국어 표시 |

### 3.3 플랫폼별 표시 방법

**앱 내 제품 추천:**

```
┌─────────────────────────────────────┐
│ 💎 AD  또는  🏷️ 제휴               │  ← 상단 명확히 표시
├─────────────────────────────────────┤
│ [제품 이미지]                        │
│                                     │
│ 제품명                              │
│                                     │
│ 💬 "구매 시 이룸이 수수료를 받을 수  │
│     있습니다"                        │  ← 하단 설명
└─────────────────────────────────────┘
```

**권장 표시 문구:**

| 상황 | 표시 문구 |
|------|----------|
| 제휴 링크 | "AD", "광고", "제휴" |
| 협찬 제품 | "협찬", "제공" |
| 커미션 발생 | "구매 시 수수료 발생" |

## 4. 다크패턴 규제 (2025.02.14 시행)

### 4.1 금지되는 6가지 다크패턴

| 유형 | 설명 | 이룸 적용 |
|------|------|----------|
| **위장 광고** | 광고를 콘텐츠로 위장 | 제품 추천에 AD 표시 |
| **거짓 희소성** | 가짜 재고 부족 알림 | 사용 금지 |
| **거짓 사회적 증거** | 조작된 리뷰/평점 | 실제 데이터만 표시 |
| **강제 행동** | 구매 강요 팝업 | 사용 금지 |
| **구독 함정** | 해지 어렵게 설계 | 쉬운 해지 제공 |
| **개인정보 착취** | 과도한 정보 수집 | 최소 수집 원칙 |

### 4.2 이룸 금지 패턴 체크

```
❌ 금지 패턴:

제품 추천:
├── "지금 안 사면 품절!" (거짓 희소성)
├── "1,234명이 보는 중" (조작 가능)
└── 광고 표시 없는 협찬 제품

구독/결제:
├── 해지 버튼 숨기기
├── 자동 갱신 고지 누락
└── 환불 절차 복잡하게 설계

정보 수집:
├── 필수 아닌 정보를 필수로 요구
└── 동의 철회 어렵게 설계
```

## 5. 이룸 구현 가이드라인

### 5.1 제품 추천 화면 요구사항

```typescript
interface ProductCard {
  // 필수 표시 정보
  productName: string;
  brand: string;
  price: number;

  // 제휴 관련 (필수)
  isAffiliate: boolean;
  affiliateLabel: 'AD' | '제휴' | '광고';
  affiliateDisclosure: string;  // "구매 시 수수료 발생"

  // 추천 근거 (투명성)
  matchScore: number;
  matchReasons: string[];
}
```

### 5.2 구매 링크 처리

```typescript
// 제휴 링크 클릭 시
function handleAffiliateClick(product: ProductCard) {
  // 1. 제휴 관계 알림 (첫 클릭 시)
  if (!hasAcknowledgedAffiliate) {
    showDisclosure(
      "이 링크를 통해 구매 시 이룸이 소정의 수수료를 받습니다. " +
      "이는 서비스 운영에 도움이 됩니다."
    );
  }

  // 2. 외부 링크로 이동
  openExternalLink(product.affiliateUrl);

  // 3. 클릭 로깅 (투명성)
  logAffiliateClick(product.id);
}
```

### 5.3 이용약관 필수 조항

```markdown
## 제휴 마케팅 정책

1. **제휴 링크 안내**
   - 이룸은 일부 제품 추천에 제휴(어필리에이트) 링크를 포함합니다.
   - 해당 링크를 통해 구매 시 이룸이 판매자로부터 수수료를 받을 수 있습니다.
   - 제휴 관계는 추천 알고리즘에 영향을 주지 않습니다.

2. **광고 표시**
   - 제휴 상품은 "AD" 또는 "제휴" 라벨로 표시됩니다.
   - 협찬/제공 받은 제품은 명확히 표시됩니다.

3. **추천 기준**
   - 제품 추천은 사용자의 피부 분석 결과를 기반으로 합니다.
   - 제휴 관계가 추천 순위에 영향을 주지 않습니다.
```

## 6. 위반 시 제재

### 6.1 전자상거래법 위반

| 위반 유형 | 제재 |
|----------|------|
| 표시 의무 위반 | 시정명령, 과태료 |
| 부당 표시·광고 | 시정명령, 과징금 |
| 다크패턴 사용 | 시정명령, 과태료 |

### 6.2 표시광고법 위반

| 위반 유형 | 제재 |
|----------|------|
| 기만적 광고 | 시정명령, 과징금 (매출액 2% 이내) |
| 경제적 이해관계 미표시 | 시정명령, 과태료 |

## 7. 체크리스트

### 7.1 출시 전 필수

- [ ] 모든 제휴 상품에 "AD" 라벨 표시
- [ ] 경제적 이해관계 공개 문구 삽입
- [ ] 이용약관에 제휴 마케팅 정책 명시
- [ ] 다크패턴 6가지 유형 점검 및 제거
- [ ] 구독 해지 절차 간소화 확인

### 7.2 정기 점검

- [ ] 공정위 가이드라인 업데이트 반영 (반기별)
- [ ] 제휴 링크 정상 작동 확인
- [ ] 사용자 신고 모니터링
