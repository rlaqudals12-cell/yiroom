# 3D 체형 측정·아바타 시각화 리서치 및 로드맵 (합격 후)

> 2026-07-06 웹 리서치 기반. 상태: **파킹 (합격 후 착수)**
> 결정: 치수 추정(cm) 대신 **SMPL 아바타 시각화**를 우선 경로로 채택.

## 1. 배경 질문

"시중 3D 체형 측정 스캐너의 원리를 응용해 스마트폰만으로 최대한 따라잡을 수 있나?"

## 2. 리서치 결론 요약

### 상용 스캐너 원리와 실측 정확도

| 방식               | 예시            | 현실                                                             |
| ------------------ | --------------- | ---------------------------------------------------------------- |
| 구조광/적외선 부스 | TC2, Styku      | 마케팅은 mm급, 독립 연구에선 기기 간 가슴둘레 ±2~5cm 상호 불일치 |
| 레이저             | Human Solutions | 의류산업 기준기 (최고 정확도)                                    |
| ToF/LiDAR          | 아이폰 Pro      | 근거리 부위용, 전신 열위                                         |

### 스마트폰 단독 상한선 (하드웨어 없이)

- 표준 패턴 = **정면+측면 2장 + 키·몸무게 입력 + 밀착 의류·A포즈**
- 최적 조건에서 둘레 오차 **±1~3cm** (Amazon MeasureNet: 줄자와 통계적 동등, npj Digital Medicine 검증), 일반 자유 촬영은 ±3~5cm+
- 단안 사진의 **스케일 모호성**이 본질 한계 → 키 입력 필수
- 오픈소스 경로: MediaPipe → SMPL/SMPL-X 피팅. 형상 SOTA는 **SHAPY** (CVPR 2022, 코드 공개)
- ⚠️ **SMPL 에셋 라이선스 (2026-07-08 보강)**: SMPL 모델 파일은 **비상업 연구 전용** — 상용 탑재는 Meshcapade 유료 라이선스 필요 ([모델 라이선스](https://smpl.is.tue.mpg.de/modellicense.html), [상용 창구](https://meshcapade.com/smpl/)). → ADR-110에서 "SMPL 원리 차용 + 절차적 파라메트릭 메시(에셋 비의존)"로 결정
- VLM(Gemini류)에 사진→치수 추정은 **비신뢰** (수치 회귀 부적합, 모델별 ICC 0.07~0.81)
- 산업 교훈: **ZOZOSUIT 실패** — 정밀도가 아니라 기대-실망 갭이 서비스를 죽임. cm 수치 노출은 리스크

### 시장 신호

- 측정 단독 비즈니스는 대부분 사이즈 추천 B2B로 피벗 (3DLOOK, Bodygram) 또는 소멸 (Presize→Meta 인수 후)
- 성공 사례들의 셀링포인트는 숫자가 아니라 **"아바타 변화 시각화"**

## 3. 이룸 로드맵 결정

| 경로                          | 판정                 | 이유                                                                         |
| ----------------------------- | -------------------- | ---------------------------------------------------------------------------- |
| (a) 골격 타입 분류 (현행)     | ✅ 유지              | 비율 기반 = 절대 치수 불필요 → 스케일 모호성 회피, 기술적으로 가장 방어 가능 |
| (b) 치수 추정 ±3~5cm          | ⚠️ 조건부            | 키 입력+2장 촬영 UX 감수할 때만. 노출은 cm가 아닌 "범위/추이"로              |
| (c) **SMPL 3D 아바타 시각화** | 🎯 **합격 후 1순위** | 수치 논란 없이 "시각적 자기이해" 가치. 가성비 최고                           |

### (c) 착수 시 스케치

1. 현행 body-v2 (MediaPipe 측정 우선 + S/W/N)의 랜드마크 → SMPL β 파라미터 근사
2. 웹: Three.js SMPL 메시 렌더 (회전 가능한 내 체형 아바타)
3. 프로필 카드 체형 축의 시각 자산으로 통합 (ADR-109 "채워지는 프로필"과 정합)
4. 변화 추이: 재분석 시 아바타 오버레이 비교 ("숫자보다 변화를 보여줘라")
