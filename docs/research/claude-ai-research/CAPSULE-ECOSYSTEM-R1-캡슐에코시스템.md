## 1. 핵심 문제 정의

### 1.1 "분석만 하는 앱" 문제

이룸은 현재 **Stage 1 (Knowing)** 에만 머물러 있다:

```
Stage 1: Knowing    — "내 피부타입은 복합성이에요"     ← 현재 이룸
Stage 2: Practicing — "그래서 이 루틴을 따라해요"       ← 미달
Stage 3: Habit      — "매일 자동으로 실천하고 있어요"   ← 미달
Stage 4: Transform  — "3개월 후 피부가 좋아졌어요"      ← 미달
```

**근본 원인**: 분석 결과를 일상 실천으로 연결하는 시스템 부재.

### 1.2 궁극의 앱 (P1) 재정의

> "온전한 나를 알고(Know), 실천하고(Practice), 습관화하여(Habit), 변화를 체감(Transform)하는 통합 웰니스 플랫폼"

| 단계    | 현재               | 목표                 | 핵심 기능   |
| ------- | ------------------ | -------------------- | ----------- |
| Stage 1 | ✅ 9모듈 분석 완료 | 유지                 | AI 분석     |
| Stage 2 | ❌ 결과만 보여줌   | ✅ 캡슐 추천         | 캡슐 시스템 |
| Stage 3 | ❌ 반복 유도 없음  | ✅ One-Button Daily  | 습관 루프   |
| Stage 4 | ❌ 변화 추적 없음  | ✅ Before/After 연동 | 변화 검증   |

## 2. 캡슐 원리 범용화

### 2.1 "캡슐" 개념의 기원

캡슐 옷장(Capsule Wardrobe)은 1970년 Susie Faux가 제안한 개념:

- 최소한의 아이템으로 최대한의 조합을 만드는 전략
- 핵심: **큐레이션(선별)** + **호환성(상호 매칭)** + **개인화(나에게 맞는)**

### 2.2 9모듈 확장 가능성

캡슐 원리는 옷장에만 한정되지 않는다. **"선택지가 과도하고, 개인화가 필요하며, 반복 사용되는 모든 영역"**에 적용 가능:

| 모듈            | 선택지 과잉    | 개인화 필요 | 반복 사용     | 캡슐 적용                  |
| --------------- | -------------- | ----------- | ------------- | -------------------------- |
| PC-1 퍼스널컬러 | 수천 가지 색상 | 시즌별 최적 | 매일 코디     | ✅ 시즌 팔레트 캡슐        |
| S-1 피부        | 수백 가지 제품 | 피부타입별  | 매일 루틴     | ✅ 5-Step 스킨케어 캡슐    |
| C-1 체형        | 수백 가지 운동 | 체형별 최적 | 주 3-5회      | ✅ Magnificent 7 운동 캡슐 |
| H-1 헤어        | 수백 가지 제품 | 모발타입별  | 매일 관리     | ✅ 3-Step 헤어케어 캡슐    |
| M-1 메이크업    | 수천 가지 제품 | 시즌+피부별 | 매일 메이크업 | ✅ 데일리 메이크업 캡슐    |
| OH-1 구강건강   | 수십 가지 제품 | 구강상태별  | 매일 2-3회    | ✅ 구강관리 캡슐           |
| W-1 운동        | 100+ 운동      | 체형+목표별 | 주 3-5회      | ✅ 주간 운동 캡슐          |
| N-1 영양        | 무한한 식품    | 영양상태별  | 매일 3끼      | ✅ 주간 식재료 캡슐        |
| R-1 리포트      | 9모듈 데이터   | 사용자별    | 일/주/월      | ✅ 캡슐 대시보드           |

### 2.3 캡슐 원리 5가지 (C1-C5)

**C1. 큐레이션 (Curation)**: 무한한 선택지 → 핵심 N개로 축소

- 옷장: 100벌 → 33벌
- 스킨케어: 50제품 → 5-Step 루틴
- 운동: 100+ → Magnificent 7
- 식재료: 무한 → 주간 7 필수 식재료

**C2. 호환성 (Compatibility)**: 선택된 아이템 간 상호 매칭

- 옷장: 상의-하의 색상 조화 (PC-1 기반)
- 스킨케어: 성분 간 상호작용 (ingredient_interactions)
- 운동: 근육군 밸런스 (상체-하체-코어)
- 식재료: 영양소 보완성 (단백질-탄수화물-지방)

**C3. 개인화 (Personalization)**: 분석 결과 기반 최적화

- 옷장: PC-1 시즌 + C-1 체형
- 스킨케어: S-1 피부타입 + 알레르기
- 운동: W-1 운동타입 + C-1 체형
- 식재료: N-1 영양상태 + 건강 목표

**C4. 로테이션 (Rotation)**: 주기적 교체/갱신

- 옷장: 계절 전환 (봄여름/가을겨울)
- 스킨케어: 계절 제품 전환 (보습↔유분)
- 운동: 4-6주 루틴 변경 (근육 적응 방지)
- 식재료: 제철 식재료 교체

**C5. 미니멀리즘 (Minimalism)**: 적을수록 좋다

- 의사결정 피로 감소 (Decision Fatigue)
- 선택의 역설 해소 (Paradox of Choice)
- "충분히 좋은" 선택 (Satisficing > Maximizing)

### 2.4 모듈별 캡슐 상세 설계

#### 캡슐 스킨케어 (S-1 기반)

```
입력: S-1 피부타입 + 알레르기 + 계절
출력: 5-Step 루틴

Step 1: 클렌저 (피부타입 매칭)
Step 2: 토너 (pH 밸런스)
Step 3: 에센스/세럼 (핵심 성분 — 피부 고민별)
Step 4: 크림 (보습 — 계절별 조절)
Step 5: 자외선 차단 (연중)

교체 주기: 계절 전환 시 Step 3, 4 재평가
호환성: Step 3 성분 vs Step 4 성분 상호작용 검증
```

#### 캡슐 운동 (W-1 + C-1 기반)

```
입력: W-1 운동타입 + C-1 체형 + 운동 경력
출력: "Magnificent 7" 주간 루틴

Mon: 스쿼트 (하체)
Tue: 벤치프레스 (상체 미기)
Wed: 데드리프트 (후면 사슬)
Thu: 휴식 또는 유산소
Fri: 오버헤드프레스 (어깨)
Sat: 로우/풀업 (등)
Sun: 코어 + 유연성

교체 주기: 4-6주 (근육 적응)
개인화: 체형별 볼륨/강도 조절
```

#### 캡슐 영양 (N-1 기반)

```
입력: N-1 영양상태 + 건강 목표 + 알레르기
출력: 주간 7 필수 식재료

1. 단백질원 (닭가슴살/두부/연어)
2. 복합 탄수화물 (현미/고구마/귀리)
3. 건강 지방 (아보카도/올리브유/견과류)
4. 녹색 채소 (시금치/브로콜리/케일)
5. 컬러 채소 (당근/파프리카/토마토)
6. 유제품/대체품 (요거트/두유)
7. 제철 과일 (계절별 교체)

교체 주기: 주 1회 (장보기 사이클)
호환성: 영양소 보완성 검증
```

### 3.2 페르소나 상세 (클러스터별 대표)

**A-1 수빈 (17세, 고등학생)**

- 첫 스킨케어 시작, YouTube에서 정보 습득하지만 뭐가 맞는지 모름
- 니즈: "내 피부에 맞는 기초 3개만 알려줘"
- 캡슐 기대: ✅ 스킨케어 입문 캡슐 (3-Step 간소화)

**B-3 지영 (28세, 뷰티 블로거)**

- 50개 이상 제품 보유, 성분 분석에 관심
- 니즈: "이 성분 조합이 안전한지 확인하고 싶어"
- 캡슐 기대: ✅ 호환성 엔진 + 카메라 성분 분석

**C-2 현우 (32세, IT 엔지니어)**

- 스킨케어 3개, 옷장 미니멀, 효율 극대화
- 니즈: "최소 아이템으로 최대 효과"
- 캡슐 기대: ✅ 핵심 캡슐 원리 그 자체

**D-4 미래 (26세, 헬스 트레이너)**

- 운동+영양+피부 통합 관리, 보충제 복용
- 니즈: "운동 루틴과 식단이 연동되면 좋겠어"
- 캡슐 기대: ✅ 운동+영양 크로스 캡슐

**E-1 정호 (35세, 대기업 과장)**

- 아침 준비 15분, 결정 피로 심각
- 니즈: "앱 켜면 오늘 뭐 할지 바로 보여줘"
- 캡슐 기대: ✅ One-Button Daily

**F-5 태민 (24세, 대학원생)**

- 남성 기초 관리 시작, "너무 여성 중심"이라고 느낌
- 니즈: "남자도 쓸 수 있는 실용적 가이드"
- 캡슐 기대: ✅ 성별 중립 캡슐 (Phase K 완료)

**G-2 서연 (22세, 패션학과)**

- 매일 코디 고민, 가상 피팅 원함
- 니즈: "이 옷이 나한테 어울리는지 미리 볼 수 있으면"
- 캡슐 기대: ✅ 가상 피팅 L1-L2 + 상황별 캡슐

**H-3 은지 (33세, 임산부)**

- 임신 중 화장품/식품 성분 걱정
- 니즈: "임산부에게 안전한 제품인지 바로 확인"
- 캡슐 기대: ✅ Safety Profile (CRITICAL)

**I-4 준혁 (21세, 대학생)**

- 월 용돈 30만원, 가성비 추구
- 니즈: "적은 비용으로 괜찮은 루틴"
- 캡슐 기대: ✅ 예산 캡슐 + Cost-per-wear

**J-1 영숙 (48세, 주부)**

- 스마트폰 조작 서투름, 건강 관심 높음
- 니즈: "복잡하지 않고 건강에 도움되는 가이드"
- 캡슐 기대: ✅ 심플 캡슐 + 큰 글씨

| 사용자 커버리지 (5)    | 55명 중 몇 클러스터가 원하는가?  |
| 기술 실현 가능성 (5)   | 현재 기술로 구현 가능한가?       |
| 기존 시스템 시너지 (5) | 기존 코드/DB를 활용할 수 있는가? |

| 기능                 | 궁극 | 빈도 | 커버 | 기술 | 시너지 | 총점   |
| -------------------- | ---- | ---- | ---- | ---- | ------ | ------ | --- |
| F1 캡슐 옷장         | 5    | 5    | 5    | 5    | 5      | **25** |
| F8 카메라 성분       | 5    | 5    | 5    | 5    | 5      | **25** |

`*` 이전 대화에서 재평가한 점수

**사용 빈도 — 핵심 인사이트**:

- **매일**: One-Button Daily, 캡슐 옷장 코디
- **주 3-5회**: 카메라 성분 (장보기/쇼핑)
- **계절 전환**: 시즌 전환 (연 4회이나 영향 범위 큼)
- **월 1-2회**: 분석 재실행, Before/After 비교

### 4.2 가상 피팅 재정의

기존 SDD-VIRTUAL-TRY-ON-EXTENSION.md는 **L3 (3D 아바타)** 중심.
캡슐 맥락에서는 **L1-L2가 실용적**:

| 레벨         | 내용                                | 난이도 | 기존 기반           |
| ------------ | ----------------------------------- | ------ | ------------------- |
| L1 색상 매칭 | PC-1 시즌 팔레트로 색상 어울림 판단 | 낮음   | 드레이핑 시뮬레이션 |
| L2 핏 판단   | C-1 체형으로 실루엣 어울림 판단     | 중간   | 체형 분류 결과      |
| L3 3D 아바타 | 실제 의상을 3D로 입혀보기           | 높음   | 미구현 (향후)       |

### 4.3 카메라 성분 분석 재평가

기존 SDD-INGREDIENT-ANALYSIS.md는 **텍스트 검색 기반**.
카메라 성분 분석은 **현장 즉시 판단**:

```
매장에서 제품 발견
  → 바코드 스캔 (기존 barcode_products DB 조회)
  → DB에 없으면: 성분표 OCR → AI 분석
  → Safety Profile과 교차 검증 (알레르기, 임산부 금지 성분)
  → 캡슐 호환성 점수 표시 ("현재 스킨케어 캡슐과 85% 호환")
  → 구매 판단 도움
```

**사용 빈도**: 주 3-5회 (장보기, 드러그스토어 방문, 온라인 쇼핑 시)
→ 이룸의 **가장 높은 빈도 기능** 중 하나

## 5. 미니멀리즘 vs 어필리에이트 모순 분석

### 5.1 문제 정의

```
캡슐 철학: "5개만 쓰세요" (제품 수 줄이기)
비즈니스:  "이 제품 사세요" (어필리에이트 수익)
```

### 5.2 해결 전략

| 전략             | 설명                                  | 예시                                |
| ---------------- | ------------------------------------- | ----------------------------------- |
| **교체 추천**    | "줄이기"가 아닌 "더 나은 것으로 교체" | "현재 세럼보다 이 성분이 더 맞아요" |
| **보충 추천**    | 캡슐에 빠진 것 채우기                 | "자외선 차단이 캡슐에 없네요"       |
| **시즌 전환**    | 계절 교체 시 자연스러운 구매 유도     | "겨울 보습 크림으로 교체할 때에요"  |
| **CPW 프레이밍** | 가격이 아닌 "착용당 비용"으로 제시    | "이 코트는 착용당 500원이에요"      |

**핵심**: "사세요"가 아닌 **"바꾸세요" + "채우세요"** → 사용자 가치와 수익 정렬

## 8. 시뮬레이션 검증 방법론

### 8.1 검증 매트릭스

각 Phase 완료 후 55명 시뮬레이션 재실행:

- 5축(기능/감정/개인화/신뢰/재방문) × 10클러스터
- 가중평균 계산, 이전 Phase 대비 향상 확인
- **목표**: Phase 8 완료 시 가중평균 90점 이상

### 8.2 역산 검증

```
가중평균 = Σ (클러스터 인원 × 클러스터 점수) / 55

현재:   (6×62 + 6×70 + 5×58 + 6×65 + 6×55 + 6×60 + 5×68 + 5×48 + 5×58 + 5×50) / 55
      = 3292 / 55 = 59.9 ≈ 60점 ✓

목표:   (6×88 + 6×93 + 5×92 + 6×93 + 6×92 + 6×88 + 5×92 + 5×90 + 5×90 + 5×86) / 55
      = 4993 / 55 = 90.8 ≈ 91점 ✓
```
