# Yiroom 만 14세 미만 미성년자 차단 완전 가이드

**핵심 결론**: 한국 개인정보보호법 제22조의2에 따라 Yiroom처럼 얼굴/신체 이미지를 수집하는 서비스는 만 14세 미만 아동의 가입을 **완전 차단**하는 방식을 권장한다. 올리브영, 무신사 등 국내 뷰티/패션 앱도 동일한 정책을 채택하고 있으며, 법정대리인 동의 시스템 구현의 복잡성과 민감정보 처리의 법적 리스크를 고려할 때 이 방식이 가장 안전하다. 위반 시 **5년 이하 징역 또는 5천만원 이하 벌금**, 매출액 3% 과징금이 부과될 수 있다.

---

## 법적 요건의 핵심 조항

개인정보보호법 제22조의2(아동의 개인정보 보호)는 만 14세 미만 아동의 개인정보 수집 시 **법정대리인의 동의**를 받아야 하며, 동의 여부를 **확인**해야 한다고 규정한다. Yiroom이 수집하는 얼굴 이미지는 **생체인식정보**로서 민감정보에 해당하여 더욱 엄격한 보호가 필요하다.

**필수 준수 사항:**
- 만 14세 미만 아동 개인정보 수집 시 법정대리인 동의 필수
- 동의 확인 방법: 휴대전화 본인인증, SMS 문자 확인, 신용카드 정보, 서면 동의 등
- 아동에게 고지 시 **이해하기 쉬운 양식과 명확한 언어** 사용 의무
- 법정대리인의 성명과 연락처는 동의 없이 아동으로부터 직접 수집 가능

**위반 시 제재 수준:**

| 위반 내용 | 제재 |
|----------|------|
| 법정대리인 동의 없이 아동 개인정보 수집 | 5년 이하 징역 또는 5천만원 이하 벌금 |
| 법정대리인 동의 확인 미이행 | 전체 매출액의 3% 이하 과징금 |
| 개인정보 파기 의무 위반 | 3천만원 이하 과태료 |
| 동의 받는 방법 위반 | 1천만원 이하 과태료 |

```
③ 본 서비스는 얼굴 이미지, 손목 이미지 등 개인의 신체적 특징과

④ 만 14세 미만의 아동이 법정대리인의 동의 없이 회원가입을 한 경우,

⑤ 이용자가 허위의 생년월일을 입력하여 만 14세 이상으로 가장한 경우, 
이로 인해 발생하는 모든 법적 책임은 해당 이용자(또는 법정대리인)에게 
있습니다.
```

## 미성년자 데이터 삭제 절차와 기한

**만 14세 미만으로 판명된 경우** 개인정보 파기 기한은 명확하다. 표준 개인정보 보호지침 제13조에 따르면 법정대리인이 동의를 거부하거나 동의 여부가 확인되지 않은 채 **5일이 경과한 경우** 해당 개인정보를 파기해야 한다. 연령 확인 시점에서 만 14세 미만으로 확인되어 가입이 차단된 경우에는 **지체 없이 즉시 파기**하는 것이 원칙이다.

**Yiroom 특수 상황**: 얼굴 이미지는 생체인식정보로서 민감정보에 해당하므로, 사후에 만 14세 미만으로 밝혀진 경우 수집된 모든 이미지와 신체 정보를 **즉시 파기**해야 한다. 접속기록은 민감정보 처리 기준으로 **2년 이상 보관**이 필수다.

**파기 기록 관리 항목:**
- 파기 일시 (정확한 날짜 및 시간)
- 파기 대상 (삭제된 개인정보 항목 및 건수)
- 파기 방법 (영구삭제, 덮어쓰기 등)
- 파기 담당자 정보
- 개인정보 보호책임자 확인 서명

전자적 파기 방법으로는 전용 소자장비(Degausser), 초기화/덮어쓰기, 영구삭제 소프트웨어가 허용되며, 데이터가 **복원되지 않도록** 수행해야 한다.

## 연령 우회와 면책 조항의 효력

개인정보보호위원회는 아동이 부모의 휴대폰이나 개인정보를 이용해 허위로 동의한 경우를 **기망 동의**로 분류한다. 이는 사업자의 통제 범위를 벗어나므로 **합리적인 연령 확인 조치**를 구현했다면 사업자 책임이 면책될 수 있다.

**면책을 위한 최소 요건:**
- 생년월일 직접 입력 또는 "만 14세 이상입니다" 체크박스 구현
- 만 14세 미만 선택/입력 시 가입 페이지 접근 차단
- 앱스토어 연령 등급 14세 이상 설정

**휴대폰 본인인증 vs 단순 생년월일 입력**: 본인인증은 정보통신망법 제23조의3에 따른 공인 방식으로 법적 효력이 강하고, 허위 입력 시 사업자 면책 가능성이 높다. 단순 생년월일 입력은 합리적 조치로 인정되나 본인인증보다 효력이 약하다.

**유효한 면책 조항 예시:**
```
회사가 합리적인 연령 확인 조치(생년월일 입력, 만 14세 이상 확인 등)를 
이행하였음에도 이용자의 허위 정보 입력으로 인해 만 14세 미만 아동이 
서비스를 이용한 경우, 회사는 이로 인한 책임을 지지 않습니다.
```

단, 약관규제법 제7조에 따라 **고의 또는 중대한 과실**로 인한 책임을 배제하는 조항은 무효이므로, 면책 범위를 명확히 한정해야 한다.

## 국내외 서비스 사례 비교 분석

### 완전 차단 방식 (권장)

**무신사, 올리브영**: 만 14세 미만 회원가입을 완전히 제한하며, 체크박스 동의와 본인인증을 조합해 사용한다. 개인정보처리방침에 "만 14세 미만 아동의 개인정보를 수집하지 않습니다"를 명시한다.

**당근마켓**: 만 14세 이상만 가입 가능하며 본인인증 없이 자기신고 방식(체크박스)을 사용한다. 내부 모니터링과 신고 시스템으로 14세 미만 계정을 탐지하며, 2023년 기준 연간 약 22,000건의 신고를 처리했다.

**틱톡**: 한국에서 만 14세 이상만 가입 가능하며, 에이지 게이트로 14세 미만 입력 시 **동일 기기에서 다른 생년월일로 재가입을 차단**한다. 안전 심사팀이 14세 미만 사용자 징후를 파악하도록 훈련받는다.

### 법정대리인 동의 방식

**카카오**: 만 14세 미만도 가입 가능하나 법정대리인 휴대폰 인증 필수. 패밀리 계정 서비스로 만 19세 이상 성인이 아동 계정을 관리할 수 있다. 아동용 동의문을 알기 쉬운 언어로 별도 제공한다.

**토스**: 만 7~13세는 "유스 홈" 전용 서비스를 제공하며, **법정대리인이 자녀 계정을 직접 개설**해야 한다. 자녀 단독 가입은 불가능하다.

**인스타그램**: 2025년 1월부터 한국에서 '10대 계정' 기능을 도입했다. 만 14~18세 사용자에게 계정 비공개 기본 설정, 60분 사용 제한, DM 제한 등을 적용하고 부모 관리감독 기능을 제공한다.

## Clerk과 React Native 기술 구현

### Clerk 메타데이터 활용

Clerk는 세 가지 메타데이터를 제공하는데, 생년월일은 **unsafeMetadata**로 회원가입 시 임시 저장한 후 백엔드 검증을 거쳐 **publicMetadata**로 이전하는 것을 권장한다.

```typescript
// 회원가입 시 unsafeMetadata에 임시 저장
await signUp.create({
  emailAddress,
  password,
  unsafeMetadata: {
    birthdate: '2000-01-15', // YYYY-MM-DD 형식
  },
});

// 백엔드에서 검증 후 publicMetadata로 이전
await clerkClient.users.updateUserMetadata(userId, {
  publicMetadata: {
    birthdate: '2000-01-15',
    ageVerified: true,
    ageVerifiedAt: new Date().toISOString(),
  },
});
```

### 연령 계산 유틸리티

```typescript
export function calculateAge(birthdate: Date): number {
  const today = new Date();
  const birth = new Date(birthdate);
  
  let age = today.getFullYear() - birth.getFullYear();
  const monthDiff = today.getMonth() - birth.getMonth();
  
  if (monthDiff < 0 || (monthDiff === 0 && today.getDate() < birth.getDate())) {
    age--;
  }
  
  return age;
}

export function isAdult(birthdate: Date, minAge: number = 14): boolean {
  return calculateAge(birthdate) >= minAge;
}
```

### 재시도 제한 정책

```typescript
const RATE_LIMIT_CONFIG = {
  ageVerification: {
    maxAttempts: 5,      // 최대 시도 횟수
    windowMinutes: 30,   // 제한 시간 창
    lockoutMinutes: 60,  // 잠금 시간
  },
};
```

디바이스 ID와 IP 조합으로 제한하며, `expo-application`의 `getInstallationIdAsync()`와 `AsyncStorage`를 활용해 클라이언트 측 제한을 구현한다. 서버 측에서는 `express-rate-limit` 등을 사용해 이중 보호한다.

## 결론: Yiroom 권장 구현 방향

Yiroom은 **만 14세 미만 완전 차단 방식**을 채택하는 것이 최적이다. 얼굴/신체 이미지라는 민감정보를 수집하는 특성상 법정대리인 동의 시스템 구현의 복잡성과 법적 리스크가 크기 때문이다. 올리브영, 무신사와 동일한 정책으로 회원가입 시 생년월일 입력과 "만 14세 이상" 체크박스를 조합하고, 개인정보처리방침에 "만 14세 미만 아동의 개인정보를 수집하지 않습니다"를 명시하면 된다.

향후 만 14세 미만 사용자를 수용해야 할 비즈니스 필요가 있다면, 카카오나 토스처럼 **법정대리인 휴대폰 본인인증**을 통한 동의 시스템을 구축하고, 아동용 동의문을 별도로 제공하며, 기능 제한이 적용된 전용 화면을 개발해야 한다. 이 경우 법정대리인과 아동 간 최소 18세 이상 나이 차이 확인, 동의 확인 SMS 발송, 동의 기록 보관 등 추가 요건이 필요하다.
