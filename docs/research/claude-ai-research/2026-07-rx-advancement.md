# 보유 자산 기반 개인 처방 고도화 리서치 (2026-07)

> **목적**: 이룸의 "화장대/옷장 등록 → 적합성 판정 → 루틴 배치 → 빈 슬롯 구매 연결 → 목표 우선순위 → 단계 계획" 흐름을 의학적·과학적 근거 위에서 고도화하는 방법을 조사한다.
> **범위**: 스킨케어 루틴 시퀀싱, 스킨 사이클링, 경쟁 서비스, 폐루프 적응, 캡슐 옷장/화장대 이론, 어드히어런스, 법적 경계.
> **주의**: 본 문서 내 "처방(Rx)"은 편의상 창업자 용어를 따르나, 제품에서의 표현 리스크는 §7에서 별도 정리한다.

## 1. 피부과 처방 시퀀싱의 과학 — "단계 계획"의 의학적 뼈대

### 1.1 핵심 원칙 (문헌 합의)

2025년 JAAD Delphi 합의 연구가 재확인한 결론은 역설적이다: **신제품·마케팅 언어의 폭발에도 불구하고 피부과가 일관되게 권장하는 핵심 성분(자외선차단제, 레티노이드, BPO, 아젤라산, 비타민C, 하이드로퀴논, 장벽 회복제)은 거의 변하지 않았다.** 즉 "무엇을 쓰는가"보다 **"어떤 순서·속도로 도입하는가"**가 차별화 지점이다.

시퀀싱의 4대 축:

| 축                                   | 원칙                                                                                        | 근거                                                                                          |
| ------------------------------------ | ------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| **장벽 우선 (Barrier First)**        | 활성 성분 도입 전/중에 세라마이드·PLG·판테놀·병풀(Centella)로 장벽을 먼저 안정화            | 레티놀 자극의 기전이 지질 장벽 붕괴·TEWL 증가·염증이므로, 장벽 회복 성분 병용이 필수          |
| **저농도 램프업 (Low & Slow)**       | 낮은 농도·낮은 빈도로 시작해 내성 형성 후 증량. "too much, too fast → 홍반·박리·조기 이탈"  | 레티놀 내성(retinization)은 시간 의존적. 다중오믹스 연구가 장벽 기능의 "단계별 재구성"을 확인 |
| **병용 금기 (Conflict Avoidance)**   | 상호 불활성화·자극 증폭 조합 회피 (§1.3 표)                                                 | pH·산화 안정성·광민감성 기전                                                                  |
| **AM/PM 배치 (Circadian Placement)** | 항산화(비타민C·나이아신아마이드)=아침(UV 방어), 레티노이드·AHA/BHA=저녁(광민감성·야간 회복) | 광민감성 및 야간 피부 회복 주기                                                               |

### 1.2 램프업 프로토콜 (규칙화 가능)

- **주 1~2회 → 격일 → 매일**의 빈도 사다리(frequency laddering)가 표준.
- 완충 기법: "샌드위치 도포"(보습제→레티놀→보습제), 저농도(0.1% 레티놀) 시작.
- 각 단계 승급 조건 = "자극 신호 없이 N회 견딤". → **이룸이 규칙 엔진으로 인코딩하기 좋은 구조**(승급/후퇴 상태기계).

### 1.3 병용 금기·궁합 표 (규칙 엔진 시드)

| 조합                         | 판정                 | 이유                                                             |
| ---------------------------- | -------------------- | ---------------------------------------------------------------- |
| 레티놀 + 비타민C(L-ascorbic) | ✗ 분리(AM/PM)        | pH 최적점 상충(C: ≤3.5 산성 / 레티놀: 중성), 상호 효능 저하·자극 |
| 비타민C + AHA/BHA            | ✗ 분리               | 산성 중첩 → 자극 증폭                                            |
| 레티놀 + BPO                 | ✗ 분리 또는 격일     | BPO가 일부 레티노이드 산화·불활성화                              |
| AHA + BHA 동시               | ✗ 주의               | 과각질 제거 → 장벽 손상                                          |
| 레티놀 + AHA/BHA 같은 밤     | ✗ 격일 + 주 1회 휴식 | 과박리·장벽 손상                                                 |
| 레티놀 + 나이아신아마이드    | ✓ 권장               | 나이아신아마이드가 레티놀 자극 완화                              |
| 레티놀 + 히알루론산          | ✓ 권장               | 건조 상쇄                                                        |
| 비타민C + 나이아신아마이드   | ✓ (구설 반박됨)      | 정상 배합농도에선 문제 없음                                      |

## 2. 스킨 사이클링 (Skin Cycling) — 주간 스케줄 엔진의 근거

### 2.1 정의·근거 수준

피부과 전문의 Whitney Bowe가 명명. **4일 야간 주기**:

1. **1일차 — 각질제거(Exfoliation)**: AHA/BHA
2. **2일차 — 레티노이드**
3. **3일차 — 회복(Recovery)**: 보습·진정
4. **4일차 — 회복**
5. → 반복

**원리**: 자극성 성분(산·레티노이드)의 사용 빈도를 의도적으로 분산 → 장벽 무결성 보호. 피부과에서 레티노이드를 "cycling"하는 것은 이미 오래된 관행(많은 사람이 매일 못 견딤). 전문의들은 **장기 사용에 안전·적절**하다고 평가.

**근거 수준**: 정식 RCT 기반 표준이라기보다 **임상 관행의 체계화 + 전문가 컨센서스**. 주의점: 각질제거 주 2회가 일부에겐 과함(전문가 다수는 주 1회 상한 권장) → **개인화 여지**가 크다 = 이룸의 강점 지점.

### 2.2 "스케줄 엔진"으로의 규칙화

스킨 사이클링은 **결정론적 주간 캘린더**로 환원 가능하다:

- 입력: 보유 제품(각질/레티노이드/보습 슬롯 채움 여부), 피부 민감도, 실측 장벽 상태.
- 출력: 요일별 야간 루틴(N일 주기, 회복일 수를 민감도로 가변).
- 가변 규칙: 민감도↑ → 회복일 3~4일로 확장(4일→5·6일 주기), 각질 빈도↓.

이는 §1의 시퀀싱 규칙과 자연스럽게 결합 → **v2 "주기 스케줄 엔진"의 코어 알고리즘**.

## 4. 적응형 처방 (Closed-Loop) — "정체 시 교체" 규칙의 안전 설계

### 4.1 사례

- **Curology**: 환자-provider 폐루프. 진행 사진·병력·목표를 provider가 리뷰해 **포뮬러를 조정**. 핵심 = 사람이 개입하는 적응형.
- **Haut.AI**: 정량 파라미터 수준 피부 평가로 치료 기간 중 미세 생물학적 변화 추적, **최대 90일** 주기 재스캔으로 변화 추적(감사·출판 가능한 데이터).

### 4.2 "조기 교체 방지"가 핵심 안전장치

효과 데이터로 처방을 갱신할 때 **가장 큰 위험은 성분 순응 기간을 무시한 조기 교체**다. 문헌상 타임라인:

| 구간      | 현상                                                 | 시사점                          |
| --------- | ---------------------------------------------------- | ------------------------------- |
| **1~4주** | 퍼지(retinization)·홍반·일시적 악화. 흔히 3주경 정점 | **"악화"를 "실패"로 오판 금지** |
| **4주**   | 진행 사진 촬영 적기, 대개 유의미 개선 아직 없음      | 판정 유보                       |
| **6~8주** | 다수 환자가 이 시점부터 개선 체감                    | **최소 판정 게이트 = 6~8주**    |

Curology 지침: "6주까지 개선 없으면 **버리지 말고** provider에게 사진 전송", "good skin comes to those who wait".

### 4.3 안전 규칙 설계 (이룸 폐루프)

```
IF 제품_활성_성분 ∈ {레티노이드, 산} AND 사용경과 < 8주:
    → 교체 제안 금지, "적응기(퍼지 가능)" 라벨 + 장벽 케어 강화 제안
IF 사용경과 ≥ 8주 AND 목표지표(예: 기미 밝기) 정체 AND 어드히어런스 ≥ 임계:
    → "교체 후보" 표시 (단, 최종 판단은 사용자/전문가)
IF 어드히어런스 < 임계:
    → 교체 아님. "일관성 먼저" 코칭 (미사용을 실패로 오판 금지)
```

**핵심**: 교체 트리거의 전제조건 = **① 최소 순응기간 경과 + ② 실사용 어드히어런스 확보**. 둘 없이는 "정체=실패" 판정 금지.

## 5. 캡슐 옷장 이론 → "캡슐 화장대"로의 전이

### 5.1 원 이론

- **캡슐 옷장**: 1970년대 런던 부티크 오너 Susie Faux 명명. 소수의 계절 적합 아이템을 믹스매치.
- **Project 333** (Courtney Carver, 2010): **33개 이하로 3개월** 나기. 핵심 메시지 = **"충분함(enough)이 무엇인지 보여준다"**.
- **cost-per-wear (착용당 비용)**: 가격 ÷ 착용횟수. 적게·잘 쓰는 것을 정량화. Whering·Indyx가 이 지표를 소비 절감·자산 가치 프레임으로 활용.
- 근거 통계: 그린피스 조사, 동아시아·유럽 소비자 최대 50%가 필요 이상으로 구매.

### 5.2 캡슐 화장대(Capsule Vanity)로 전이 — 신뢰 프레임

| 옷장 개념        | 화장대 전이                                             | 이룸 신뢰 메시지                                      |
| ---------------- | ------------------------------------------------------- | ----------------------------------------------------- |
| 33개 캡슐        | **최소 제품 × 최대 커버리지** (5축·목표 커버 최소 세트) | "이 6개면 당신 목표에 충분합니다"                     |
| cost-per-wear    | **cost-per-use / 유통기한 대비 소진율**                 | "이미 있는데 또 사지 마세요"                          |
| 중복 아이템 지적 | **중복 성분·기능 제품 경고** (레티놀 3개, 산 4개 등)    | "겹치는 제품이 3개예요"                               |
| 소비 절감        | **빈 슬롯만 구매 연결**                                 | "덜 사게도 말해주는 앱" (수익 원칙 §CLAUDE.md와 정합) |

이 프레임은 이룸 제1가치("사용자는 영원히 무료, 구독 최소화, 덜 사게 말해준다")와 **직접 정합**하며 어필리에이트 수익과도 충돌하지 않는다(빈 슬롯 채움 = 실수요).

## 6. 행동과학 / 어드히어런스 — 지속률 설계

### 6.1 문헌 핵심 (피부과 어드히어런스)

- **레짐 단순화가 최강 레버**: 제품 수·복잡도 감소가 어드히어런스를 유의하게 향상. "간결하고 관리 가능한 루틴일수록 잘 지킨다."
- **리마인더**: 자동 문자·앱 알림이 어드히어런스·자기관리·피부중증도·삶의질을 유의 개선(아토피 문자 파일럿 등).
- **습관 스택(Habit Stacking)**: 기존 습관에 새 단계를 연결(양치 후 → 세럼). 트리거-행동 연결로 습관 형성 강화.
- **의사소통·교육·잦은 체크인**이 보조.

### 6.2 이룸 결합점

| 문헌 원칙     | 이룸 기능 결합                                              |
| ------------- | ----------------------------------------------------------- |
| 레짐 단순화   | 캡슐 화장대(최소 세트) + "오늘 할 3단계" 축약 브리핑        |
| 리마인더      | 아침/저녁·사이클링 요일 알림(§2 스케줄 엔진과 연동)         |
| 습관 스택     | "양치 후 / 샤워 후"에 루틴 앵커링 온보딩                    |
| 체크인        | 체크박스 완료 → 어드히어런스 데이터 축적 → §4 폐루프 입력   |
| 단계적 복잡도 | 초기 3단계 → 견디면 활성 성분 단계 추가(램프업 §1.2와 통합) |

**어드히어런스 데이터는 §4 폐루프의 필수 입력**이다: 정체가 "제품 문제"인지 "미사용 문제"인지 구분하는 유일한 신호.

## 7. 법적 경계 — "처방"이라는 단어의 리스크

### 7.1 핵심 원칙: **의도된 용도(intended use)·클레임이 분류를 결정한다**

- **미국 (FD&C Act)**: 제품이 "신체의 구조·기능에 영향을 주거나 질병을 진단·완화·치료·예방"한다고 **주장**하면 화장품이 아니라 **의약품(drug)**으로 분류된다. 성분이 아니라 **클레임**이 분류를 바꾼다. (2022 MoCRA로 FDA 화장품 권한 확대)
- **한국 (화장품법)**: 일반화장품 / **기능성화장품**(미백·주름·자외선차단 등) / **의약외품**(여드름·탈모 완화 등) 3분류. 기능성화장품(탈모완화·여드름·가려움·붉은선 완화 등)엔 **"질병의 예방 및 치료를 위한 의약품이 아님"** 문구 표시 의무 — 이것이 정확히 **의약품 오인 방지** 규정.

### 7.2 이룸에 대한 실전 리스크 (⚠️ 가장 중요한 1개)

**"처방(Rx/prescription)"·"치료"·"기미를 없앤다/개선을 보장" 같은 표현은 이룸(화장품 추천 앱)을 규제상 "의약품/의료행위" 영역으로 끌어들일 수 있다.** 특히:

- "처방" = 의료행위 연상 → **의료법·화장품법·표시광고법** 리스크.
- "기미 치료/제거", "N주 후 미백 전환" 같은 **질병·치료·효능 보장** 클레임 = 기능성화장품 범위 초과 시 위법.

### 7.3 안전 표현 가이드 (권고)

| 위험 표현                   | 안전 대체                                                              |
| --------------------------- | ---------------------------------------------------------------------- |
| 개인 **처방** / Rx          | 개인 맞춤 **루틴 제안 / 큐레이션 / 가이드**                            |
| 기미 **치료 / 제거**        | 기미 **케어 / 톤 관리 목표** (기능성화장품 인정 범위 내 "미백에 도움") |
| N주 후 **미백으로 전환**    | N주 후 **미백 목표 단계로 진행 제안**                                  |
| 효과 **보장 / 확실히 개선** | 일반적 경향 안내 + **개인차 있음** 고지                                |
| 의학적 **진단**             | 상태 **분석 / 참고 정보**                                              |

**필수 고지 문구(제품 내)**: "본 정보는 일반적 피부관리 참고용이며 의학적 진단·처방·치료를 대체하지 않습니다. 증상 지속 시 피부과 전문의 상담을 권장합니다." + 기능성 클레임 사용 시 "질병의 예방·치료를 위한 의약품이 아님" 병기.

> 이룸에는 이미 관련 경계 문서 존재 → 본 §7과 교차검토 권장: `docs/research/claude-ai-research/C-2-R1-의료기기법경계.md`, `S-2-*`(피부).

## 8. 이룸 처방 엔진 고도화 로드맵 스케치

> 제1원칙(P1 궁극의 형태) 관점: 궁극 = "실측 상태 ↔ 보유 자산 ↔ 목표"를 폐루프로 잇는 **개인 뷰티 컨설턴트**. 아래는 안전한 3단계 진화.

### v1 — 규칙 기반 처방 (현재 → 강화)

- §1.3 병용 금기·궁합 표를 **결정론적 규칙 엔진**으로 인코딩.
- 보유 제품 → 슬롯(세정/항산화/각질/레티노이드/보습/자차) 매핑 → 적합성 판정 + **중복 경고**(캡슐 화장대 §5).
- AM/PM 배치 + 목표 우선순위(기미·장벽 등) 정렬.
- 빈 슬롯만 구매 연결. **출력물 = 오늘의 3~5단계 브리핑 + 체크박스**(§6 단순화).

### v2 — 주기 스케줄 엔진 (사이클링)

- §2 스킨 사이클링을 **주간 캘린더 상태기계**로: 각질→레티노이드→회복×N.
- 민감도·실측 장벽 상태로 회복일 수 가변(4일→6일).
- §1.2 램프업 사다리 통합: 승급/후퇴 조건 = "자극 없이 N회 견딤".
- 습관 스택 앵커 + 요일별 리마인더(§6).

### v3 — 폐루프 적응 (closed-loop)

- 실측 재스캔(예: 4주 간격, Haut.AI식 파라미터 추적) + 어드히어런스 데이터 입력.
- §4.3 안전 규칙: **최소 순응기간(8주) + 어드히어런스 확보** 전에는 교체 금지, 정체=실패 오판 방지.
- "장벽 회복 4주 → 미백 목표 단계 진행" 같은 **단계 전환 계획**을 데이터 게이트로 자동 제안(사용자 최종 결정).

### 가장 가치 큰 고도화 TOP 3

1. **주기 스케줄 엔진(v2)**: 정적 리스트 → "요일별 오늘 할 일". 경쟁사 대부분이 못 하는 실행 레이어. 어드히어런스·차별화 동시 확보.
2. **폐루프 조기교체 방지 안전규칙(v3 코어)**: "정체 시 교체"를 **순응기간+어드히어런스 게이트**로 감싸는 것 자체가 신뢰·안전의 해자. 경쟁 앱은 이 구분을 안 한다.
3. **캡슐 화장대(중복 경고 + 빈 슬롯만 구매)**: "덜 사게 말해준다"는 이룸 정체성의 제품화. 신뢰 → 리텐션 → (실수요) 어필리에이트 정합.
