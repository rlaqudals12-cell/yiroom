## 🔄 캡슐 루틴 개념

> 원본 출처: 패션의 "캡슐 워드로브" (적은 아이템으로 다양한 코디)

### 문제 정의

```
[문제]
사용자가 운동 루틴에서 겪는 근본 문제:
"계획대로 안 됨" → 기구 없음, 시간 없음, 의지 없음
```

### 제1원리 해결

```
Q: 왜 계획대로 안 되면 포기하나?
A: 대안을 모름 + 대안 찾기 귀찮음

Q: 그럼 진짜 해결책은?
A: "어떤 상황에서도 할 수 있는 유연한 시스템"
   → 대체 운동이 아니라 "상황 적응형 루틴"
   → 이게 캡슐 루틴의 본질!
```

---

## 🎯 캡슐 이론의 본질

```
"최소한의 구성으로 최대한의 활용"
"어떤 상황에서도 대안이 있는 유연한 시스템"
```

---

## 📊 전 영역 적용

| 영역 | 캡슐 이론 적용 |
|------|---------------|
| **패션** | 30-40개 아이템 → 100+ 코디 생성 |
| | 아이템 없으면 대체 조합 추천 |
| **운동** | 기구 없으면 맨몸 운동 대체 |
| | 헬스장 기구 사용 중 → 대체 기구 |
| **영양** | 재료 없으면 대체 재료 추천 |
| | 보유 재료로 만들 수 있는 레시피 |
| **스킨케어** | 제품 없으면 대체 제품 추천 |
| | 성분 기반 대안 제시 |

---

## ⚙️ 엔진 로직

```
[캡슐 루틴 엔진]

Input: 사용자 인벤토리 + 추천 요청

처리:
  1. 보유 아이템으로 가능한 조합 탐색
  2. 부족한 아이템 식별
  3. 대체 옵션 생성

Output: 1순위 추천 + 대체 옵션 N개
```

---

## 🔀 적용 예시

### 운동

```
[사용자 상황]
보유 기구: 덤벨(5kg), 요가매트
목표: 가슴 운동

[캡슐 루틴 적용]
1순위: 덤벨 플라이 (덤벨 활용)
2순위: 푸시업 (맨몸)
3순위: 딥스 (의자 활용)
```

### 영양

```
[사용자 상황]
보유 재료: 닭가슴살, 양파, 마늘
목표: 고단백 저녁

[캡슐 루틴 적용]
1순위: 닭가슴살 스테이크 (보유 재료)
대체: 브로콜리 없음 → 양배추로 대체
추가 구매 추천: 브로콜리, 아스파라거스
```
