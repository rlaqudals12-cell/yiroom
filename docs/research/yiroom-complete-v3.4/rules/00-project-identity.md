## 🧠 제1원리 사고법 (First Principles Thinking)

> ⭐ 일론 머스크의 핵심 사고방식 - 모든 개발에 적용

### 제1원리 vs 유추적 사고

```
[유추적 사고 - ❌ 피하기]
"경쟁사가 하니까 우리도 하자"
"요즘 다들 이거 쓰니까"
"예쁜 UI 참고해서 따라하자"

[제1원리 사고 - ✅ 적용하기]
"근본적인 이유가 뭐지?"
"우리 문제를 해결하는 가장 단순한 방법은?"
"사용자가 진짜 원하는 건 뭐지?"
```

### 이룸에 적용한 제1원리

```
Q1: 사람들이 뷰티앱을 쓰는 근본 이유는?
A1: "더 나은 나"가 되고 싶어서

Q2: "더 나은 나"가 되려면 뭐가 필요한가?
A2: 현재 나를 정확히 아는 것 + 개선 방향 아는 것

Q3: 왜 기존 앱들로는 안 되나?
A3: 파편화됨. 퍼스널컬러 따로, 피부 따로, 운동 따로.
    → 통합된 "나"에 대한 이해 불가능

[결론]
이룸의 본질 = "통합된 자기 이해 플랫폼"
```

### 개발 시 제1원리 질문법

| 상황 | ❌ 피하기 | ✅ 적용하기 |
|------|----------|------------|
| 기능 추가 전 | "경쟁사에 있으니까" | "이게 '통합된 자기 이해'에 기여하나?" |
| 기술 선택 전 | "요즘 다들 이거 쓰니까" | "우리 문제를 해결하는 가장 단순한 방법은?" |
| 버그 수정 전 | "일단 에러 안 나게 고치자" | "근본 원인은? 구조적 문제인가?" |
| UI 설계 전 | "예쁜 디자인 참고하자" | "사용자가 이 화면에서 진짜 원하는 건?" |

## 📌 핵심 원칙 요약

1. **통합 우선**: 개별 기능보다 연결성 중시
2. **근본 해결**: 표면적 해결보다 근본 원인 해결
3. **사용자 중심**: 기술보다 사용자 가치 우선
