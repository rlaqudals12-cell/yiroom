## 1. 궁극의 목적지 (P1)

### 한 문장

> **"나를 가장 잘 아는 존재, 그래서 세상과 나를 연결해주는 존재"**

### 3단계 진화

| Stage | 이름   | 설명                                                                 | 시점 |
| :---: | ------ | -------------------------------------------------------------------- | ---- |
| **1** | 분석기 | 사진 찍으면 분석해줌. 모듈별 각각 결과. "당신은 봄 웜톤이에요"       | 현재 |
| **2** | 조언자 | 분석 결과를 연결해서 "그래서 뭘 하라"고 알려줌. 8개 모듈 크로스 추천 | 다음 |
| **3** | 동반자 | 사용자의 변화를 함께 걸어감. 인과 분석 + 방향 제시                   | 궁극 |

### 궁극의 사용자 경험

```
사용자가 이룸을 쓰면:
1. 자기에게 뭐가 맞는지 안다 (분석)
2. 그래서 뭘 해야 하는지 안다 (행동)
3. 한 결과가 다른 영역에 어떤 영향을 주는지 안다 (연결)
4. 시간이 지나면서 뭐가 달라졌는지 안다 (변화)
5. 앞으로 뭘 하면 더 좋아지는지 안다 (방향)
```

### 정체성 플랫폼 (MBTI 모델)

```
MBTI: "나는 INTJ야" → 자기 설명 언어 → 대화 소재 → 소비 기준
이룸: "나는 봄 웜톤 · 복합성 피부 · S타입이야" → 동일한 효과 + "그래서 뭘 해" 연결

MBTI보다 강한 이유:
- 과학적 근거 (색채학, 피부생리학, 운동생리학)
- 행동 연결 ("이 색 사", "이 크림 써", "이 운동 해")
- 변화 추적 (고정이 아니라 진화)
```

### 궁극의 아침 시나리오

```
아침 7시, 알림 하나:

☀️ 오늘 자외선 지수 높아요
→ 피부: SPF50 선크림 먼저, 메이크업은 가벼운 톤으로
→ 패션: 봄 웜톤 + 자외선 차단 = 린넨 베이지 셔츠 추천
→ 영양: 비타민C 오늘 꼭 드세요 (자외선 손상 복구)

💪 어제 하체 운동했으니
→ 운동: 오늘은 상체 + 가벼운 스트레칭
→ 영양: 단백질 30g 더 드세요 (근육 회복기)
→ 피부: 땀 배출 후 약산성 세안 잊지 마세요

→ 피부 따로, 운동 따로, 영양 따로가 아님.
→ "하나의 맥락"에서 모든 추천이 나옴.
```

### 궁극의 쇼핑 시나리오

```
사용자가 올리브영에서 파운데이션 앞에 섰을 때:

이룸: "이 제품은 당신과 87% 매칭이에요"
- 퍼스널컬러(봄 웜톤) ✓ 맞는 색상
- 피부(복합성) ✓ 유분 조절 성분
- 알레르기 ✓ 기피 성분 없음
- 가격 ✓ 예산 범위 내

vs "이 제품은 42% 매칭이에요"
- 퍼스널컬러 ✗ 쿨톤용 (당신은 웜톤)
- 알레르기 ⚠️ 니켈 함유 (민감 성분)

→ 사용자는 더 이상 "이게 나한테 맞나?" 고민 안 함.
```

### 궁극의 3개월 리포트

```
"3개월 리포트"

피부: 바이탈리티 62점 → 78점 (+16)
  원인 분석: 오메가-3 꾸준히 복용(영양) + 주 2회 유산소(운동) + 세라마이드 크림(제품)

체형: 삼각형 → 허리 비율 개선 중
  원인 분석: 하체 운동 주 3회 + 단백질 섭취 증가

→ "어떤 행동이 어떤 결과를 만들었는지" 인과관계가 보임.
→ 사용자는 자기 몸의 매뉴얼을 얻는 것.
```

### 진짜 경쟁자

```
이룸의 경쟁자는 전문가도, Perfect Corp도 아님.
"아무것도 안 하는 것"이 진짜 경쟁자.

전문가 이용 경험: 한국 인구의 2-10%
나머지 90%: "관심은 있는데 돈/시간/심리적 장벽 때문에 안 함"

전문가: 100점, 비싸고 접근성 낮음
이룸:   90점, 무료이고 즉시 사용 가능
아무것도 안 함: 0점

90%의 사람에게: 90점 vs 0점 → 이룸이 압승
```

## 2. 수익 모델

### 핵심 원칙

```
사용자는 영원히 무료.
돈은 "사용자에게 접근하고 싶은 기업"이 냄.
구독은 최대한 피함 — 파이(사용자 수)를 키우는 것이 최우선.
```

### ADR-054 v2.0과의 관계

```
ADR-054 v2.0 (현재 문서):
- Phase 1A: 어필리에이트 ARPU ₩2-4
- Phase 1B: +1회 구매 (AI 리포트 PDF ₩2,900, VTO 언락 ₩1,900)
- Phase 2: +구독 (MRR ₩100만 시점)

이번 대화에서 재검토한 방향:
- 1회 구매: "기능 제한 해제" → "가치 기반 큐레이션"으로 성격 변경
- 구독: 최대한 회피 → B2B 구독만 (사용자 구독 아님)
- 핵심 전환: ARPU ₩2-4 → ₩20-50 (매칭 정확도 올려서)

→ ADR-054 v3.0으로 업데이트 필요 (확정 후)
→ 현재는 방향성만 정리, 코드 변경 없음
```

### 왜 구독이 아닌가

```
구독 = "돈 내야 써"
→ 무료/유료 분리 → 무료는 반쪽 경험 → 공유 안 함 → 바이럴 죽음
→ CAC 폭등 → 자본 경쟁 → 경쟁사(L'Oreal, Perfect Corp)에게 밀림

파이를 키워야 경쟁력:
- 사용자 많을수록 → 매칭 데이터 축적 → 정확도 상승 → 전환율 상승
- 데이터 모트 형성 → 경쟁사 진입 장벽
```

### 수익 3단계

| Stage | MAU      | 수익원                   | 사용자 비용 | ARPU       |
| :---: | -------- | ------------------------ | :---------: | ---------- |
| **1** | 0~50K    | 어필리에이트 (정밀 매칭) |    무료     | ₩20-50     |
| **2** | 50K~500K | + 브랜드 검증 매칭       |    무료     | ₩100-300   |
| **3** | 500K+    | + B2B API (뷰티 인프라)  |    무료     | ₩500-1,000 |

### 플라이휠

```
무료 분석 → 정체성 획득("봄 웜톤") → 자발적 공유
→ 친구 가입(CAC=₩0) → 사용자 수↑ → 매칭 데이터↑
→ 매칭 정확도↑ → 전환율↑ → 어필리에이트 수익↑
→ (수익이 늘어도 사용자에겐 무료)
```

### 1회 구매 (선택적, 기능 제한 아님)

```
분석/추천 = 전부 무료 (제한 없음)
유료 = "가치 기반 큐레이션"
- "봄 웜톤 · 복합성 피부 맞춤 세트 5종" = ₩49,000
- 이룸이 선별한 제품 번들 (검색 시간 절약 + 확실한 매칭)
- "돈 안 내면 불완전" ❌ → "돈 내면 더 편함" ✅
```

### B2B Stage 3 시나리오 — "뷰티 여권"

```
"이룸 프로필" = 뷰티 여권 (신용점수처럼 따라다님)

- 올리브영 API 연동 → "이룸 프로필로 필터링" → API 사용료
- 아모레 신제품 → "이룸 검증 매칭" 뱃지 → 노출비
- 이미지 컨설턴트 → 이룸 Pro(B2B 구독) → 고객 사전 분석 데이터
- 피부과/미용실 → 이룸 히스토리 공유 → 예약 수수료
- 뷰티 이커머스 → "이룸 프로필로 필터링" 기능 제공
- 오프라인 매장 → QR 스캔 → "당신에게 맞는 제품은 3번 선반"
```

### 매칭 정확도 = 수익의 핵심

```
현재: "세라마이드 좋아요" → 사용자가 직접 검색 → 이룸 우회 구매 → ARPU ₩2-4
목표: "이 제품이 당신과 87% 매칭" → 이룸 통해 구매가 합리적 → ARPU ₩20-50+

매칭 정확도 올리는 3가지:
1. 프롬프트 고도화 (원리/논문 주입) → 분석 품질↑
2. 크로스 모듈 통합 → PC+피부+체형+알레르기 조합 매칭
3. 집단 지성 → "나와 같은 프로필 12,847명 중 89%가 효과 봄"
```

## 3. 전문가 대비 포지셔닝

### 커버 목표: 90%

```
❌ "전문가를 대체합니다" → 거짓말
✅ "전문가 90% 수준을, 무료로, 즉시, 24/7 제공"
✅ 나머지 10%(물리적 접촉 필요)는 전문가 연결

90% 달성 핵심: 원리/논문 기반 프롬프트 고도화
```

### 전문가별 커버 가능 범위

| 전문가            | 커버 가능                          | 대체 불가 (10%)             |
| ----------------- | ---------------------------------- | --------------------------- |
| 이미지 컨설턴트   | 시즌 분류, 팔레트, 코디 색상       | 극미세 뉘앙스 (경계 케이스) |
| 피부 전문가       | 타입 진단, 성분 추천, 루틴, 세안법 | 촉진, 피부 두께 판단        |
| 체형 전문가       | 체형 분류, 패션, 운동 매칭         | 근육량/체지방 정밀 측정     |
| 운동 코치         | 플랜 생성, 체형별 처방, 부상 회피  | 실시간 자세 미세 교정       |
| 영양사            | 칼로리/매크로, 영양소, 시너지      | 혈액검사 기반 의료 식단     |
| 헤어 전문가       | 타입, 컬러 추천, 케어 루틴         | 모발 촉감, 시술 기법        |
| 메이크업 아티스트 | 컬러 추천, 스타일, VTO             | 블렌딩 기법, 실시간 수정    |

### 전문가가 "못 하는 것"을 이룸이 하는 것

```
1. 24/7 연속 추적 — 매일 옆에 있는 존재만 가능
2. 8개 영역 크로스 인과 분석 — 전문가들이 한 자리에 모이는 일은 없음
3. N=100만 집단 지성 — "나와 같은 프로필에게 뭐가 효과 있었는지"
4. 상황 적응형 추천 — "오늘, 지금, 이 날씨에서" 실시간 조언
```

## 4. 프롬프트 고도화 — 90% 달성 핵심

### 원리

```
전문가의 뇌에 있는 것 = 원리 + 논문 + 경험
이걸 프롬프트에 주입 = AI가 전문가의 판단 프레임워크로 분석

Level 1 (현재): "분석해주세요" → 70%
Level 2: "이 원리에 따라 분석해주세요" → 85%
Level 3: "원리 + 논문 + 검증 규칙" → 90%
Level 4: "위 + 사용자 과거 데이터 + 유사 프로필 통계" → 90%+
```

### 모듈별 주입할 핵심 원리/논문

| 모듈       | 핵심 원리                             | 주요 논문/교과서                     |
| ---------- | ------------------------------------- | ------------------------------------ |
| **PC-1**   | 먼셀 색체계, Lab 색공간, 12톤 시스템  | Caygill(1942), Halal et al.(2020)    |
| **S-1**    | Baumann 16타입, TEWL, 피부 장벽 1:1:1 | Draelos(2018), Kligman(1986)         |
| **C-1**    | ACSM 체형 분류, Kibbe 시스템          | ACSM Guidelines                      |
| **W-1**    | 과부하 원칙, 근비대 기전, Janda 교정  | Schoenfeld(2010), Selye(1936)        |
| **N-1**    | Mifflin-St Jeor, 영양소 시너지/길항   | ISSN Position Stands, Kerksick(2017) |
| **H-1**    | Trichology 기초, 모발-영양 상관       | Almohanna et al.(2019)               |
| **M-1**    | 화장품 색채 이론, 얼굴형별 기법       | Color theory for cosmetics           |
| **OH-1**   | VITA 셰이드, CIEDE2000                | ISO 11664-6                          |
| **크로스** | 피부-영양 상관, 운동-영양 타이밍      | Schagen(2012), ISSN                  |

### 서버 측 검증 규칙 확장

```
현재: PC-1만 혈관색↔톤 교차 검증 (blue→쿨, green→웜)
필요: 전 모듈에 "원리 위반 자동 교정" 확장

예시:
- S-1: Fitzpatrick I-II인데 "oily skin" 판정 → 재검증 (밝은 피부는 건성 경향)
- C-1: 어깨/엉덩이 비율이 역삼각형인데 "pear" 판정 → 자동 교정
- N-1: 단백질 150g 추천인데 체중 50kg → 체중 대비 과다 검증 (3g/kg 초과)
- W-1: 무릎 부상 입력인데 풀스쿼트 포함 → 자동 제외 + 대체 운동
```

### 실행 파이프라인

```
논문/교과서 → docs/principles/ (원리 문서화)
→ lib/gemini.ts (프롬프트에 원리 주입)
→ Gemini Vision (원리 기반 분석)
→ 서버 측 검증 (원리 위반 시 자동 교정) ← 전 모듈 확장 필요
→ 사용자에게 결과 + 근거 표시
```

## 5. 8개 모듈 현황 및 개선점

### 모듈 간 연동 원칙

```
비전: "모든 모듈이 연동되어 최고의 결과를 이끌어내야 하며 독립적으로 연동되어선 안 돼"
현재: 구조는 있으나(BeautyProfile, CapsuleEngine, CCS) 연동의 "깊이"가 모듈마다 다름
핵심 갭: 3개 이상 모듈이 합쳐진 통합 추천의 깊이가 얕음
```

### PC-1 퍼스널컬러

```
현재: Gemini Vision → 혈관색+언더톤+대비도 → 시즌 분류 (정확도 70-80%)
연동: M-1, H-1에 시즌 전달 ✅
구조: PC-1 = 진단 (색채학), M-1 = 적용 (실용) → 분리가 맞음

개선:
- 색 보정 카드(Color Checker) 도입 → CIE-3 구현 → 정확도 85-90%
- 영상 기반 분석 (10초 영상 → 다각도 자동 캡처)
- 다회 분석 학습 (3번 분석 → 교차 검증 → 95%+)
- PC-1 → M-1 자동 연결 CTA 강화 ("이 컬러로 메이크업 추천받기")
- PC-1 → 패션 통합 추천 강화 ("이 시즌에 맞는 코디 3세트")
```

### S-1 피부 분석

```
현재: CIE 파이프라인 + Gemini 12존 분석. 성분 기반 루틴 추천.
연동: N-1(영양→피부), M-1(피부→커버력), H-1(두피↔피부) ✅

개선:
- "성분 추천" → "구체 제품 추천" 전환 (어필리에이트 연결)

- 관리법 개인화: 생리 주기 연동, 환경 데이터(미세먼지/습도) 연동
- 피부 시술 추천 (ADR-045): 설계만 존재, 구현 필요. 면책 고지 필수.
  → 출시 후 Phase 1B, 법적 검토 선행
- Baumann 16타입 도입 (단순 건성/지성/복합 → 16가지 세분화)
```

## 8. 경쟁 방어벽

### 5대 방어벽

```
1. 데이터 모트: 사용자 프로필 축적 → 매칭 정확도 상승 → 복제 불가
2. 정체성 락인: "나는 봄 웜톤" = 이룸에서 받은 정체성 → 전환 비용 높음
3. 크로스 모듈: 8개 모듈 통합 = 복제 비용 극도로 높음 (2-3년)
4. 시간 데이터: "3개월간의 피부 변화" = 새 앱으로 대체 불가
5. 집단 지성: N=100만 사용자 통계 = 어떤 전문가도 못 가짐
```

### 경쟁사 구체 비교

```
Perfect Corp (ModiFace):
  강점: VTO 기술 선도, L'Oreal 자본
  약점: "통합 프로필" 없음. 퍼스널컬러/피부/메이크업 파편화.
  이룸 차별: 8개 모듈 크로스 연동 (2-3년 복제 비용)

Samsung Health:
  강점: 운동/건강 + 하드웨어(Galaxy Watch)
  약점: 뷰티 도메인 0 (퍼스널컬러, 피부, 메이크업 전혀 없음)
  이룸 차별: 뷰티+웰니스 통합 (영역 자체가 다름)

화해/글로우픽:
  강점: 제품 리뷰 커뮤니티, 성분 DB
  약점: "나에게 맞는지" 분석 없음 ("평점 4.5" ≠ "내 피부에 맞음")
  이룸 차별: 프로필 기반 개인화 매칭
```

## 11. 모듈별 궁극의 형태 (100점 정의)

```

## 16. 데이터 전략

### 축적

```
분석 데이터: 8모듈 결과 + 이미지 메타 + 신뢰도 + 시간축 변화
행동 데이터: 제품 클릭/구매 + 효과 피드백 + 캡슐 열람 + 운동/식단/수면
환경 데이터: 날씨/미세먼지/UV(API) + 계절 + 생리 주기(입력)
```

### 활용 (3단계)

```
1단계 (개인): 개인 매칭 정확도 향상
2단계 (집단, MAU 10K+): 유사 프로필 익명 통계
3단계 (인사이트, MAU 100K+): B2B 프로필 그룹 단위 통계 판매
```

### 보호 (신뢰가 생명)

```
1. 투명성: "당신의 데이터는 이렇게 사용됩니다" 명시
2. 제어권: 사용자 언제든 전체 삭제 가능 (GDPR 수준)
3. 익명화: B2B는 그룹 통계만 (개인 식별 불가)
4. 판매 금지: 원본 데이터 제3자 판매 영구 금지
5. 동의: 각 수집 단계마다 명시적 동의 (ADR-023)
6. 암호화: 민감 정보 암호화 저장 (ADR-070)

원칙: 데이터 보호 ↔ 수익은 충돌하지 않음.
"데이터 판매"가 아니라 "매칭 정확도"로 수익. 신뢰 = 장기 수익.
```

## 17. AI 모델 전략

### 단기 (Phase 1A)

```
- 프롬프트 고도화 Level 2→3 (원리/논문 주입)
- 서버 측 검증 전 모듈 확장
- 모델 추상화 레이어 (lib/ai/provider.ts) → 교체 용이성
- 목표: 70% → 90%
```

### 중기 (Phase 1B~2, MAU 10K+)

```
- 이룸 데이터 파인튜닝 검토 (분석 결과 + 전문가 검증 샘플)
- 프롬프트 Level 4 (과거 데이터 + 유사 프로필 통계 주입)
- 예측 모델 검토 (시계열 → "3개월 후 피부 점수")
```

### 장기 (Phase 3)

```
- 모델 독립성 확보 (Gemini/GPT/Claude 교체 가능)
- 핵심 가치 = 프롬프트(원리) + 검증 규칙 + 데이터 (모델이 아님)
- 자체 도메인 모델 ROI 평가 (파인튜닝 비용 vs 프롬프트 효과)
```

## 18. 사용자 획득 채널 전략

### 첫 1,000명 (Cold Start)

```
바이럴은 사용자가 있어야 시작됨. 첫 1,000명은 어디서?

1. ASO (앱스토어 검색 최적화)
   키워드: "퍼스널컬러 진단", "피부 분석 무료", "체형 분석 AI"
   → 검색 의도가 명확한 사용자 = 전환율 높음
   → 스크린샷, 설명 최적화 필수

2. SEO (웹 검색)
   "퍼스널컬러 테스트 무료" → 이룸 웹 랜딩 → 앱 유도
   "내 피부 타입 확인" → 이룸 웹 랜딩
   → 블로그/콘텐츠 마케팅 연동

3. 커뮤니티 시딩
   에브리타임 (대학생), 화해 (뷰티), 디시 뷰티갤, 여성시대
   "퍼스널컬러 AI 진단 해봤는데 꽤 정확함" → 자연스러운 노출
   → 광고가 아닌 실제 사용 후기 톤

4. 숏폼 콘텐츠
   "내 퍼스널컬러 AI가 맞춰봄" 틱톡/릴스 (30초)
   → 결과가 재미있으면 자발적 제작
   → 나노 인플루언서 10명 시딩 (ADR-056)

5. 지인 네트워크
   → 솔로 파운더의 직접 네트워크 100-200명
   → "베타 테스터 모집" → 피드백 + 초기 데이터 + 입소문
```

### Phase별 채널 확장

```
Phase 1A (0→3K): ASO + SEO + 커뮤니티 시딩 + 나노 인플루언서 10명
Phase 1B (3K→14K): + 마이크로 인플루언서 50명 + 숏폼 콘텐츠 본격화
Phase 2 (14K→50K): + 유료 광고 시작 (Instagram/TikTok) + 브랜드 콜라보
Phase 3 (50K+): + 퍼포먼스 마케팅 + TV/매거진 PR
```

## 19. 파괴적 리스크 시나리오

### 시나리오 A: Apple/Google이 OS에 피부 분석 내장

```
위협: 10억+ 디바이스에 무료 탑재 → 이룸의 "무료 분석" 차별점 소멸
확률: 중간 (Apple Health에 피부 기능 루머 존재)

생존 조건:
- Apple은 "단일 분석"만 제공 (피부 OR 운동 OR 영양)
- 이룸은 "8개 모듈 통합 + 크로스 인과 분석" = Apple이 안 하는 영역
- 데이터 모트: 이미 축적된 사용자 프로필 + 시간축 데이터

대응:
- Apple Health 연동 → "Apple 분석 + 이룸 통합" 포지셔닝
- 분석 자체보다 "연결 + 추적 + 매칭" 가치 강화
```

### 시나리오 B: AI 회사가 직접 뷰티 앱 출시

```
위협: Gemini/GPT가 중간 레이어(이룸)를 건너뛰고 직접 서비스
확률: 낮음 (AI 회사는 수평 플랫폼 선호, 수직 앱은 파트너에게)

생존 조건:
- 이룸의 가치는 "모델"이 아니라 "도메인 지식 + 데이터 + UX"
- AI 모델은 교체 가능 (추상화 레이어, 섹션 17)
- 도메인 프롬프트 + 검증 규칙 + 사용자 데이터 = 이룸 고유

대응:
- 모델 독립성 확보 (Gemini→GPT→Claude 교체 가능)
- 핵심 가치를 모델이 아닌 "원리+데이터+UX"에 집중
```

### 시나리오 C: 올리브영/아모레가 자체 분석 기능 출시

```
위협: 이미 유통 채널 + 고객 데이터를 가진 플레이어
확률: 높음 (올리브영 앱에 스킨 분석 이미 일부 존재)

생존 조건:
- 올리브영은 "자사 제품만" 추천 (중립성 없음)
- 이룸은 "모든 브랜드" 중립 추천 (사용자 신뢰)
- 올리브영은 "피부만", 이룸은 "8개 모듈 통합"

대응:
- 중립성을 핵심 가치로 강화 ("이룸은 특정 브랜드 편들지 않음")
- 오히려 올리브영 API 연동 → "올리브영 제품 중 당신 매칭 TOP 5"
- 유통사는 경쟁자가 아니라 파트너로 전환
```

### 시나리오 D: 규제 리스크 (의료기기 판정)

```
위협: 피부 분석이 "의료기기"로 분류되면 허가 필요 (식약처)
확률: 낮음 (현재 "참고 정보" 수준)

생존 조건:
- "진단"이 아닌 "참고 정보" 포지셔닝 유지
- 면책 고지 철저 (ADR-024, 이미 구현)
- "의료 행위 3종 영구 제외" (ADR-046)

대응:
- "AI가 생성한 참고 정보"라는 표현 일관 유지
- 의료 용어 사용 금지 ("진단"→"분석", "처방"→"추천")
- 법률 자문 정기 검토 (Phase 1B부터)
```

Phase 1B — 혼자 가능하지만 속도 제한:
  ⚠️ 색 보정 카드 + 영상 분석 (UX 설계 + 구현 = 2-3주)
  ⚠️ 제품→결과 추적 루프 (DB + UI + 알림 = 2-3주)
  ⚠️ 3모듈 통합 추천 깊이 (설계 + 프롬프트 + 테스트)
  ⚠️ 마이크로 인플루언서 50명 관리 → 어려움 시작

Phase 2 — 팀 필요:
  ❌ 집단 지성 (데이터 엔지니어링 + ML = 전문 인력)
  ❌ 브랜드 파트너십 영업 (비즈니스 전담)
  ❌ 프로필 기반 소셜 커뮤니티 (모더레이션 + 운영)
  ❌ 유료 광고 운영 (퍼포먼스 마케팅 전문)

Phase 3 — 팀 필수:
  ❌ B2B API (서버 인프라 + 영업 + 기술 지원)
  ❌ 전문가 플랫폼 (양면 마켓플레이스 운영)
```
