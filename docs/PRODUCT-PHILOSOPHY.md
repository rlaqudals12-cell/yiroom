## 1. 궁극의 목적지

### 한 문장

> **"나를 가장 잘 아는 존재, 그래서 세상과 나를 연결해주는 존재"**

### 3단계 진화

| Stage | 이름   | 설명                 | 핵심 기능                              |
| :---: | ------ | -------------------- | -------------------------------------- |
| **1** | 분석기 | 사진 찍으면 분석해줌 | 8개 모듈 독립 분석, 결과 표시          |
| **2** | 조언자 | "그래서 뭘 하라"까지 | 크로스 모듈 추천, 제품 매칭, 루틴 제안 |
| **3** | 동반자 | 변화를 함께 걸어감   | 인과 분석, 예측, 선제 조언, 습관 형성  |

### 사용자가 이룸을 쓰면

```
1. 자기에게 뭐가 맞는지 안다 (분석)
2. 그래서 뭘 해야 하는지 안다 (행동)
3. 한 결과가 다른 영역에 어떤 영향을 주는지 안다 (연결)
4. 시간이 지나면서 뭐가 달라졌는지 안다 (변화)
5. 앞으로 뭘 하면 더 좋아지는지 안다 (방향)
```

## 2. 정체성 플랫폼 (MBTI 모델)

```
MBTI: "나는 INTJ야" → 자기 설명 언어 → 대화 소재 → 소비 기준
이룸: "나는 봄 웜톤 · 복합성 피부 · S타입이야" → 동일 효과 + "그래서 뭘 해" 연결
```

### MBTI보다 강한 이유

- **과학적 근거**: 색채학, 피부생리학, 운동생리학 (논문 7편+ 기반)
- **행동 연결**: "이 색 사", "이 크림 써", "이 운동 해" (분석→행동 직결)
- **변화 추적**: 고정이 아니라 진화 (3개월 리포트로 인과 분석)

## 3. 수익 원칙

### 핵심

```
사용자는 영원히 무료.
돈은 "사용자에게 접근하고 싶은 기업"이 냄.
구독은 최대한 피함 — 파이(사용자 수)를 키우는 것이 최우선.
```

### 왜 구독이 아닌가

```
구독 = "돈 내야 써"
→ 무료/유료 분리 → 무료는 반쪽 경험 → 공유 안 함 → 바이럴 죽음
→ CAC 폭등 → 자본 경쟁 → 경쟁사(L'Oreal, Perfect Corp)에게 밀림

파이를 키워야 경쟁력:
- 사용자 많을수록 → 매칭 데이터 축적 → 정확도 상승 → 전환율 상승
- 데이터 모트 형성 → 경쟁사 진입 장벽
```

### 수익 3단계

| Stage | MAU      | 수익원                   | 사용자 비용 |
| :---: | -------- | ------------------------ | :---------: |
| **1** | 0~50K    | 어필리에이트 (정밀 매칭) |    무료     |
| **2** | 50K~500K | + 브랜드 검증 매칭       |    무료     |
| **3** | 500K+    | + B2B API (뷰티 인프라)  |    무료     |

### 매칭 정확도 = 수익

```
매칭이 정확할수록 → 사용자가 이룸 통해 구매 → 어필리에이트 수익
매칭이 부정확하면 → 사용자가 직접 검색 → 이룸 우회 → 수익 0

따라서: 프롬프트 고도화 + 크로스 모듈 연동 = 수익의 기반
```

## 4. 경쟁 포지셔닝

### 진짜 경쟁자

```
이룸의 경쟁자는 전문가도, Perfect Corp도 아님.
"아무것도 안 하는 것"이 진짜 경쟁자.

전문가 이용 경험: 한국 인구의 2-10%
나머지 90%: "관심은 있는데 돈/시간/심리적 장벽 때문에 안 함"

전문가: 100점, 비싸고 접근성 낮음
이룸:   90점, 무료이고 즉시 사용 가능
아무것도 안 함: 0점

90%의 사람에게: 90점 vs 0점 → 이룸이 압승
```

### 전문가와의 관계

```
❌ "전문가를 대체합니다" → 거짓말
✅ "전문가 90% 수준을, 무료로, 즉시, 24/7 제공"
✅ 나머지 10%(물리적 접촉 필요)는 전문가 연결

이룸은 전문가의 적이 아니라, 전문가에게 고객을 보내는 파이프라인
```

### 전문가가 못 하는 것을 이룸이 하는 것

```
1. 24/7 연속 추적 — 매일 옆에 있는 존재만 가능
2. 8개 영역 크로스 인과 분석 — 전문가들이 한 자리에 모이는 일은 없음
3. N=100만 집단 지성 — "나와 같은 프로필에게 뭐가 효과 있었는지"
4. 상황 적응형 추천 — "오늘, 지금, 이 날씨에서" 실시간 조언
```

### 경쟁사 비교

| 경쟁사         | 강점                 | 이룸 차별점                                             |
| -------------- | -------------------- | ------------------------------------------------------- |
| Perfect Corp   | VTO 기술 선도        | 8개 모듈 크로스 연동 (2-3년 복제 비용)                  |
| Samsung Health | 운동/건강 + 하드웨어 | 뷰티 도메인 통합 (영역 자체가 다름)                     |
| 화해/글로우픽  | 제품 리뷰 커뮤니티   | 프로필 기반 개인화 매칭 ("평점 4.5" ≠ "내 피부에 맞음") |

## 5. 5대 방어벽

```
1. 데이터 모트: 사용자 프로필 축적 → 매칭 정확도 상승 → 복제 불가
2. 정체성 락인: "나는 봄 웜톤" = 이룸에서 받은 정체성 → 전환 비용 높음
3. 크로스 모듈: 8개 모듈 통합 = 복제 비용 극도로 높음 (2-3년)
4. 시간 데이터: "3개월간의 피부 변화" = 새 앱으로 대체 불가
5. 집단 지성: N=100만 사용자 통계 = 어떤 전문가도 못 가짐
```

## 6. 의사결정 프레임워크

이 철학에 기반한 의사결정:

| 상황         | 질문                                   | 기준                         |
| ------------ | -------------------------------------- | ---------------------------- |
| 새 기능 추가 | "사용자를 더 잘 아는 데 기여하는가?"   | Stage 1→2→3 진화에 기여해야  |
| 수익화 검토  | "사용자 경험을 훼손하는가?"            | 무료 원칙 위반이면 기각      |
| 경쟁 대응    | "우리만의 방어벽을 강화하는가?"        | 5대 방어벽 중 하나 이상      |
| 기술 선택    | "프롬프트/데이터/UX 중 뭘 강화하는가?" | 모델이 아닌 핵심 자산에 투자 |
| 우선순위     | "파이를 키우는 데 직접 기여하는가?"    | 바이럴/리텐션 기여도 기준    |

## 7. 플라이휠

```
무료 분석 → 정체성 획득("봄 웜톤") → 자발적 공유
→ 친구 가입(CAC=₩0) → 사용자 수↑ → 매칭 데이터↑
→ 매칭 정확도↑ → 전환율↑ → 어필리에이트 수익↑
→ (수익이 늘어도 사용자에겐 무료) → 다시 무료 분석
```

이 플라이휠의 어떤 단계든 약해지면 전체가 멈춤.
**모든 의사결정은 이 플라이휠을 강화하는 방향**이어야 함.
