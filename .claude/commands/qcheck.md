# /qcheck 명령어

변경사항에 대한 빠른 품질 검사를 수행합니다.

## 대상

$ARGUMENTS (미지정 시 최근 변경사항 전체)

## 실행 내용

지정된 대상에 대해 SKEPTICAL senior engineer 관점으로:

### 1. 함수 체크리스트

변경/추가된 각 함수에 대해:

- [ ] 단일 책임 원칙 (하나의 기능만)
- [ ] 명확한 함수명
- [ ] 적절한 에러 처리
- [ ] 한국어 주석 존재
- [ ] 타입 정의 완전성

### 2. 테스트 체크리스트

변경/추가된 각 테스트에 대해:

- [ ] 정상 케이스 테스트
- [ ] 엣지 케이스 테스트
- [ ] 에러 케이스 테스트
- [ ] 테스트 설명 명확

### 3. UI 체크리스트 (해당 시)

- [ ] 신조어/슬랭 없음
- [ ] 자연스러운 한국어
- [ ] 접근성 고려
- [ ] 로딩/에러 상태 처리

## 출력 형식

```markdown
## 🔍 품질 검사 결과

### 함수 검사 (X개)

| 함수명 | SRP | 이름 | 에러처리 | 주석 | 타입 |
| ------ | --- | ---- | -------- | ---- | ---- |
| func1  | ✅  | ✅   | ⚠️       | ✅   | ✅   |

### 테스트 검사 (X개)

| 테스트 | 정상 | 엣지 | 에러 | 설명 |
| ------ | ---- | ---- | ---- | ---- |
| test1  | ✅   | ⚠️   | ❌   | ✅   |

### 개선 필요 사항

1. [구체적 개선 사항]

### 종합 점수: X/10
```

## 사용 예시

```
/qcheck
/qcheck --functions-only
/qcheck --verbose
```
