# /ux-check 명령어

변경된 UI 컴포넌트/페이지에 대해 UX 체크리스트를 자동 점검합니다.

## 대상

$ARGUMENTS (미지정 시 최근 git 변경 파일 중 UI 관련 파일)

## 실행 절차

### 1. 대상 파일 식별 + 화면 유형 추론

- 인자가 있으면: 해당 파일/화면 관련 컴포넌트 탐색
- 인자가 없으면: `git diff --name-only HEAD` 에서 `.tsx` 파일 추출
- 페이지(`app/`)와 컴포넌트(`components/`) 모두 포함
- UI 관련 파일이 없으면: "점검 대상 UI 파일이 없습니다" 출력 후 종료

파일 경로에서 **화면 유형**을 추론하여 체크리스트의 "화면 유형별 프로필" 적용:

- `home`, `dashboard`, `(tabs)/index` → 홈/대시보드 프로필
- `analysis/*/result` → 분석 결과 프로필
- `analysis/*` (result 제외) → 분석 입력 프로필
- `settings`, `profile` → 설정 프로필
- `capsule` → Daily Capsule 프로필
- 매칭 불가 시 → 전체 항목 점검 (프로필 미적용)

### 2. 체크리스트 로드 및 판정

`docs/checklists/ux-pr-checklist.md` 파일을 읽는다.
각 항목의 **"판정 기준"**과 **"탐지 방법"** 컬럼을 기준으로 대상 코드를 분석한다.

> 중요: 탐지 로직은 체크리스트 파일이 Single Source of Truth.
> 이 명령어 파일에 탐지 패턴을 중복 정의하지 않는다.

판정 결과:

- **pass**: 코드에서 준수가 확인됨
- **fail**: 위반 발견 (구체적 위치 명시)
- **warn**: 자동 판정 불확실, 수동 확인 필요 (체크리스트에서 "warn 고정" 표시된 항목)
- **n/a**: 해당 없음 (예: 비동기 로드 없는 컴포넌트의 C1 로딩 상태)

### 3. 모드

#### 기본 모드 (전체)

A~G 전 카테고리 43항목 점검 (화면 유형 프로필 적용 시 n/a 자동 스킵).

#### --quick 모드

Critical/High 심각도만 빠르게 점검:

- **A1, A3** (영어 raw value, err.message) — Critical
- **D1~D4** (감정 설계 금지 패턴) — Critical
- **A2, B1, B2** (해요체, data-testid, 터치 44px) — High
- **B6** (Safe Area — 모바일 대상 시) — High
- **C1~C3** (3종 상태) — High
- **D6** (프라이버시 안내 — 카메라/업로드 대상 시) — High
- **D9** (자율성 존중 — 지시형 표현) — High
- **D10** (비의료 고지 — 피부/구강/영양 결과 시) — High

= 14~16항목 (모바일/카메라/건강분석 포함 시 +2), 1분 내 완료.

### 4. 사용자 관점 시뮬레이션 (기본 모드에서 자동 실행)

코드 분석 후, 대상 화면에 대해 시뮬레이션을 수행한다.

#### 4a. 1초 테스트 (Glance Test — Jobs T3)

> "1초 만에 이 화면의 목적이 보이는가?"

화면의 JSX 구조에서 시각적 위계를 분석:

- **1초 후 기억에 남는 것**(가장 큰 텍스트/아이콘/색상 요소)이 핵심 가치와 일치하는가?
- 시선 흐름이 **핵심 → CTA** 순서로 자연스러운가?
- 장식 요소가 핵심을 가리지 않는가?

#### 4b. 페르소나 시뮬레이션

화면 유형에 따라 **가장 중요한 페르소나**를 자동 선택하여 시뮬레이션:

| 화면 유형   | 1순위 페르소나 | 2순위     | 핵심 질문                                             |
| ----------- | -------------- | --------- | ----------------------------------------------------- |
| 홈/대시보드 | **P3 일상화**  | P1 탐색자 | "오늘 새로운 게 있나?" / "뭘 먼저 해야 해?"           |
| 분석 결과   | **P2 학습자**  | P4 복귀자 | "이게 나한테 무슨 의미지?" / "예전 결과랑 뭐가 달라?" |
| 분석 입력   | **P1 탐색자**  | —         | "어떻게 시작하지? 어려울까?"                          |
| 캡슐        | **P3 일상화**  | P4 복귀자 | "오늘 나한테 맞는 조언이 있나?"                       |
| 설정        | **P2 학습자**  | —         | "원하는 설정을 바로 찾을 수 있나?"                    |

각 페르소나 관점으로:

- **3초 진입**: "이 화면의 핵심 가치를 이해할 수 있는가?"
- **CTA 명확성**: "다음 행동이 명확한가?"
- **Delight 존재**: "이 경험에 '와!' 하는 순간이 있는가?" (Jobs T6)
- 문제 발견 시 점검 결과 하단에 "사용자 시뮬레이션 소견" 추가

### 5. 궁극의 형태 체크 (해당 시)

대상이 주요 화면(홈, 분석 결과, 대시보드)인 경우:

- "이 화면이 궁극의 형태 대비 몇 %인가?" 소견 추가
- 부족한 부분이 의도적 제외인지 누락인지 판단

## 출력 형식

```markdown
## UX 점검 결과

**대상**: [파일 목록]
**화면 유형**: [홈/분석 결과/분석 입력/설정/캡슐/기타]
**모드**: 전체 / Quick
**판정**: X/Y 통과 (Z개 실패, W개 경고, N개 n/a)
**이전 대비**: [+N 개선 / -N 악화 / 이력 없음] (--save 시)

### 점검 결과

| #   | 카테고리 | 항목                  | 판정      | 위치/비고   |
| --- | -------- | --------------------- | --------- | ----------- |
| A1  | 한국어   | 영어 raw value 미노출 | pass/fail | [파일:라인] |
| A2  | 한국어   | 해요체 통일           | pass/fail |             |
| ... |          |                       |           |             |

### 실패 항목 상세

#### [항목 ID]. [항목명] (심각도)

- **위치**: `파일:라인`
- **현재**: [문제 코드/텍스트]
- **수정 방향**: [구체적 개선안]

### 경고 항목 (수동 확인 필요)

- [E2] L2 가치 제공: [확인 포인트]
- ...

### 사용자 시뮬레이션 소견 (기본 모드)

- 1초 테스트 (Glance): [핵심이 즉시 보이는가?]
- [1순위 페르소나] 3초 진입: [소견]
- [2순위 페르소나] 3초 진입: [소견] (해당 시)
- CTA 명확성: [소견]
- Delight 순간: [있음/없음 — 구체적 설명]

### 종합

- 통과율: X%
- 심각도: Critical N개 / High N개 / Medium N개
- 권장 조치: [즉시 수정 / 다음 커밋에서 / 검토 후 판단]
```

### 6. 실패 패턴 대조 (자동)

점검 결과의 fail 항목을 `docs/checklists/ux-known-issues.md`의 기존 패턴과 대조한다.

- 기존 패턴과 일치하면: "기존 패턴 KI-00X 재발" 경고 + 발견 횟수 증가
- 새로운 유형의 fail이면: "신규 패턴 후보" 표시
- 대조 후 known-issues 파일의 발견 횟수를 업데이트한다

### 7. 이력 저장 (--save 모드)

`--save` 플래그 사용 시 점검 결과를 파일로 저장:

- 저장 위치: `docs/checklists/history/YYYY-MM-DD-{화면유형}.md`
- 이전 이력이 존재하면 통과율 비교 표시 ("18/32 → 24/32, +6 개선")
- 궁극의 형태 % 트렌드 추적

## 사용 예시

```
/ux-check                          # 최근 변경 전체 점검
/ux-check 홈 화면                  # 특정 화면 지정 (홈 프로필 적용)
/ux-check components/home/*.tsx    # 파일 지정
/ux-check --quick                  # Critical+High만 빠르게
/ux-check --save                   # 결과 이력 저장
/ux-check 홈 화면 --quick --save   # 조합 사용
```
